# 构建工具选型

## 定义
前端构建工具选型是指在 Laravel 全栈项目中，根据项目规模、团队技术栈、性能需求，在 Vite、Webpack、Laravel Mix 三套方案中做出合理选择。

## 核心原理

### 三者架构对比

| 维度 | Laravel Mix | Webpack 5 | Vite 5+ |
|------|-------------|-----------|---------|
| 底层引擎 | Webpack 4 | Webpack 5 | esbuild + Rollup |
| 开发服务器 | webpack-dev-server | webpack-dev-server | 原生 ESM + esbuild |
| 模块格式 | CommonJS | CommonJS/ESM | 原生 ESM |
| HMR 粒度 | 模块级全量 | 模块级全量 | 精确到变更模块 |
| 生产打包 | Webpack | Webpack | Rollup |
| 配置复杂度 | 低（封装） | 高 | 中（有预设） |
| Tree Shaking | 基础 | 改进 | 原生 ESM 自动 |
| 启动速度 | 慢（10-30s） | 慢（10-30s） | 快（<1s） |

### 选型决策树

```
新项目？
├── 是 → Vite（Laravel 9+ 默认）
└── 否 → 老项目需要迁移？
    ├── 依赖 Webpack 特有插件 → 保持 Webpack 5
    ├── Laravel Mix 项目且无性能问题 → 保持 Mix
    └── 需要更好的 DX → 迁移到 Vite
```

### 迁移路径
```
Laravel Mix → Vite（推荐）
1. npm install vite laravel-vite-plugin
2. 创建 vite.config.ts
3. 更新 package.json scripts
4. 修改 blade 模板引用
5. 迁移 Mix 特有配置到 Vite 等价物
```

## 常见问题

**Q: Laravel 9+ 必须用 Vite 吗？**
A: 默认使用 Vite，但 Laravel Mix 仍然可用。新项目强烈建议 Vite。

**Q: Webpack 5 还有优势吗？**
A: 在需要复杂的自定义 loader/plugin、Module Federation 微前端时，Webpack 仍有优势。
