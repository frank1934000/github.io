# 面向对象编程（OOP）

## 定义

面向对象编程 = 把「数据」和「操作数据的方法」封装到一起，再用继承/多态在不改旧代码的前提下扩展新行为。对立面是面向过程和函数式编程。

## 三大特性

### 1. 封装（Encapsulation）

把内部状态藏起来，只暴露必要接口：

```php
class BankAccount {
    private int $balance = 0;

    public function deposit(int $amount): void {
        if ($amount <= 0) throw new InvalidArgumentException();
        $this->balance += $amount;
    }
    public function balance(): int { return $this->balance; }
}
```

调用方不需要知道余额怎么存的；以后改成数据库/Redis，外面零改动。

### 2. 继承（Inheritance）

子类自动获得父类的属性和方法：

```php
class Animal {
    public function eat(): string { return 'eating'; }
}
class Dog extends Animal {
    public function bark(): string { return 'woof'; }
}
```

> ⚠️ 继承是强耦合，父类改一行可能炸所有子类。**「优先组合，不要继承」**是设计模式的第一条铁律。

### 3. 多态（Polymorphism）

同一个调用，不同的实现。通过接口/抽象类实现：

```php
interface Payment {
    public function pay(int $amount): bool;
}
class Alipay   implements Payment { /*...*/ }
class WeChatPay implements Payment { /*...*/ }

function checkout(Payment $p, int $amount): void {
    $p->pay($amount);   // 不关心具体是哪种支付
}
```

扩展新支付方式，`checkout()` 一个字都不用改。

## SOLID 五大原则

| 字母 | 名称 | 一句话 | 反例 |
|------|------|--------|------|
| **S** | 单一职责 | 一个类只做一件事 | `User` 类同时管理用户数据 + 发邮件 + 存数据库 |
| **O** | 开闭原则 | 对扩展开放，对修改关闭 | 加新支付方式就要改 if-else |
| **L** | 里氏替换 | 子类必须能替换父类不出错 | `Square extends Rectangle` 违反父类语义 |
| **I** | 接口隔离 | 接口要小，不要逼实现类用不到的方法 | `IBird` 里塞 `fly()`，企鹅类被迫抛异常 |
| **D** | 依赖反转 | 依赖抽象，不依赖具体类 | 控制器直接 `new MysqlUserRepo()` |

## 常见问题

**Q: OOP 是唯一正解吗？**
A: 不是。三种范式各有适用场景。简单脚本用面向过程更清晰，数据管道用函数式更合适。

**Q: 什么时候用继承，什么时候用组合？**
A: 优先组合。只有在「is-a」关系真正成立（且父类稳定）时才用继承。
