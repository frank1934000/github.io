# Eloquent 模型观察者（Observers）

## 定义

Observer 监听模型的生命周期事件（creating/created/updating/updated/deleting/deleted），适合在模型状态变化时执行副作用。

## 创建观察者

```bash
php artisan make:observer OrderObserver --model=Order
```

```php
class OrderObserver {
    public function created(Order $order): void {
        // 订单创建后：扣减库存、发通知
        event(new OrderPlaced($order));
    }

    public function updating(Order $order): void {
        // 状态变更前：记录旧值
        $order->old_status = $order->getOriginal('status');
    }

    public function updated(Order $order): void {
        // 状态变更后：审计日志
        if ($order->wasChanged('status')) {
            AuditLog::record($order, 'status_changed', [
                'from' => $order->old_status,
                'to'   => $order->status,
            ]);
        }
    }

    public function deleting(Order $order): void {
        // 删除前：检查关联数据
        if ($order->items()->exists()) {
            throw new LogicException('订单有关联商品，不能删除');
        }
    }
}
```

## 注册观察者

```php
// AppServiceProvider
Order::observe(OrderObserver::class);
```

## 生命周期事件

| 事件 | 触发时机 | 常见用途 |
|------|----------|----------|
| `creating` | 首次保存前 | 生成 UUID、默认值 |
| `created` | 首次保存后 | 触发事件、发送通知 |
| `updating` | 更新前 | 验证状态流转 |
| `updated` | 更新后 | 审计日志、同步缓存 |
| `saving` | 保存前（创建+更新） | 通用前置逻辑 |
| `saved` | 保存后（创建+更新） | 通用后置逻辑 |
| `deleting` | 删除前 | 检查关联、软删除 |
| `deleted` | 删除后 | 清理关联、触发事件 |

## 踩坑记录

- **Observer 内修改同一模型**：updated 内再 update → 无限循环
- **批量操作不触发**：`Model::where(...)->update(...)` 不走 Observer
- **事务边界**：Observer 在事务内执行，回滚也会触发
- **测试困难**：Observer 内副作用难以 mock

## 常见问题

**Q: Observer 和 Event/Listener 什么时候用哪个？**
A: Observer 专注于单个模型的生命周期；Event/Listener 适合跨模型、跨服务的解耦。
