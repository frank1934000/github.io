# PHP 自动加载机制

## 定义

自动加载（Autoload）= 使用类时才自动包含类文件，而非一开始就将所有类文件 include 进来。也称为 **lazy loading**。

## 演进历程

### 1. `__autoload()`（PHP 5，已废弃）

```php
function __autoload($className) {
    require_once($className . "class.php");
}
```

缺点：只能注册一个函数，多人协作冲突。

### 2. SPL Autoload（PHP 5.1+）

```php
spl_autoload_register(function ($className) {
    require_once("classes/" . $className . ".php");
});
```

支持注册多个加载器，按优先级链式调用。

### 3. Composer PSR-4（现代标准）

```json
// composer.json
{
    "autoload": {
        "psr-4": {
            "App\\": "app/"
        }
    }
}
```

运行 `composer dump-autoload` 生成映射表，`vendor/autoload.php` 一键加载。

## SPL Autoload 函数

| 函数 | 作用 |
|------|------|
| `spl_autoload_register()` | 注册 autoload 函数 |
| `spl_autoload_unregister()` | 注销已注册的函数 |
| `spl_autoload_functions()` | 返回所有已注册的函数 |
| `spl_autoload_call()` | 尝试所有已注册的函数来加载类 |
| `spl_autoload()` | autoload 的默认实现 |

## 优点

1. 使用类之前无需 include 或 require
2. 使用类时才 require/include，实现 lazy loading
3. 无需考虑引入类的实际磁盘地址，逻辑和实体文件分离

## 常见问题

**Q: PSR-4 和 PSR-0 有什么区别？**
A: PSR-0 已废弃。PSR-4 更简洁，命名空间直接映射目录结构，不强制 `_` 转 `/`。

**Q: 为什么 `composer dump-autoload` 后新类找不到？**
A: 检查 `composer.json` 的 autoload 配置、命名空间拼写、以及是否运行了 `composer install`。
