# Eloquent 查询作用域（Scopes）

## 定义

Scopes 是 Eloquent 模型中封装查询条件的机制，避免重复编写相同的 `where` 子句。

## Local Scopes

在模型内定义，调用时手动应用：

```php
class Order extends Model {
    public function scopeActive(Builder $query): Builder {
        return $query->where('status', 'active');
    }

    public function scopeRecent(Builder $query, int $days = 7): Builder {
        return $query->where('created_at', '>=', now()->subDays($days));
    }
}

// 使用
$orders = Order::active()->recent(30)->get();
```

## Global Scopes

自动应用于所有查询：

```php
// 定义
class ActiveScope implements Scope {
    public function apply(Builder $builder, Model $model): void {
        $builder->where('is_deleted', false);
    }
}

// 注册
class Order extends Model {
    protected static function booted(): void {
        static::addGlobalScope(new ActiveScope);
    }
}

// 取消全局 scope
Order::withoutGlobalScope(ActiveScope::class)->get();
```

## 动态条件封装

```php
class OrderFilter {
    public function __invoke(Builder $query, $value): Builder {
        return match ($this->field) {
            'status'   => $query->where('status', $value),
            'date'     => $query->whereDate('created_at', $value),
            'amount'   => $query->where('total', '>=', $value),
            default    => $query,
        };
    }
}
```

## 踩坑记录

- **Global Scope 陷阱**：`withoutGlobalScopes()` 会移除所有全局 scope，包括软删除
- **Scope 返回值**：必须返回 `$query`，否则链式调用断裂
- **性能影响**：每个 scope 都是额外的 where，过多嵌套影响查询计划

## 常见问题

**Q: Local Scope 和 Global Scope 什么时候用哪个？**
A: 99% 的场景用 Local Scope。Global Scope 适合软删除、多租户等「永远生效」的过滤。
