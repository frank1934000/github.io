# 代码风格与重构

## 工具概览

- **PHP-CS-Fixer / Pint**：代码风格自动修复
- **Rector**：自动化代码重构与 PHP 版本升级

## PHP-CS-Fixer / Pint

### Pint（Laravel 官方）

```bash
composer require --dev laravel/pint
./vendor/bin/pint              # 检查并修复
./vendor/bin/pint --test       # 只检查不修复
./vendor/bin/pint --diff       # 显示差异
```

### 配置

```php
// pint.json
{
    "preset": "laravel",
    "rules": {
        "ordered_imports": {"sort_algorithm": "alpha"},
        "no_unused_imports": true,
        "single_quote": true,
        "trailing_comma_in_multiline": true
    }
}
```

### 自定义规则

```php
// .php-cs-fixer.php
return (new PhpCsFixer\Config())
    ->setRules([
        '@PSR12' => true,
        'array_syntax' => ['syntax' => 'short'],
        'concat_space' => ['spacing' => 'one'],
    ])
    ->setFinder(
        PhpCsFixer\Finder::create()->in('app')
    );
```

## Rector（自动化重构）

### 安装使用

```bash
composer require --dev rector/rector
vendor/bin/rector process app/ --dry-run  # 预览
vendor/bin/rector process app/            # 执行
```

### PHP 版本升级

```php
// rector.php
use Rector\Config\RectorConfig;

return RectorConfig::configure()
    ->withPaths([__DIR__ . '/app'])
    ->withSets([
        LevelSetList::UP_TO_PHP_82,
        LaravelSetList::LARAVEL_100,
    ]);
```

### 自定义规则

```php
// 将所有 array() 转为 []
use Rector\Php70\Rector\FuncCall\ArrayFunctionCallToArrayDestructRector;

// 将所有 mysql_* 转为 PDO
use Rector\MysqlToMysqli\Rector\FuncCall\MysqlFuncCallToMysqliRector;
```

## CI 集成

```yaml
# GitHub Actions
- name: Check Code Style
  run: ./vendor/bin/pint --test

- name: Static Analysis
  run: ./vendor/bin/phpstan analyse --level=8

- name: Rector Check
  run: ./vendor/bin/rector process --dry-run
```

## 踩坑记录

- **Pint 和 PHP-CS-Fixer 冲突**：只用一个，不要混用
- **Rector 破坏代码**：`--dry-run` 先预览，确认无误再执行
- **团队协作**：统一配置文件提交到 Git，CI 强制检查

## 常见问题

**Q: Pint 和 PHP-CS-Fixer 选哪个？**
A: Laravel 项目推荐 Pint（官方出品、开箱即用）。其他项目用 PHP-CS-Fixer。
