# PHP 静态分析工具

## 定义

- **PHPStan**：PHP 静态类型检查工具，Level 0-8 逐级严格
- **Psalm**：Facebook 出品的静态分析工具，类型推断能力强

## PHPStan

### 安装使用

```bash
composer require --dev phpstan/phpstan
vendor/bin/phpstan analyse --level=8 app/
```

### Level 说明

| Level | 检查内容 |
|-------|----------|
| 0 | 基本检查（未定义变量、调用不存在的方法） |
| 1-3 | 类型检查（参数类型、返回类型） |
| 4-5 | 严格类型（null 安全、类型推断） |
| 6-7 | 高级检查（泛型、模板类型） |
| 8 | 最严格（所有可能的类型错误） |

### 配置文件

```neon
# phpstan.neon
parameters:
    level: 8
    paths:
        - app
    ignoreErrors:
        - '#Call to an undefined method Illuminate\\Support\\HigherOrder#'
    reportUnmatchedIgnoredErrors: false
```

### 渐进式升级

```bash
# 从 Level 5 开始，逐步升级
vendor/bin/phpstan analyse --level=5 app/ --generate-baseline
# 修复后再升 Level
```

## Psalm

```bash
composer require --dev vimeo/psalm
vendor/bin/psalm --init
vendor/bin/psalm
```

### 优势

- 泛型支持更好
- 类型推断更智能
- 支持 `@psalm-assert`、`@psalm-return` 等高级注解

## PHPStan vs Psalm

| 特性 | PHPStan | Psalm |
|------|---------|-------|
| Level 系统 | ✅ 0-8 | ❌ Issue 级别 |
| 泛型支持 | 一般 | 强 |
| Laravel 支持 | ✅ larastan | ✅ psalm-laravel-plugin |
| 社区活跃度 | 更高 | 稳定 |

## 踩坑记录

- **Level 跳级**：从 0 直接到 8 会报几千个错误 → 渐进式升级
- **误报**：复杂泛型或 Laravel 宏可能误报 → 用 baseline 忽略
- **CI 集成**：每次 PR 自动检查，阻止新类型错误引入
