# 进程、线程与协程

## 定义

| 概念 | 定义 | 资源开销 | 切换方式 |
|------|------|----------|----------|
| **进程** | 独立的程序实例，拥有独立内存空间 | 重（MB 级） | OS 内核态切换 |
| **线程** | 进程内的执行单元，共享进程内存 | 中（KB 级） | OS 内核态切换 |
| **协程** | 用户态的轻量级线程，由程序自行调度 | 轻（KB 级） | 用户态切换（无系统调用） |

## PHP 中的并发模型

### 1. 多进程（FPM 模式）

```
Nginx → FPM Master → Worker 1 → 处理请求 → 返回
                    → Worker 2 → 处理请求 → 返回
                    → Worker N → ...
```

- 每个 worker 是独立进程，天然隔离
- **不能共享状态**（需要 Redis/DB 做中间存储）

### 2. 多线程（pthreads 扩展）

- PHP 5.x 有 pthreads，但 PHP 7+ 移除
- PHP 8 的 parallel 扩展提供有限支持
- **不推荐**：PHP 生态不支持多线程

### 3. 协程（PHP 8.1 Fibers / Swoole）

```php
// PHP 8.1 Fiber
$fiber = new Fiber(function (): void {
    $value = Fiber::suspend('fiber-suspended');
    echo "Resumed with: $value";
});

$result = $fiber->start();  // 'fiber-suspended'
$fiber->resume('hello');    // "Resumed with: hello"
```

## Swoole 协程

```php
Co\run(function () {
    go(function () {
        Co::sleep(1);
        echo "协程1\n";
    });
    go(function () {
        Co::sleep(0.5);
        echo "协程2\n";
    });
});
// 输出：协程2 → 协程1（并发执行）
```

## 常见问题

**Q: 协程和线程的本质区别？**
A: 协程是用户态调度，切换不需要系统调用，开销极小。线程由 OS 调度，切换需要内核态转换。

**Q: PHP Fiber 和 Swoole 协程有什么区别？**
A: Fiber 是 PHP 8.1 原生支持的底层原语，需要手动 suspend/resume。Swoole 协程是自动调度的，遇到 IO 自动让出。
