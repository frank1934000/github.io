# 数据处理实战

> Batch Job 批量处理、数据导入导出、ETL 管道、分布式 ID 生成——Laravel 大数据量场景的工程化方案。

## 定义

数据处理是 Web 应用中常见但容易出问题的领域。本文覆盖大数据量批量处理、文件导入导出、ETL 数据管道构建、分布式 ID 生成等实战场景。

## 核心原理

### Batch Job 批量处理

大数据量处理的核心挑战是**内存治理**和**进度追踪**：

| 策略 | 适用场景 | 内存占用 |
|------|----------|----------|
| chunk() | 分块读取处理 | 低（每块固定） |
| cursor() | 逐条处理 | 极低 |
| lazy() | 延迟加载 | 极低 |
| 队列化 | 异步批处理 | 零（Worker 处理） |

### 数据导入导出

**大文件处理三原则**：
1. 流式读写（不一次性加载到内存）
2. 队列化处理（异步执行，不阻塞请求）
3. 进度反馈（让用户知道处理进度）

### ETL 管道

```
Extract（抽取）→ Transform（转换）→ Load（加载）
```

- **Extract**：从数据库/API/文件抽取数据
- **Transform**：清洗、转换、聚合
- **Load**：写入目标系统

### 分布式 ID 生成

| 方案 | 特点 | 适用场景 |
|------|------|----------|
| 自增 ID | 简单、有序、有泄露风险 | 内部系统 |
| UUID v4 | 无序、无泄露风险 | 通用 |
| ULID | 时间有序、可排序 | 需要排序的场景 |
| Snowflake | 时间有序、分布式唯一 | 高并发分布式 |
| UUID v7 | 时间有序、标准化 | 现代应用首选 |

### 大文件导入

来自博客：[Laravel 数据导入导出实战：Excel/CSV 大文件处理与队列化踩坑记录](/2026/06/01/Laravel-数据导入导出实战-Excel-CSV-大文件处理与队列化踩坑记录/)

```php
// 队列化导入，不阻塞 HTTP 请求
class ImportUsers implements ShouldQueue
{
    use InteractsWithQueue, Queueable;

    public function handle(): void
    {
        $file = $this->file;
        $reader = Reader::createFromPath($file);

        foreach ($reader->getRecords() as $record) {
            User::updateOrCreate(
                ['email' => $record['email']],
                ['name' => $record['name']]
            );

            // 每 1000 条释放一次内存
            if ($this->batchSize % 1000 === 0) {
                gc_collect_cycles();
            }
        }
    }
}
```

### 分布式 ID 选型

来自博客：[分布式 ID 生成实战：Snowflake/ULID/UUIDv7 在 Laravel 中的选型](/2026/06/01/分布式ID生成实战-Snowflake-ULID-UUIDv7在Laravel中的选型/)

```php
// ULID
use Symfony\Component\Uid\Ulid;
$id = new Ulid(); // 01ARZ3NDEKTSV4RRFFQ69G5FAV 时间有序

// Snowflake
class SnowflakeIdGenerator
{
    public function next(): string
    {
        $timestamp = (int)(microtime(true) * 1000);
        $workerId = config('app.worker_id', 1);
        $sequence = $this->nextSequence();

        return (($timestamp - EPOCH) << 22)
            | ($workerId << 12)
            | $sequence;
    }
}
```

## 常见问题

**Q: chunk 和 cursor 该选哪个？**
A: chunk 适合需要批量操作（如 bulk insert）的场景；cursor 适合逐条处理且内存敏感的场景。cursor 不支持 eager loading。

**Q: UUID v7 和 ULID 有什么区别？**
A: 两者都是时间有序的 ID。UUID v7 是 IETF 标准（RFC 9562），ULID 是社区规范。UUID v7 更标准化，ULID 更紧凑（26 字符 vs 36 字符）。

**Q: Snowflake 的 worker_id 怎么分配？**
A: 可以用环境变量、配置文件、ZooKeeper/etcd 自增分配、或 Redis INCR 动态分配。确保同一时间每个实例的 worker_id 唯一。
