# 性能优化

## 定义
PHP 和 Laravel 的性能优化涵盖字节码缓存、常驻内存、协程并发、连接池等多个层次。博客中覆盖了 OPcache、Octane/Swoole、Fiber 并发、PHP-FPM 调优、缓存策略等实践。

## 核心原理

### OPcache（字节码缓存）
- PHP 脚本编译为 Opcodes 后缓存在内存中
- opcache.revalidate_freq 控制文件检查频率
- opcache.max_accelerated_files 限制缓存文件数
- 生产环境配置最佳实践
- 内存优化与预加载（Preloading）

### Octane / Swoole（常驻内存）
- 传统 FPM：每次请求 boot/teardown 整个应用
- Octane：应用常驻内存，跳过重复 boot
- Swoole 协程并发：单进程处理多个请求
- RoadRunner：Go 驱动的应用服务器
- 注意事项：全局状态污染、内存泄漏检测

### PHP-FPM 调优
- 长连接 vs 短连接的性能差异
- pm.max_children / pm.start_servers 配置
- 数据库连接池（pconnect）与 FPM 的关系
- 慢日志分析与 Worker 进程监控

### Fiber 并发（PHP 8.1）
- 原生协程，支持 suspend/resume
- Laravel Concurrency facade：并发执行多个任务
- 并发 API 聚合与错误隔离
- 与 Swoole 协程的区别

### 缓存策略
- Route 缓存（route:cache）：路由注册加速
- Config 缓存（config:cache）：配置文件合并
- View 缓存（view:cache）：编译模板预生成
- Query 缓存：数据库查询结果缓存
- 应用级缓存：Redis/Memcached 驱动

### 性能预算
- Lighthouse CI 前端性能监控
- k6 API 响应时间预算
- 预算驱动开发（Budget-Driven Development）
- 持续性能回归检测

## 常见问题
- **Octane 有什么坑？** 全局变量污染、单例状态残留、内存泄漏，需要特别注意代码无状态
- **OPcache 什么时候会失效？** 文件修改后超过 revalidate_freq、opcache_reset() 调用
- **FPM 和 Octane 能共存吗？** 可以，FPM 处理普通请求，Octane 处理高并发 API
