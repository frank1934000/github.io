# Laravel Dusk 浏览器自动化测试

## 定义

Laravel Dusk 是基于 ChromeDriver 的浏览器自动化测试框架，用于 E2E（端到端）测试。

## 安装配置

```bash
composer require --dev laravel/dusk
php artisan dusk:install
php artisan dusk
```

## 基本用法

```php
class OrderFlowTest extends DuskTestCase {
    use DatabaseMigrations;

    public function test_complete_order_flow(): void {
        $user = User::factory()->create();

        $this->browse(function (Browser $browser) use ($user) {
            $browser->loginAs($user)
                ->visit('/products')
                ->click('@add-to-cart')
                ->visit('/cart')
                ->click('@checkout')
                ->waitFor('@payment-form')
                ->type('card_number', '4242424242424242')
                ->click('@pay')
                ->waitForText('订单创建成功')
                ->assertSee('订单号');
        });
    }
}
```

## 选择器最佳实践

```php
// ✅ 使用 data-dusk 属性
<button data-dusk="submit-btn">提交</button>
$browser->click('@submit-btn');

// ✅ 使用语义化选择器
$browser->type('@email-input', 'test@example.com');

// ❌ 避免 CSS 选择器（脆弱）
$browser->click('.btn.btn-primary:nth-child(3)');
```

## CI 集成

```yaml
# GitHub Actions
- name: Run Dusk Tests
  run: |
    php artisan serve &
    php artisan dusk
```

## 踩坑记录

- **动态等待**：不要用 `sleep()`，用 `waitFor()` / `waitForText()`
- **选择器变更**：前端重构后选择器失效 → 用 `data-dusk` 属性
- **测试速度**：Dusk 测试比单元测试慢 100 倍，只测关键流程
- **并行执行**：多个 Dusk 测试同时跑可能端口冲突

## 常见问题

**Q: Dusk 和 Cypress 选哪个？**
A: Dusk 适合 Laravel 项目，和框架集成紧密；Cypress 适合纯前端项目。

**Q: E2E 测试要覆盖多少？**
A: 只覆盖核心业务流程（注册、下单、支付）。细节用单元测试覆盖。
