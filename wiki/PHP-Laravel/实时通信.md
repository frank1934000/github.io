# 实时通信

## 定义
Laravel 提供了多种实时通信方案，从轻量的 SSE 到全双工的 WebSocket。博客中覆盖了 SSE、Laravel Reverb、WebSocket、FCM Web Push 等实践。

## 核心原理

### SSE（Server-Sent Events）
- 服务端单向推送，基于 HTTP 长连接
- Text/Event-Stream 协议
- 自动重连机制
- 轻量方案，适合通知、进度条、实时数据流
- Laravel 原生支持（StreamedResponse）

### Laravel Reverb
- Laravel 官方 WebSocket 服务器
- Pusher 协议兼容（可替代 Pusher）
- 频道（Channel）：公共、私有、存在
- 事件广播（Broadcasting）
- 多实例部署与水平扩展
- 订单状态实时推送实战

### WebSocket
- 全双工通信协议
- ws:// 和 wss:// 安全连接
- 心跳保活机制
- 与 Reverb/Pusher/Soketi 的关系

### FCM Web Push（浏览器推送）
- Firebase Cloud Messaging 集成
- Service Worker 注册与订阅
- Web Push API 标准
- 推送通知实战

### Livewire / Volt（全栈组件）
- 服务端渲染 + 实时更新
- 单文件 Blade 组件
- Wire:model 双向绑定
- 与 Reverb 集成的实时更新

## 常见问题
- **SSE 和 WebSocket 选哪个？** 单向推送用 SSE（更简单），双向通信用 WebSocket
- **Reverb 和 Pusher 什么关系？** Reverb 实现了 Pusher 协议，可以免费替代 Pusher 云服务
- **WebSocket 多实例怎么部署？** 需要 sticky session 或 Redis pub/sub 做消息广播
