# Laravel 队列深度实战

## 定义

队列将耗时任务（发邮件、生成报表、同步数据）异步执行，不阻塞主请求。

## Job 定义

```bash
php artisan make:job ProcessOrder
```

```php
class ProcessOrder implements ShouldQueue {
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public int $tries = 3;           // 最大重试次数
    public int $backoff = 60;        // 重试间隔（秒）
    public int $timeout = 120;       // 超时时间
    public int $maxExceptions = 3;   // 最大异常次数

    public function __construct(public Order $order) {}

    public function handle(): void {
        // 业务逻辑
    }

    public function failed(Throwable $e): void {
        // 失败处理：通知管理员、记录日志
    }
}

// 调度
ProcessOrder::dispatch($order);
ProcessOrder::dispatch($order)->onQueue('high')->delay(now()->addMinutes(5));
```

## 批量任务

```php
$jobs = collect($orders)->map(fn ($o) => new ProcessOrder($o));
Bus::batch($jobs)
    ->then(fn () => Log::info('全部完成'))
    ->catch(fn () => Log::error('有任务失败'))
    ->onQueue('batch')
    ->dispatch();
```

## 延迟队列

```php
// 延迟 5 分钟
ProcessOrder::dispatch($order)->delay(now()->addMinutes(5));

// 定时任务中检查过期订单
$schedule->call(function () {
    Order::where('expires_at', '<', now())
        ->each(fn ($o) => CancelOrder::dispatch($o));
})->everyMinute();
```

## 队列驱动对比

| 驱动 | 优点 | 缺点 |
|------|------|------|
| Redis | 高性能、支持延迟队列 | 需要额外运维 |
| Database | 无需额外组件 | 性能一般、锁竞争 |
| SQS | AWS 托管、高可用 | 延迟高、功能受限 |

## 踩坑记录

- **序列化模型**：Job 中传 Eloquent 模型，序列化后加载关联可能 N+1
- **超时未设**：默认 60s，长任务需手动设置 `$timeout`
- **队列优先级**：高优先级队列饿死低优先级 → 用 `--queue=high,default,low`

## 常见问题

**Q: Job 失败了怎么办？**
A: `$tries` 重试耗尽后进入 `failed_jobs` 表，可通过 `failed:retry` 手动重试或配置死信队列。

**Q: 如何保证 Job 不丢失？**
A: Redis 用 RDB+AOF 持久化；Database 天然持久化；关键 Job 用 Inbox/Outbox 模式。
