# Laravel 授权管理（Policies & Gates）

## 定义

- **Policy**：针对模型的授权逻辑（如 OrderPolicy）
- **Gate**：通用的授权检查（不绑定模型）
- **RBAC**：基于角色的访问控制

## Policy

```bash
php artisan make:policy OrderPolicy --model=Order
```

```php
class OrderPolicy {
    public function view(User $user, Order $order): bool {
        return $user->id === $order->user_id;
    }

    public function update(User $user, Order $order): bool {
        return $user->id === $order->user_id
            && $order->status === 'pending';
    }

    public function cancel(User $user, Order $order): bool {
        return $user->hasRole('admin')
            || ($user->id === $order->user_id && $order->canBeCancelled());
    }
}

// 控制器中使用
public function update(Request $request, Order $order): JsonResponse {
    $this->authorize('update', $order);
    // ...
}
```

## Gate

```php
// 定义
Gate::define('view-admin-dashboard', function (User $user) {
    return $user->hasRole('admin');
});

// 使用
if (Gate::allows('view-admin-dashboard')) {
    // 允许访问
}

// 中间件
Route::middleware('can:view-admin-dashboard')->group(function () {
    Route::get('/admin', [AdminController::class, 'index']);
});
```

## RBAC 角色权限

```php
// 角色定义
class Role {
    case Admin;
    case Manager;
    case User;
}

// 多租户隔离
class TenantScope implements Scope {
    public function apply(Builder $builder, Model $model): void {
        $builder->where('tenant_id', auth()->user()->tenant_id);
    }
}
```

## 踩坑记录

- **Policy 未注册**：自定义 Policy 类名不规范导致未自动发现 → 手动注册
- **Super Admin 绕过**：`before()` 方法可绕过所有检查，慎用
- **多租户泄漏**：忘记加 tenant scope 导致跨租户数据可见

## 常见问题

**Q: Policy 和 Gate 什么时候用哪个？**
A: Policy 绑定模型，适合 CRUD 授权；Gate 不绑定模型，适合通用权限检查。
