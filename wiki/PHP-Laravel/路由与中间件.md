# 路由与中间件

## 定义
路由是 Laravel 请求入口的分发器，中间件是请求管道中的过滤层。博客中覆盖了路由注册、中间件栈、请求管道、链路追踪等实践。

## 核心原理

### 路由机制
- 路由注册：Route::get/post/put/delete/resource/apiResource
- 路由参数与约束（where 正则）
- 路由分组（prefix、middleware、namespace）
- 路由缓存（route:cache）与性能优化
- 命名路由与 URL 生成

### 中间件栈
- 全局中间件 vs 路由中间件 vs 中间件组
- 中间件参数传递（middleware('throttle:60,1')）
- Terminable 中间件（响应发送后执行）
- 中间件优先级排序

### 请求管道
- 请求进入 → 全局中间件 → 路由中间件 → 控制器 → 响应
- before/after 钩子模式
- 中间件短路（直接返回响应）

### 链路追踪
- 中间件中注入 Trace ID / Request ID
- 跨服务请求头透传
- 日志关联与排查

## 常见问题
- **中间件的执行顺序怎么控制？** 通过 $middlewarePriority 数组或中间件组排序
- **什么时候用 Terminable？** 需要在响应发送后执行的操作（如写日志、发送事件）
- **路由缓存有什么限制？** 不能使用闭包路由，必须用控制器方法
