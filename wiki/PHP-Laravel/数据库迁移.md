# Laravel 数据库迁移（Migrations）

## 定义

Migration 是数据库结构的版本控制，用 PHP 代码定义表结构变更，可追踪、可回滚、可团队协作。

## 基本用法

```bash
php artisan make:migration create_orders_table
php artisan migrate
php artisan migrate:rollback
php artisan migrate:status
```

```php
class CreateOrdersTable extends Migration {
    public function up(): void {
        Schema::create('orders', function (Blueprint $table) {
            $table->id();
            $table->foreignId('user_id')->constrained();
            $table->decimal('total', 10, 2);
            $table->enum('status', ['pending', 'paid', 'shipped']);
            $table->timestamps();
            $table->softDeletes();

            $table->index(['user_id', 'status']);
        });
    }

    public function down(): void {
        Schema::dropIfExists('orders');
    }
}
```

## 零停机迁移策略

### 添加列（安全）

```php
// 新列默认 nullable，不锁表
$table->string('new_column')->nullable();
```

### 修改列（危险）

```php
// MySQL 需要 `doctrine/dbal` 包
// 大表修改会锁表，用 pt-online-schema-change 或 gh-ost
$table->string('name', 100)->change();
```

### 重命名列（危险）

```php
// 避免直接重命名，先加新列 → 迁移数据 → 删旧列
$table->renameColumn('old_name', 'new_name');  // ⚠️ 慎用
```

## 数据迁移

```php
class MigrateOrderStatus extends Migration {
    public function up(): void {
        // 结构变更
        Schema::table('orders', function (Blueprint $table) {
            $table->string('status_v2')->default('pending');
        });

        // 数据迁移
        DB::table('orders')
            ->where('status', 1)
            ->update(['status_v2' => 'paid']);

        // 清理旧列
        Schema::table('orders', function (Blueprint $table) {
            $table->dropColumn('status');
        });
    }
}
```

## 踩坑记录

- **生产环境迁移**：先在 staging 测试，大表用在线 DDL 工具
- **回滚不干净**：`down()` 写得不完整，回滚后残留列/索引
- **迁移顺序**：多人协作时迁移文件名冲突，用时间戳而非序号

## 常见问题

**Q: 生产环境能直接 `migrate` 吗？**
A: 小表可以。大表（百万级）要用 `pt-online-schema-change` 或 `gh-ost` 做在线 DDL。
