# 接口与抽象类

## 定义

- **接口（Interface）**：定义行为契约，不含实现。一个类可实现多个接口。
- **抽象类（Abstract Class）**：可包含部分实现。一个类只能继承一个抽象类。

## 核心区别

| 特性 | 接口 | 抽象类 |
|------|------|--------|
| 方法实现 | 不能有（PHP 8.0 前） | 可以有 |
| 属性 | 只能有常量 | 可以有普通属性 |
| 多实现 | ✅ 一个类可实现多个 | ❌ 只能继承一个 |
| 构造函数 | 不能有 | 可以有 |
| 访问修饰符 | 都是 public | 可以 protected/private |

## 代码示例

```php
// 接口 —— 定义行为契约
interface Payable {
    public function pay(float $amount): bool;
    public function refund(float $amount): bool;
}

// 抽象类 —— 提供部分实现
abstract class PaymentGateway implements Payable {
    protected string $apiKey;

    public function __construct(string $apiKey) {
        $this->apiKey = $apiKey;
    }

    // 子类必须实现
    abstract protected function doPay(float $amount): bool;

    // 公共逻辑
    public function pay(float $amount): bool {
        if ($amount <= 0) throw new InvalidArgumentException();
        return $this->doPay($amount);
    }
}

// 具体实现
class AlipayGateway extends PaymentGateway {
    protected function doPay(float $amount): bool { /*...*/ }
    public function refund(float $amount): bool { /*...*/ }
}
```

## 常见问题

**Q: 什么时候用接口，什么时候用抽象类？**
A: 需要多继承或定义跨层级契约时用接口；需要共享实现代码时用抽象类。

**Q: PHP 8 接口可以有方法体了？**
A: PHP 8.0 引入了接口默认方法（default methods），但不推荐滥用。
