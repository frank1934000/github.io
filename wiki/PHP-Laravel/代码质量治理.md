# 代码质量治理

## 定义
PHP/Laravel 项目的代码质量治理涵盖代码风格统一、静态类型分析、自动化重构和 AI 辅助审查。博客中覆盖了 Pint、PHPStan、PHP-CS-Fixer、Rector、AI Code Review 等工具链。

## 核心原理

### Pint（代码风格自动修复）
- Laravel 官方代码风格工具（基于 PHP-CS-Fixer）
- preset: laravel / psr12
- 自动修复 vs 仅检查（--test）
- CI 流水线集成门禁

### PHPStan（静态类型分析）
- Level 0-8 渐进式升级路径
- Level 8 最严格检查（含 mixed 类型）
- Baseline 文件：忽略存量问题
- Laravel Larastan 扩展
- 类型安全最佳实践

### PHP-CS-Fixer（代码规范）
- 风格规则配置（.php-cs-fixer.php）
- 与 Pint 的关系与区别
- 团队协作的代码规范自动化

### Rector（自动化重构）
- PHP 版本升级自动化
- 代码模式批量替换
- 与 CI 集成的渐进式重构

### AI 辅助代码审查
- Claude/GPT 提升 Code Review 效率
- AI 生成 Review Comments
- 代码异味检测
- 与 PR 工作流集成

### 设计模式与幂等性
- 消息幂等性设计模式
- Inbox/Outbox 模式
- 去重表与重试补偿
- 事件消费的幂等保证

## 常见问题
- **PHPStan 和 Psalm 选哪个？** PHPStan 社区更活跃、Larastan 支持更好，推荐 PHPStan
- **Level 8 太严格怎么办？** 用 baseline 忽略存量问题，新代码强制 Level 8
- **Pint 和 PHP-CS-Fixer 需要同时用吗？** 不需要，Pint 已经封装了 PHP-CS-Fixer，Laravel 项目用 Pint 即可
