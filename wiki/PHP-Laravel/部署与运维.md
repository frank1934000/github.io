# 部署与运维

## 定义
Laravel 应用的部署与运维覆盖从本地 Docker 开发环境到云平台部署、服务网格、消息总线和可观测性。博客中覆盖了 Docker Compose、Laravel Vapor、Laravel Cloud、Istio、NATS、多租户 SaaS 等实践。

## 核心原理

### Docker Compose（本地开发环境）
- PHP-FPM + Nginx + MySQL + Redis 容器编排
- 多服务一键启动
- Volume 挂载与热重载
- 微服务部署经验

### Laravel Vapor（AWS Serverless）
- Lambda + API Gateway 无服务器架构
- 冷启动治理与优化
- 报表导出与异步任务拆分
- 临时存储踩坑记录
- 成本分析与预算

### Laravel Cloud（官方 PaaS）
- 一键部署与自动扩缩
- 开发者体验评测
- 与 Vapor 的对比

### Istio（服务网格）
- mTLS 自动加密
- 灰度发布（金丝雀）
- 连接池优化
- 流量管理与故障注入

### NATS JetStream（消息总线）
- 订单通知削峰
- Ack 确认与重投机制
- KV 配置同步
- 与 Laravel Queue 集成

### Scheduler 多实例部署
- onOneServer 限制与 K8s CronJob 取舍
- Redis 原子锁保护
- 重入防止策略

### Composer 自动化
- 脚本钩子（pre/post-install）
- CI/CD 集成
- 生产环境优化安装（--no-dev --optimize-autoloader）

### 可观测性
- kkday/log + kkday/monitor + kkday/tracing
- 日志聚合、指标采集与分布式追踪
- Laravel Telescope（开发环境）

### 多租户 SaaS
- 共享库 + tenant_id 隔离
- 独立库模式
- 混合架构设计
- 租户识别、连接切换、队列串租

## 常见问题
- **Vapor 冷启动怎么优化？** 使用 Provisioned Concurrency、精简依赖、预加载关键类
- **Docker 开发环境和生产不一致怎么办？** 使用多阶段构建，开发和生产用同一个 Dockerfile 基础镜像
- **多租户选共享库还是独立库？** 数据隔离要求高选独立库，成本敏感选共享库 + RLS
