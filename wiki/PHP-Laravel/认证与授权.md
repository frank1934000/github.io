# 认证与授权

## 定义
Laravel 提供了完整的认证与授权体系，从 API Token 到 OAuth2、从角色权限到无密码登录。博客中覆盖了 Sanctum/Passport、JWT、Policies/Gates、RBAC、WebAuthn/Passkey、OWASP 安全防护等主题。

## 核心原理

### Sanctum（轻量 API 认证）
- SPA 认证：Cookie + CSRF Token
- API Token：每个用户生成多个 Token（能力/范围控制）
- 移动端 Token 认证
- 适合前后端分离的 SPA 和移动应用

### Passport（完整 OAuth2）
- Authorization Code Grant（第三方授权）
- Client Credentials Grant（服务间认证）
- Personal Access Token（个人访问令牌）
- 适合需要完整 OAuth2 流程的场景

### JWT 与 Token 管理
- Firebase JWT vs 自建 Token 选型
- 双 Token 轮换（Access Token + Refresh Token）
- 多端登录管理
- Token 刷新机制与并发续签
- 黑名单与吊销策略

### Policies 与 Gates（授权）
- Policy：模型级别的授权策略（PostPolicy、OrderPolicy）
- Gate：非模型级的通用授权检查
- before() 方法：管理员绕过检查
- 中间件授权：can、ability、middleware

### RBAC（基于角色的访问控制）
- 角色（Role）与权限（Permission）模型
- spatie/laravel-permission 集成
- 多租户权限隔离
- 动态权限分配

### WebAuthn / Passkey（无密码登录）
- FIDO2 标准的无密码认证
- 设备注册与挑战-响应流程
- 后台无密码登录实践
- 挑战过期与设备绑定

### OWASP Top 10 安全防护
- SQL 注入：参数绑定、Eloquent 安全查询
- XSS：Blade {{ }} 自动转义、CSP 头
- CSRF：VerifyCsrfToken 中间件
- SSRF：URL 白名单、内部网络隔离
- 多层防御策略

## 常见问题
- **Sanctum 和 Passport 选哪个？** SPA/移动 API 用 Sanctum，需要完整 OAuth2（第三方授权）用 Passport
- **JWT 存哪里？** 前端存 HttpOnly Cookie 或 localStorage，HttpOnly Cookie 更安全
- **RBAC 和 ABAC 的区别？** RBAC 按角色授权，ABAC 按属性（时间、IP、资源属性）授权
