# Laravel 认证方案（Sanctum / Passport）

## 定义

- **Sanctum**：轻量级 SPA/API 令牌认证，适合前后端分离项目
- **Passport**：完整的 OAuth 2.0 实现，适合第三方授权

## Sanctum

### 安装配置

```bash
composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate
```

### API 令牌认证

```php
// 创建令牌
$token = $user->createToken('api-token')->plainTextToken;

// 中间件保护
Route::middleware('auth:sanctum')->get('/user', fn (Request $req) => $req->user());

// 权限范围
$token = $user->createToken('read-only', ['read'])->plainTextToken;

// 检查权限
$user->tokenCan('read');
```

### SPA 认证

```php
// 前端通过 Cookie 认证（同域）
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/api/user', fn (Request $req) => $req->user());
});
```

## Passport

### 安装配置

```bash
composer require laravel/passport
php artisan passport:install
```

### OAuth 2.0 授权流程

```php
// 授权码模式
Route::get('/auth/redirect', function () {
    return Socialite::driver('github')->redirect();
});

Route::get('/auth/callback', function () {
    $user = Socialite::driver('github')->user();
    // 创建/更新用户，颁发 token
});

// 自定义 Grant Type
class WechatGrant extends GrantTypeAbstract {
    public function respondToAccessTokenRequest(): void {
        // 微信扫码登录逻辑
    }
}
```

## 双 Token 轮换

```php
// Access Token（短期，15分钟）
// Refresh Token（长期，7天）

// Token 刷新
public function refresh(Request $request): JsonResponse {
    $user = $request->user();
    $user->tokens()->delete();
    $token = $user->createToken('refreshed');
    return response()->json([
        'access_token' => $token->plainTextToken,
        'expires_in' => config('sanctum.expiration') * 60,
    ]);
}
```

## 踩坑记录

- **多端登录冲突**：同用户多设备登录，Token 覆盖 → 用 `createToken` 按设备创建
- **Token 过期处理**：前端需监听 401 自动刷新 Token
- **并发续签**：多个请求同时触发 Token 刷新 → 用锁机制

## 常见问题

**Q: Sanctum 和 Passport 选哪个？**
A: Sanctum 适合自家 SPA 和移动端 API；Passport 需要 OAuth 2.0 协议（第三方授权）时才用。

**Q: Token 存在哪里？**
A: Sanctum 存 `personal_access_tokens` 表；Passport 存 `oauth_access_tokens` 表。
