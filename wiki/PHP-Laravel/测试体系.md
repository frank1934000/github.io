# 测试体系

## 定义
Laravel 提供了完整的测试工具链，从单元测试到浏览器自动化、从快照回归到变异测试。博客中覆盖了 Pest、PHPUnit、ParaTest、Dusk、Snapshot Testing 等实践。

## 核心原理

### Pest PHP（优雅测试框架）
- 基于 PHPUnit 的上层封装
- 优雅的 expect() API
- 数据驱动测试（dataset）
- 自定义 Expectations
- Arch Testing（架构规则测试）
- Mutation Testing（变异测试）

### PHPUnit（单元测试基础）
- TestCase 基类与 setUp/tearDown
- Mock 与 Stub（Mockery 集成）
- 断言方法大全
- 测试覆盖率（coverage）

### ParaTest（并行测试）
- 多进程并行执行测试用例
- 大幅缩短 CI 测试时间
- 与 PHPUnit 兼容
- 数据库隔离策略

### Snapshot Testing（快照测试）
- API 响应快照回归测试
- 自动检测接口契约变化
- 快照更新与版本管理
- 用「拍快照」守护接口契约

### Dusk（浏览器 E2E 测试）
- Chrome 浏览器自动化
- 页面元素选择器治理
- 动态等待策略
- CI 流水线集成
- 截图与视频录制

### 并发测试
- 数据驱动测试（多组输入验证）
- 并发执行加速
- 测试数据工厂与 Seeder
- 数据库事务回滚

## 常见问题
- **Pest 和 PHPUnit 选哪个？** Pest 语法更优雅，底层兼容 PHPUnit，推荐新项目用 Pest
- **100% 覆盖率有必要吗？** 核心业务逻辑建议高覆盖，工具类/配置类可以降低要求
- **E2E 测试太慢怎么办？** 只覆盖关键路径，用 Snapshot Testing 替代部分回归测试
