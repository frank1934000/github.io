# Laravel 缓存策略

## 定义

Laravel 提供统一的缓存抽象层，支持多种后端（Redis/Memcached/Database/File），覆盖路由配置、视图、查询等多层缓存。

## 缓存后端对比

| 后端 | 性能 | 持久化 | 适用场景 |
|------|------|--------|----------|
| Redis | 高 | ✅ | 生产推荐 |
| Memcached | 高 | ❌ | 简单 KV |
| Database | 中 | ✅ | 小项目 |
| File | 低 | ✅ | 开发环境 |

## 多层缓存

### Route/Config/View 缓存

```bash
php artisan route:cache        # 路由缓存
php artisan config:cache       # 配置缓存
php artisan view:cache         # 视图缓存
php artisan event:cache        # 事件缓存
```

### Query 缓存

```php
// 手动缓存查询结果
$users = Cache::remember('users.active', 3600, function () {
    return User::where('active', true)->get();
});

// 模型自动缓存
class User extends Model {
    protected static function booted(): void {
        static::saved(fn () => Cache::forget('users.active'));
    }
}
```

### Response Cache（全页缓存）

```php
// spatie/laravel-response-cache
Route::middleware('cacheResponse:60')->group(function () {
    Route::get('/products', [ProductController::class, 'index']);
});
```

### 局部缓存

```blade
@cache('product-' . $product->id, 3600)
    <div>{{ $product->name }}</div>
@endcache
```

## 缓存失效策略

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| TTL | 不常变化的数据 | 配置、字典表 |
| 事件驱动 | 数据变更即失效 | 用户信息更新 |
| 主动刷新 | 定时任务更新 | 报表数据 |
| 缓存穿透保护 | 高并发热点数据 | mutex + rebuild |

## 踩坑记录

- **缓存雪崩**：大量 key 同时过期 → 随机 TTL
- **缓存击穿**：热点 key 过期瞬间大量请求 → 互斥锁重建
- **缓存穿透**：查询不存在的数据 → 布隆过滤器或缓存空值

## 常见问题

**Q: 缓存和数据库的一致性怎么保证？**
A: 最终一致性方案：写入时删除缓存（Cache Aside），下次读取时重建。强一致性用分布式锁。
