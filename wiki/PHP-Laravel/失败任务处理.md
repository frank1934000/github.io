# Laravel 失败任务处理

## 定义

当队列 Job 执行失败时，需要重试机制、死信队列和告警通知来保证任务不丢失。

## 重试机制

```php
class ProcessOrder implements ShouldQueue {
    public int $tries = 3;           // 最大重试次数
    public int $backoff = 60;        // 重试间隔（秒）
    public int $maxExceptions = 3;   // 异常次数上限

    // 指数退避
    public function backoff(): array {
        return [60, 300, 900];  // 1分钟 → 5分钟 → 15分钟
    }
}
```

## 失败处理

```php
class ProcessOrder implements ShouldQueue {
    public function failed(Throwable $e): void {
        // 1. 通知管理员
        Notification::route('slack', '#alerts')
            ->notify(new JobFailedNotification($this->order, $e));

        // 2. 记录失败日志
        FailedJob::create([
            'job_id' => $this->job->getJobId(),
            'order_id' => $this->order->id,
            'exception' => $e->getMessage(),
        ]);

        // 3. 回滚副作用
        $this->order->update(['status' => 'processing_failed']);
    }
}
```

## 死信队列

```php
// config/queue.php
'redis' => [
    'driver' => 'redis',
    'queue' => 'default',
    'retry_after' => 90,  // 超过此时间未完成视为失败
],

// 监听死信队列
class DeadLetterProcessor {
    public function handle(array $payload): void {
        // 处理最终失败的任务
        Log::critical('Dead letter received', $payload);
    }
}
```

## 手动管理失败任务

```bash
# 查看失败任务
php artisan queue:failed

# 重试所有失败任务
php artisan queue:retry all

# 重试指定任务
php artisan queue:retry 1,2,3

# 删除失败任务
php artisan queue:forget 1

# 清空所有失败任务
php artisan queue:flush
```

## 踩坑记录

- **failed() 方法中抛异常**：会导致死循环，必须 try-catch
- **重试幂等性**：Job 重试时业务逻辑必须幂等（如用 idempotency key）
- **告警风暴**：大量任务同时失败触发大量告警 → 合并告警、设置冷却
