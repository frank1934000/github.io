# Server-Driven UI

## 定义

Server-Driven UI（服务端驱动 UI）是一种**由后端控制前端页面结构和内容**的架构模式。后端返回 JSON 格式的 UI 描述协议，前端根据协议渲染组件，而非前端硬编码页面布局。这使得 UI 变更只需后端发布，无需客户端更新。

## 核心原理

### 1. 传统前端 vs Server-Driven UI

```
传统 SPA：
  前端硬编码组件 → 调用 API 获取数据 → 前端渲染
  UI 变更 → 前端重新构建部署 → 用户更新客户端

Server-Driven UI：
  后端返回 UI 描述 JSON → 前端按协议渲染
  UI 变更 → 后端发布即可 → 无需客户端更新
```

### 2. UI 描述协议示例

```json
{
  "type": "page",
  "title": "商品详情",
  "children": [
    {
      "type": "image_carousel",
      "data_key": "images",
      "props": { "auto_play": true, "interval": 3000 }
    },
    {
      "type": "text",
      "data_key": "title",
      "props": { "style": "heading_lg" }
    },
    {
      "type": "price_display",
      "data_key": "price",
      "props": { "currency": "CNY", "show_original": true }
    },
    {
      "type": "button",
      "text": "立即购买",
      "action": { "type": "navigate", "target": "/checkout" }
    }
  ]
}
```

### 3. 架构分层

```
┌─────────────────────────────────────────┐
│           BFF (Backend-for-Frontend)     │
│  ┌──────────┐  ┌──────────┐             │
│  │ 数据聚合  │  │ UI 组装  │             │
│  └──────────┘  └──────────┘             │
└─────────────────┬───────────────────────┘
                  │ UI 描述 JSON
┌─────────────────▼───────────────────────┐
│           前端渲染引擎                    │
│  ┌──────────┐  ┌──────────┐             │
│  │ 组件注册  │  │ 协议解析  │             │
│  └──────────┘  └──────────┘             │
└─────────────────────────────────────────┘
```

### 4. 优劣分析

| 优势 | 劣势 |
|------|------|
| UI 变更无需客户端发版 | 前端灵活性受限 |
| A/B 测试更简单（后端控制） | 复杂交互实现困难 |
| 多端统一（iOS/Android/Web） | 调试难度增加 |
| 减少前端维护成本 | 需要维护 UI 协议规范 |
| 紧急修复快速上线 | 首屏渲染可能更慢 |

### 5. 适用场景

- **电商商品详情页**：频繁变更布局、促销组件
- **运营活动页**：后端配置即可上线
- **多端一致性要求高**：一套后端协议驱动多端
- **配置化页面**：管理员后台配置 UI 布局

### 6. 不适用场景

- **高度交互应用**：如在线文档编辑器、设计工具
- **实时性要求极高**：如股票交易界面
- **前端性能敏感**：如游戏、动画密集型应用

### Laravel BFF 实现

1. **UI 组件注册表**：前端注册可渲染的组件类型
2. **BFF UI 组装器**：Laravel 服务根据业务逻辑组装 UI 描述
3. **数据绑定**：`data_key` 字段引用 API 返回的数据字段
4. **条件渲染**：支持 `visible_if` 条件判断
5. **版本管理**：UI 协议版本控制，向前兼容

## 常见问题

### Q: 与低代码平台的区别？
A: 低代码面向非技术人员拖拽构建页面。Server-Driven UI 面向开发者，后端代码控制 UI 结构，更灵活可控。

### Q: 如何处理复杂表单交互？
A: 表单描述可以包含验证规则、联动逻辑、错误提示。但过于复杂的交互仍建议前端硬编码。

### Q: 性能如何优化？
A: (1) UI 描述缓存；(2) 增量更新（diff）；(3) 预加载关键组件；(4) 骨架屏兜底。
