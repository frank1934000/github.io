# 舱壁隔离模式（Bulkhead Pattern）

## 定义

舱壁隔离模式得名于船舶的**水密舱壁**设计——船体被分隔成多个独立舱室，即使一个舱室进水，其他舱室仍能保持干燥，船不会沉没。在软件系统中，将资源（线程池、连接池、队列）隔离到独立的"舱室"中，防止一个组件的故障扩散到整个系统。

## 核心原理

### 无隔离 vs 舱壁隔离

```
❌ 无隔离：共享连接池
┌────────────────────────────────┐
│       共享 DB 连接池 (100)       │
│  ┌────────┐  ┌────────┐       │
│  │订单服务 │  │报表服务 │       │
│  │ 90 连接 │  │ 10 连接 │       │
│  └────────┘  └────────┘       │
│                                │
│  报告查询慢 → 占满连接 → 订单服务也挂了！│
└────────────────────────────────┘

✅ 舱壁隔离：独立连接池
┌──────────────┐  ┌──────────────┐
│ 订单连接池(80) │  │ 报表连接池(20) │
│  ┌────────┐  │  │  ┌────────┐  │
│  │订单服务 │  │  │  │报表服务 │  │
│  └────────┘  │  │  └────────┘  │
└──────────────┘  └──────────────┘
  报告查询慢 → 只影响报表服务 → 订单服务不受影响
```

### Laravel 中的三种隔离维度

#### 1. HTTP Client 连接池隔离

```php
// 不同服务使用独立的 HTTP Client 配置
class OrderService
{
    private PendingRequest $client;

    public function __construct()
    {
        $this->client = Http::timeout(5)
            ->connectTimeout(2)
            ->retry(2, 1000)
            ->withOptions([
                // 独立连接池
                'curl' => [CURLOPT_MAXCONNECTS => 50],
            ]);
    }
}

class ReportService
{
    private PendingRequest $client;

    public function __construct()
    {
        $this->client = Http::timeout(30) // 报表允许更长超时
            ->connectTimeout(5)
            ->withOptions([
                'curl' => [CURLOPT_MAXCONNECTS => 10], // 独立且更小的连接池
            ]);
    }
}
```

#### 2. Queue 队列隔离

```php
// config/queue.php
'connections' => [
    'redis-critical' => [    // 关键业务队列
        'driver' => 'redis',
        'queue' => 'critical',
        'retry_after' => 90,
        'block_for' => 5,
    ],
    'redis-reports' => [     // 报表队列（隔离）
        'driver' => 'redis',
        'queue' => 'reports',
        'retry_after' => 300,
        'block_for' => 5,
    ],
],

// Horizon 配置：独立 worker 进程
'environments' => [
    'production' => [
        'supervisor-critical' => [
            'connection' => 'redis-critical',
            'queue' => ['critical'],
            'processes' => 10,
            'balance' => 'simple',
        ],
        'supervisor-reports' => [
            'connection' => 'redis-reports',
            'queue' => ['reports'],
            'processes' => 3,  // 独立且更少的进程
            'balance' => 'simple',
        ],
    ],
],
```

#### 3. 数据库连接池隔离

```php
// config/database.php
'connections' => [
    'mysql' => [           // 主库：业务读写
        'driver' => 'mysql',
        'pool' => [
            'min_connections' => 10,
            'max_connections' => 50,
        ],
    ],
    'mysql-analytics' => [ // 分析库：报表查询（隔离）
        'driver' => 'mysql',
        'host' => 'analytics-db',
        'pool' => [
            'min_connections' => 2,
            'max_connections' => 10,
        ],
    ],
],
```

### 熔断器配合

舱壁隔离通常与熔断器（Circuit Breaker）配合使用：

```php
// 舱壁 + 熔断
$circuitBreaker = new CircuitBreaker(
    failureThreshold: 5,
    resetTimeout: 30,
);

$result = $circuitBreaker->call(
    fn() => $this->externalService->call(),
    fallback: fn() => $this->getCachedData(), // 降级方案
);
```

## 资源隔离层次

```
┌─────────────────────────────────────────┐
│  线程/进程级隔离（最强）                    │
│  独立 worker 进程 → Horizon 队列           │
├─────────────────────────────────────────┤
│  连接池级隔离（中等）                       │
│  独立 DB/HTTP 连接池 → Laravel config      │
├─────────────────────────────────────────┤
│  逻辑级隔离（最弱）                        │
│  限流 + 超时 + 重试 → RateLimiter          │
└─────────────────────────────────────────┘
```

## 常见问题

**Q: 舱壁隔离会浪费资源吗？**
A: 会。隔离意味着资源不能共享，可能导致利用率下降。需要根据业务重要性分配资源比例（如关键业务 80%、报表 20%）。

**Q: 什么时候需要舱壁隔离？**
A: 当你有"关键路径"和"非关键路径"共享同一资源池时。如果所有服务同等重要且性能特征相似，舱壁隔离的收益不大。
