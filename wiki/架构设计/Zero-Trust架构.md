# 零信任架构（Zero Trust）

## 定义

零信任架构（Zero Trust Architecture）是一种**"永不信任，始终验证"**的安全模型。它摒弃了传统"内网可信、外网不可信"的假设，要求对每一次访问请求都进行身份验证、授权和加密，无论请求来自内部还是外部网络。

## 核心原理

### 1. 零信任三原则

| 原则 | 说明 |
|------|------|
| **永不信任** | 不因网络位置、IP 地址、设备类型而自动信任 |
| **始终验证** | 每次请求都验证身份、设备状态、上下文风险 |
| **最小权限** | 仅授予完成当前操作所需的最小权限 |

### 2. 传统安全 vs 零信任

```
传统模型（城堡+护城河）：
  外网 → 防火墙 → 内网（可信）
                   │
              所有内部服务互信

零信任模型：
  任何请求 → 身份验证 → 设备验证 → 上下文评估 → 授权决策 → 加密通信
      │                    │              │              │
  不分内外网         设备合规检查     风险评分       最小权限
```

### 3. 核心组件

| 组件 | 功能 |
|------|------|
| **身份提供者 (IdP)** | 统一身份管理（OAuth2/OIDC/SAML） |
| **策略引擎 (PE)** | 基于上下文的风险评估和授权决策 |
| **策略执行点 (PEP)** | 在请求路径上执行策略（API Gateway/Service Mesh） |
| **设备信任** | 设备合规检查（证书、OS 版本、安全补丁） |
| **微分段** | 网络层面隔离服务，限制横向移动 |

### 4. 实施层次

```
Layer 1: 身份层
  ├── 统一身份（IdP）
  ├── MFA 多因素认证
  └── Service Identity（mTLS 证书）

Layer 2: 设备层
  ├── 设备证书
  ├── 合规检查（OS、补丁、加密）
  └── 设备信任评分

Layer 3: 网络层
  ├── 微分段（NetworkPolicy / Service Mesh）
  ├── 加密通信（mTLS / WireGuard）
  └── 不依赖网络位置

Layer 4: 应用层
  ├── API 级别授权（RBAC/ABAC）
  ├── 请求上下文评估
  └── 审计日志

Layer 5: 数据层
  ├── 数据分类分级
  ├── 加密存储
  └── 访问审计
```

## 实战案例

来自博客文章：[Zero Trust 架构实战：从 VPN 到零信任——Laravel 微服务中的身份验证与网络分段](/2026/06/01/Zero-Trust-架构实战-从VPN到零信任-Laravel微服务中的身份验证与网络分段/)

### Laravel 微服务落地

1. **mTLS 服务间通信**：每个 Laravel 服务持有证书，通过 HTTPS 双向认证
2. **API Gateway 统一策略**：Kong/APISIX 集成 OAuth2/OIDC，统一认证
3. **Service Mesh 微分段**：Istio/Linkerd 实现网络层面的服务隔离
4. **RBAC + ABAC**：Laravel 中间件实现基于角色+属性的授权
5. **审计日志**：每次访问记录请求者、资源、动作、时间、IP、设备

### 从 VPN 到零信任的迁移路径

```
Phase 1: 身份统一
  └── SSO + MFA，所有服务通过 IdP 认证

Phase 2: 服务身份
  └── mTLS 证书，Service Mesh 自动管理

Phase 3: 网络微分段
  └── K8s NetworkPolicy，Service Mesh AuthorizationPolicy

Phase 4: 持续验证
  └── 设备合规检查 + 上下文风险评估

Phase 5: 全面零信任
  └── 移除 VPN，所有访问通过零信任网关
```

## 常见问题

### Q: 零信任需要完全放弃 VPN 吗？
A: 不一定。可以渐进式迁移：先用零信任网关替代部分 VPN 访问，逐步减少 VPN 依赖。

### Q: 零信任会增加多少延迟？
A: mTLS 握手增加约 1-5ms，策略评估约 1-2ms。相比安全收益，延迟代价可接受。

### Q: 小团队需要零信任吗？
A: 小团队可以从"身份统一 + mTLS + 最小权限"开始，不需要完整的零信任框架。

### Q: 零信任与 API Gateway 的关系？
A: API Gateway 是零信任的策略执行点（PEP），但零信任还包括网络层微分段、设备信任等更广泛的范畴。
