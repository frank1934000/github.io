# API 网关（API Gateway）

## 定义

API 网关是微服务架构的**统一入口**，负责路由分发、鉴权、限流、熔断、日志、灰度发布等横切关注点。所有外部请求先经过网关，再转发到后端服务。

## 核心原理

### 为什么需要 API Gateway

```
❌ 没有 Gateway 的微服务调用拓扑
  BFF ──→ 订单（各自处理鉴权/限流/CORS/日志）
     ──→ 商品
     ──→ 用户
     ──→ 支付
     ──→ 库存
     ──→ 搜索
  （重复代码 × 6）

✅ 有 Gateway 的微服务调用拓扑
  Client → API Gateway（统一鉴权/限流/CORS/日志）
              ├── 订单服务（只关注业务）
              ├── 商品服务
              ├── 用户服务
              └── ...
```

### 核心能力

| 能力 | 说明 | 实现方式 |
|------|------|---------|
| **路由分发** | 根据路径/Header 转发到不同服务 | 路由规则配置 |
| **统一鉴权** | JWT 验证、OAuth2、API Key | 插件/中间件 |
| **限流熔断** | 保护后端服务不被打爆 | 令牌桶/滑动窗口 |
| **灰度发布** | 按比例/条件将流量导到新版本 | 权重路由/标签路由 |
| **日志监控** | 请求日志、响应时间、错误率 | 插件集成 |
| **协议转换** | HTTP ↔ gRPC、REST ↔ GraphQL | 插件 |
| **CORS 处理** | 跨域资源共享 | 中间件 |

### 主流网关对比

| 网关 | 语言 | 特点 | 适用场景 |
|------|------|------|---------|
| **Kong** | Lua (Nginx) | 插件生态丰富、企业级 | 大型微服务 |
| **Apache APISIX** | Lua (Nginx) | 高性能、etcd 配置中心 | 高并发场景 |
| **Nginx + Lua** | Lua | 轻量、灵活 | 小规模场景 |
| **Spring Cloud Gateway** | Java | Spring 生态集成 | Java 微服务 |

### 网关 vs Nginx 反向代理

| 维度 | Nginx 反向代理 | API Gateway |
|------|---------------|-------------|
| 路由 | 静态配置 | 动态路由（服务发现） |
| 鉴权 | 需自行实现 | 内置插件 |
| 限流 | limit_req 模块 | 精细化限流（按用户/IP/API） |
| 灰度 | 需自行实现 | 内置支持 |
| 可观测性 | 需额外配置 | 内置 Metrics/Tracing |

## 常见问题

**Q: API Gateway 会成为性能瓶颈吗？**
A: 会。Kong/APISIX 基于 Nginx，单实例可处理 10 万+ QPS。瓶颈通常在插件（如 JWT 验证），需要合理配置。

**Q: Gateway 和 BFF 可以合并吗？**
A: 可以但不推荐。Gateway 关注横切关注点（鉴权/限流），BFF 关注业务聚合。合并会导致职责不清。

**Q: 怎么做灰度发布？**
A: 通过 Header 标签（如 `X-Gray: true`）或权重比例（如 10% 流量到新版本）实现。
