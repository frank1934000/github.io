# 窗口函数

## 定义

窗口函数（Window Function）是 MySQL 8.0 引入的 SQL 功能，能在不减少结果集行数的情况下，对"窗口"内的数据进行聚合、排名和偏移计算。与 GROUP BY 不同，窗口函数保留每一行的细节。

## 核心语法

```sql
函数名(...) OVER (
  [PARTITION BY 分区列]
  [ORDER BY 排序列]
  [ROWS/RANGE 窗口范围]
)
```

## 三大排名函数

| 函数 | 并列处理 | 示例（分数 100, 100, 90） |
|---|---|---|
| `ROW_NUMBER()` | 不并列，顺序递增 | 1, 2, 3 |
| `RANK()` | 并列同名次，跳过后续 | 1, 1, 3 |
| `DENSE_RANK()` | 并列同名次，不跳过 | 1, 1, 2 |

### 分组 Top N 实战

```sql
-- 每个国家近 30 天销量 Top 3 商品
SELECT *
FROM (
    SELECT
        country_code, product_id, product_name, paid_amount,
        ROW_NUMBER() OVER (
            PARTITION BY country_code
            ORDER BY paid_amount DESC
        ) AS rn,
        RANK() OVER (
            PARTITION BY country_code
            ORDER BY paid_amount DESC
        ) AS rnk,
        DENSE_RANK() OVER (
            PARTITION BY country_code
            ORDER BY paid_amount DESC
        ) AS dense_rnk
    FROM product_daily_sales
    WHERE stat_date >= CURDATE() - INTERVAL 30 DAY
) ranked
WHERE rn <= 3;
```

## 常用窗口函数

| 类别 | 函数 | 用途 |
|---|---|---|
| 排名 | ROW_NUMBER, RANK, DENSE_RANK | 分组排名 |
| 聚合 | SUM, AVG, COUNT, MAX, MIN | 累计/滑动聚合 |
| 偏移 | LAG, LEAD | 前一行/后一行值 |
| 首尾 | FIRST_VALUE, LAST_VALUE | 窗口内首尾值 |
| 分布 | NTILE | 等分桶 |

### 环比计算

```sql
-- 计算每日销售额的日环比
SELECT
    stat_date,
    paid_amount,
    LAG(paid_amount, 1) OVER (ORDER BY stat_date) AS prev_day,
    paid_amount - LAG(paid_amount, 1) OVER (ORDER BY stat_date) AS day_over_day
FROM daily_sales;
```

## 实战踩坑

### 1. 临时表放大

窗口函数在 GROUP BY 之后执行，如果先聚合再窗口计算，中间结果集可能膨胀。建议先用 CTE 或子查询收窄数据。

### 2. 索引配合

```sql
-- 窗口函数的 ORDER BY 需要索引配合
KEY idx_country_date_amount (country_code, stat_date, paid_amount, product_id)
```

### 3. 分页口径

窗口函数生成的排名是在分页之前计算的，如果在窗口函数后加 LIMIT/OFFSET，排名可能不连续。

## 常见问题

**Q: ROW_NUMBER 和 RANK 什么时候用哪个？**
A: 需要唯一排名（如分页取 Top N）用 ROW_NUMBER；需要并列排名（如成绩单排名）用 RANK 或 DENSE_RANK。

**Q: 窗口函数能替代 GROUP BY 吗？**
A: 不能完全替代。GROUP BY 减少行数（聚合），窗口函数保留所有行。需要聚合结果时用 GROUP BY；需要保留明细并附加聚合信息时用窗口函数。

**Q: MySQL 5.7 怎么实现窗口函数的效果？**
A: 用变量模拟（`@rownum := @rownum + 1`），但写法复杂、不可靠且无法优化。建议升级到 8.0+。
