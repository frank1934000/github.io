# 覆盖索引（Covering Index）

## 定义

> **覆盖索引 = 一个 SELECT 需要的所有字段，都能从二级索引里直接拿到，不需要回表。**

EXPLAIN 显示 `Using index` 就是覆盖了。

## 核心原理

InnoDB 的二级索引存的是**主键值**，不是数据行：

```sql
SELECT name FROM users WHERE age = 25;

-- 走 idx_age 二级索引
-- → 拿到主键 id 列表
-- → 用 id 回到聚簇索引取整行  ← 这一步叫"回表"（随机IO）
-- → 取出 name
```

如果二级索引里直接就有 `name`，就省了回表这一步。

## 用联合索引实现覆盖

```sql
-- ❌ 普通索引，需要回表
ALTER TABLE users ADD INDEX idx_age (age);
SELECT name FROM users WHERE age = 25;

-- ✅ 覆盖索引，name 直接在索引里
ALTER TABLE users ADD INDEX idx_age_name (age, name);
SELECT name FROM users WHERE age = 25;
-- EXPLAIN: Using index
```

## EXPLAIN 判断标准

| Extra 字段 | 含义 |
|-----------|------|
| **Using index** | ✅ 覆盖索引，没回表 |
| **Using where; Using index** | 用了索引过滤 + 覆盖 |
| **Using index condition** | 索引下推（ICP），过滤了一部分但仍回表 |
| 空 / `Using where` | ❌ 回表了 |

## 典型应用场景

### 1. 列表页只查少数字段

```sql
-- 商品列表只展示 id, title, price
ALTER TABLE products ADD INDEX idx_cover (category_id, price, title);

SELECT id, title, price FROM products
WHERE category_id = 1 ORDER BY price LIMIT 20;
```

### 2. COUNT 优化

```sql
-- COUNT(*) 在 InnoDB 要扫聚簇索引（数据行）
-- 但如果索引列不含 NULL，COUNT(索引列) 可以走更小的索引树
```

### 3. 排序优化

```sql
-- ORDER BY 字段也在索引中，避免 filesort
SELECT id, title, price FROM products
WHERE category_id = 1 ORDER BY price;
```
