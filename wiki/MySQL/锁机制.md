# MySQL 锁机制

## 定义

锁机制是计算机协调**多个进程或线程**并发访问某一资源的机制。

## 锁的类型与特点

| 锁类型 | 开销 | 加锁速度 | 粒度 | 锁冲突 | 并发度 |
|--------|------|---------|------|--------|--------|
| **表级锁** | 大 | 快 | 大 | 概率高 | 小 |
| **页面锁** | 中 | 中 | 中 | 中 | 一般 |
| **行级锁** | 大 | 慢 | 小 | 概率小 | 高 |

### 引擎支持

| 引擎 | 支持的锁 |
|------|---------|
| MyISAM | 只支持**表锁** |
| InnoDB | 默认支持**行锁**，也支持表锁 |

## 锁的维度划分

### 共享锁（S Lock / 读锁）
- 多个事务可以同时持有同一行的共享锁
- 用于 `SELECT ... LOCK IN SHARE MODE`

### 排他锁（X Lock / 写锁）
- 一个事务持有排他锁时，其他事务不能读也不能写
- 用于 `SELECT ... FOR UPDATE` / INSERT / UPDATE / DELETE

### 意向锁（Intention Lock）
- **意向共享锁（IS）**：事务打算给行加共享锁前，先在表上加 IS
- **意向排他锁（IX）**：事务打算给行加排他锁前，先在表上加 IX
- 作用：快速判断表中是否有行被锁定，避免逐行检查

### 间隙锁（Gap Lock）
- 锁定一个**范围**，但不包括记录本身
- 用于**可重复读**级别下防止幻读
- `SELECT * FROM users WHERE id > 10 FOR UPDATE` 会锁住 id > 10 的所有间隙

## 死锁

### 产生原因
1. 系统资源不足
2. 进程运行推进的顺序/速度不同
3. 资源分配不当

### 必要条件
1. **互斥**：一个资源每次只能被一个进程使用
2. **请求与保持**：进程因请求资源而阻塞时，对已获得的资源保持不放
3. **不可剥夺**：进程已获得的资源，在未使用完之前，不能强行剥夺
4. **循环等待**：若干进程之间形成头尾相接的循环等待关系

### 避免死锁

1. **合理设计索引**：区分度高的列放到组合索引前面，通过索引定位更少的行，减少锁竞争
2. **固定访问顺序**：多个表操作时，按固定顺序访问
3. **减小事务粒度**：大事务拆成小事务
4. **使用低隔离级别**：读已提交比可重复读锁范围小
5. **设置锁等待超时**：`innodb_lock_wait_timeout`

```sql
-- 查看锁等待情况
SHOW ENGINE INNODB STATUS;

-- 查看当前锁
SELECT * FROM information_schema.INNODB_LOCKS;
SELECT * FROM information_schema.INNODB_LOCK_WAITS;
```
