# MySQL 存储引擎对比

## 定义

MySQL 支持多种存储引擎，其中最常用的是 **InnoDB** 和 **MyISAM**。MySQL 5.5.5 开始，InnoDB 成为默认存储引擎。

## InnoDB vs MyISAM 全方位对比

| 特性 | MyISAM | InnoDB |
|------|--------|--------|
| **事务** | ❌ 不支持 | ✅ 支持 |
| **外键** | ❌ 不支持 | ✅ 支持 |
| **聚簇索引** | ❌ 不支持 | ✅ 支持 |
| **索引和数据** | 分开存储 | 存储在一起 |
| **MVCC** | ❌ 不支持 | ✅ 支持 |
| **锁粒度** | 只支持**表锁** | 支持**行锁、表锁、页锁** |
| **并发能力** | 低 | 高（通过 MVCC 支持） |
| **备份** | 不支持在线热备 | ✅ 支持在线热备 |
| **全文索引** | ✅ 支持 | ❌ 不支持（5.6+ 已支持） |
| **主键范围** | 小 | 大 |
| **崩溃恢复** | 慢，易丢失数据 | 概率低，恢复快 |
| **count(*)** | 保存行数，直接返回 | 全表扫描（不加 WHERE 时） |
| **读写场景** | 读多写少 | 更新和插入较多 |

## 数据文件对比

### MyISAM
| 文件 | 扩展名 | 说明 |
|------|--------|------|
| 表定义 | `.frm` | 文件存储表定义 |
| 数据文件 | `.MYD` | MYData 数据文件 |
| 索引文件 | `.MYI` | MYIndex 索引文件 |

### InnoDB
| 文件 | 扩展名 | 说明 |
|------|--------|------|
| 表定义 | `.frm` | 文件存储表定义 |
| 数据+索引 | `.ibd` | 数据和索引存储在一起 |
| 大小限制 | - | 只受限于操作系统文件大小（一般 2GB） |

## 索引存储方式对比

### MyISAM 索引
- B+树叶子节点存储的是**行数据的地址**
- 查找时：找到叶子节点地址 → 根据地址查找数据（两次查找）

### InnoDB 索引
- **主键索引（聚簇索引）**：叶子节点直接存储行数据
- **非主键索引（二级索引）**：叶子节点存储索引值 + 主键值
- 非主键查询需要**回表**

## 选型建议

| 场景 | 推荐引擎 |
|------|---------|
| 事务系统（OLTP） | ✅ InnoDB |
| 高并发写入 | ✅ InnoDB |
| 读多写少 | MyISAM / InnoDB |
| 全文搜索 | MyISAM（或 ES 替代） |
| 数据安全要求高 | ✅ InnoDB（crash-safe） |
