# 聚簇索引与非聚簇索引

## 定义

在 InnoDB 中，索引 B+Tree 的叶子节点存储了**整行数据**的是主键索引，称为**聚簇索引**（Clustered Index），即将数据存储与索引放到了一块，找到索引也就找到了数据。

而索引 B+Tree 的叶子节点存储了**主键的值**的是非主键索引，称为**非聚簇索引**（Non-clustered Index），也叫**二级索引**（Secondary Index）。

## 核心区别

| 特性 | 聚簇索引 | 非聚簇索引（二级索引） |
|------|---------|---------------------|
| 叶子节点存储 | **整行数据** | **主键值** |
| 一张表数量 | 只能有**一个** | 可以有**多个** |
| 查询效率 | 主键查询只需**一次** | 可能需要**回表** |
| IO 类型 | 顺序 IO | 首次顺序 IO，回表为**随机 IO** |

## 回表机制

```
SELECT name FROM users WHERE age = 25;

-- 1. 走 idx_age 二级索引
-- 2. 拿到主键 id 列表
-- 3. 用 id 回到聚簇索引取整行  ← 这一步叫"回表"
-- 4. 取出 name
```

**回表 = 一次随机 IO**。需要回表的次数越多，随机 IO 次数越多，越倾向于使用全表扫描。

## 非聚簇索引一定回表吗？

**不一定！** 如果查询所需字段全部命中了索引（覆盖索引），则不需要回表。

```sql
-- 假设在 score 上建立了索引
SELECT score FROM student WHERE score > 90;
-- 索引叶子节点已包含 score 信息，不会回表
```

## 联合索引与顺序

MySQL 可以使用多个字段建立联合索引。联合索引中需要按照建立索引时的字段顺序使用，否则无法命中。

**原因**：联合索引 `(name, age, school)` 的排序为：先按 name 排序，name 相同按 age 排序，age 也相等按 school 排序。查询时必须先用 name 进行等值查询，之后才能用 age 做索引查找。

## 引擎对比

| 引擎 | 主键索引 | 二级索引 |
|------|---------|---------|
| **InnoDB** | 聚簇索引 | 非聚簇索引 |
| **MyISAM** | 非聚簇索引 | 非聚簇索引 |
