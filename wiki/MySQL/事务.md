# MySQL 事务（ACID 与隔离级别）

## 定义

事务是数据库操作的最小执行单元，要么全部成功，要么全部失败。

## ACID 四大特性

| 特性 | 英文 | 说明 |
|------|------|------|
| **原子性** | Atomicity | 一个事务中的操作要么全部成功，要么全部失败 |
| **一致性** | Consistency | 总是从一个一致性的状态转换到另一个一致性的状态 |
| **隔离性** | Isolation | 一个事务的修改在提交前，其他事务是感知不到的 |
| **持久性** | Durable | 永久保存在数据库中 |

> **原子性、隔离性、持久性都是为了保障一致性而存在的，一致性也是最终的目的。**

## 四种隔离级别

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 说明 |
|---------|------|-----------|------|------|
| **读未提交** READ UNCOMMITTED | ✅ | ✅ | ✅ | 读到别的事务还没提交的数据 |
| **读已提交** READ COMMITTED | ❌ | ✅ | ✅ | 读到别的事务已提交的数据 |
| **可重复读** REPEATABLE READ | ❌ | ❌ | ✅ | 事务内看到的数据和事务开始时一致（MySQL 默认） |
| **可序列化** SERIALIZABLE | ❌ | ❌ | ❌ | 读写锁冲突时，后面事务等待前面完成 |

> 级别越高，隔离越严实。大部分公司选择 MySQL 默认的**可重复读**。

## 隔离级别实战分析

假如有 user 表 `(id=1, age=24)`，执行 `age = age + 1`：

| 事务1 | 事务2 |
|-------|-------|
| 启动事务，查询 age (a1) | |
| | 启动事务，查询 age (a2) |
| | 执行 age = age + 1 |
| 查询 age (a3) | |
| | 提交事务 |
| 查询 age (a4) | |
| 提交事务 | |
| 查询 age (a5) | |

**各隔离级别下 a1-a5 的值**：

| 隔离级别 | a1 | a2 | a3 | a4 | a5 |
|---------|----|----|----|----|-----|
| 读未提交 | 24 | 24 | **25** | **25** | **25** |
| 读已提交 | 24 | 24 | 24 | **25** | **25** |
| 可重复读 | 24 | 24 | 24 | 24 | **25** |
| 串行化 | 24 | 24 | 24 | 24 | **25** |

## 可重复读如何实现？

MySQL 的可重复读通过 **MVCC（多版本控制）** 实现：

- 每行数据有多个版本
- 事务开始时获取一个一致性视图
- 通过 **undo log** 计算出历史版本
- 间隙锁（Gap Lock）解决幻读问题
