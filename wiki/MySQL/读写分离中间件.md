# 读写分离中间件实战：ProxySQL / MaxScale

## 定义

读写分离中间件位于应用与数据库之间，自动将写请求路由到主库、读请求路由到从库，对应用层透明。相比应用层手动分离（如 Laravel 的 `DB::connection('read')`），中间件方案更灵活、更可靠。

## 核心架构

```
应用层（Laravel）
    ↓
ProxySQL / MaxScale（中间件层）
    ├→ Master（写）
    ├→ Slave 1（读）
    ├→ Slave 2（读）
    └→ Slave 3（读）
```

## ProxySQL

### 核心特性

- **MySQL 协议代理**：应用无需修改代码，连接 ProxySQL 即可
- **查询路由**：基于规则将 SELECT 路由到从库，INSERT/UPDATE/DELETE 路由到主库
- **连接池复用**：减少后端 MySQL 连接数，解决连接风暴
- **查询缓存**：内置查询缓存，减少后端压力
- **故障检测与自动切换**：后端实例故障时自动摘除，恢复后自动加回
- **读写分离规则**：基于正则匹配、用户、schema 等维度

### 配置示例

```sql
-- 添加后端 MySQL 服务器
INSERT INTO mysql_servers (hostgroup_id, hostname, port) VALUES
(10, '10.0.0.1', 3306),   -- 写组（hostgroup 10）
(20, '10.0.0.2', 3306),   -- 读组（hostgroup 20）
(20, '10.0.0.3', 3306);   -- 读组（hostgroup 20）

-- 配置读写分离规则
INSERT INTO mysql_query_rules (rule_id, match_pattern, destination_hostgroup) VALUES
(1, '^SELECT .* FOR UPDATE$', 10),   -- FOR UPDATE → 写组
(2, '^SELECT', 20);                   -- 其他 SELECT → 读组

-- 加载配置
LOAD MYSQL SERVERS TO RUNTIME;
LOAD MYSQL QUERY RULES TO RUNTIME;
SAVE MYSQL SERVERS TO DISK;
SAVE MYSQL QUERY RULES TO DISK;
```

### 连接池配置

```sql
-- 最大连接数
SET mysql-max_connections = 2048;

-- 连接复用模式
SET mysql-connection_max_age_ms = 60000;  -- 连接最大存活 60s
SET mysql-free_connections_pct = 10;       -- 保留 10% 空闲连接
```

## MaxScale

### 核心特性

- **MariaDB 官方出品**，与 MariaDB/MySQL 深度集成
- **多种路由模块**：readwritesplit（读写分离）、readconnroute（只读路由）
- **自动故障转移**：集成 MariaDB Monitor，支持自动主从切换
- **查询过滤**：支持正则过滤、QLA（查询日志分析）
- **GUI 管理界面**：MaxCtrl / REST API

### 配置示例

```ini
[Read-Write-Service]
type=service
router=readwritesplit
servers=master,slave1,slave2
user=maxscale_user
password=encrypted_password
max_slave_connections=100%
slave_selection_criteria=LEAST_CURRENT_OPERATIONS

[Read-Write-Listener]
type=listener
service=Read-Write-Service
protocol=MariaDBClient
port=3306

[Monitor]
type=monitor
module=mysqlmon
servers=master,slave1,slave2
monitor_interval=2000ms
```

| 维度 | ProxySQL | MaxScale |

| 开发商 | 社区（Sysown） | MariaDB 官方 |

| 查询缓存 | ✅ 内置 | ❌ 无 |

## 主从延迟治理

读写分离的最大挑战是**主从延迟**——从库读到的数据可能不是最新的。

### 延迟检测

```sql
-- 在从库执行
SHOW SLAVE STATUS\G
-- 关注 Seconds_Behind_Master
```

### 延迟治理策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **关键读走主库** | 写后读（如订单创建后立即查询）强制路由到主库 | 一致性要求高的场景 |
| **延迟阈值** | 延迟超过阈值时，读请求回退到主库 | ProxySQL 支持 |
| **半同步复制** | 主库等待至少一个从库确认收到 binlog | 金融级场景 |
| **GTID 复制** | 基于全局事务 ID 确保数据一致性 | 高可靠性场景 |
| **并行复制** | 从库多线程回放 relay log，减少延迟 | MySQL 5.7+ |

### Laravel 中的关键读走主库

```php
// 写后读：强制走主库
DB::transaction(function () use ($data) {
    $order = Order::create($data);
    // 事务内默认走主库
});

// 事务外强制走主库
$order = DB::connection('mysql')
    ->table('orders')
    ->where('id', $orderId)
    ->first();
```

## 常见问题

**Q: ProxySQL 和 Laravel 数据库连接池冲突吗？**
A: 不冲突。ProxySQL 管理应用到数据库的连接，Laravel 的连接池（如 Swoole）管理 PHP 进程到 ProxySQL 的连接，两者配合使用。

**Q: 主从延迟怎么监控？**
A: ProxySQL 内置 `stats_mysql_query_digest` 表；MaxScale 有 `maxctrl list servers` 命令。也可以通过 Prometheus + Grafana 监控。

**Q: 什么时候不需要读写分离中间件？**
A: 小规模应用（单主单从）且读写比例不高时，Laravel 应用层配置 `config/database.php` 的 read/write host 即可。
