# 多租户模式

## 定义

多租户模式是 SaaS 产品中**不同租户数据隔离**的架构模式。核心目标是在共享基础设施的前提下，实现租户间数据的安全隔离，同时平衡成本、性能和运维复杂度。

## 三种方案对比

| 维度 | 共享库 + Row-Level (RLS) | Schema-per-Tenant | 独立库 (Database-per-Tenant) |
|------|------------------------|-------------------|----------------------------|
| **安全性** | ⭐⭐ 最弱 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐⭐ 最强 |
| **性能** | ⭐⭐⭐⭐ 好 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐⭐ 最好 |
| **运维成本** | ⭐⭐⭐⭐⭐ 最低 | ⭐⭐⭐ 中等 | ⭐⭐ 最高 |
| **扩展性** | ⭐⭐ 受限于单库 | ⭐⭐⭐⭐ 较好 | ⭐⭐⭐⭐⭐ 最好 |
| **适用规模** | 初创 / < 100 租户 | 中型 / 100~10000 租户 | 企业级 / 合规要求高 |
| **数据备份** | 整库备份 | 可按 Schema 备份 | 可按库备份/恢复 |
| **迁移难度** | 简单 | 中等 | 复杂 |

### 方案一：共享库 + Row-Level Security (RLS)

每张表添加 `tenant_id` 字段，通过 Global Scope 自动过滤：

```php
// Trait: BelongsToTenant
trait BelongsToTenant
{
    protected static function bootBelongsToTenant(): void
    {
        static::addGlobalScope('tenant', function (Builder $builder) {
            $builder->where('tenant_id', tenant('id'));
        });

        static::creating(function (Model $model) {
            $model->tenant_id = tenant('id');
        });
    }
}

// 使用
class Order extends Model
{
    use BelongsToTenant;
}

// 所有查询自动带 tenant_id 过滤
Order::all(); // WHERE tenant_id = 1
```

**优点**：成本最低，一套数据库服务所有租户
**风险**：Bug 或 SQL 注入可能导致跨租户数据泄露

### 方案二：Schema-per-Tenant

每个租户一个独立的 Schema（PostgreSQL）或 Database（MySQL）：

```php
// 中间件切换 Schema
class TenantMiddleware
{
    public function handle(Request $request, Closure $next)
    {
        $tenant = auth()->user()->tenant;

        // PostgreSQL: 切换 search_path
        DB::statement("SET search_path TO tenant_{$tenant->id}, public");

        // MySQL: 切换数据库
        DB::statement("USE tenant_{$tenant->id}");

        return $next($request);
    }
}
```

**优点**：中等隔离，可按租户备份/恢复
**挑战**：Schema 数量过多时管理复杂，连接池需特殊处理

### 方案三：独立库 (Database-per-Tenant)

每个租户一个独立的数据库实例或 Schema：

```php
// 动态切换连接
config(['database.connections.tenant' => [
    'driver' => 'mysql',
    'host' => $tenant->db_host,
    'database' => $tenant->db_name,
    'username' => $tenant->db_user,
    'password' => $tenant->db_pass,
]]);

// 使用租户连接
Order::on('tenant')->get();
```

**优点**：最强隔离，满足合规要求（GDPR、SOC2）
**挑战**：运维成本高，连接池管理复杂，跨租户查询困难

## Laravel 实现详解

### Global Scope 实现

```php
// app/Scopes/TenantScope.php
class TenantScope implements Scope
{
    public function apply(Builder $builder, Model $model)
    {
        if (tenant()) {
            $builder->where($model->getTable() . '.tenant_id', '=', tenant('id'));
        }
    }
}

// 临时绕过租户过滤（管理员操作）
Order::withoutGlobalScope(TenantScope::class)->get();
```

### 连接池管理

Schema-per-Tenant 模式下，连接池需要特殊配置：

```php
// 动态创建连接（惰性初始化）
app()->bind('tenant.connection', function () {
    return DB::connection('tenant_' . tenant('id'));
});

// ProxySQL 配置：按用户名路由到不同 Schema
// INSERT INTO mysql_users (username, default_schema) VALUES ('tenant_1', 'tenant_1');
```

         (最低成本)      /         \

                   独立库         Schema-per-Tenant

### 阶段建议

| 阶段 | 租户规模 | 推荐方案 | 关键考量 |
|------|----------|----------|----------|
| 初创期 | < 100 | 共享库 + RLS | 快速迭代，低成本 |
| 增长期 | 100~10000 | Schema-per-Tenant | 隔离性与成本平衡 |
| 企业级 | > 10000 或合规 | 独立库 | 数据安全、合规要求 |
