# 最左前缀原则

## 定义

最左前缀原则就是**最左优先**。在创建多列索引时，要根据业务需求，WHERE 子句中使用最频繁的一列放在最左边。

> MySQL 会一直向右匹配，直到遇到**范围查询**（>、<、BETWEEN、LIKE）就停止匹配。

## 核心规则

### 规则一：范围查询后停止匹配

```sql
-- 索引 (a, b, c, d)
WHERE a = 1 AND b = 2 AND c > 3 AND d = 4
-- a ✅ b ✅ c ✅ d ❌（c 是范围查询，d 用不到索引）

-- 如果建立 (a, b, d, c) 的索引
-- a ✅ b ✅ d ✅ c ✅
-- a, b, d 的顺序可以任意调整
```

### 规则二：= 和 IN 可以乱序

```sql
-- 索引 (a, b, c)
WHERE a = 1 AND b = 2 AND c = 3
-- a, b, c 任意顺序都可以命中
-- MySQL 查询优化器会自动优化
```

### 规则三：字符串索引的最左 M 个字符

```sql
-- 假设联合索引 (col3, col2)
WHERE col3 LIKE 'Eri%'
-- ✅ 可以快速定位到 Eric

WHERE col3 LIKE '%se'
-- ❌ 前缀不确定，全表扫描
```

## 实例分析

```sql
-- INDEX (a, b, c)
WHERE a = 1               -- ✅ 走索引
WHERE a = 1 AND b = 2     -- ✅ 走索引
WHERE a = 1 AND c = 3     -- ✅ a 走索引，c 用不上
WHERE b = 2 AND c = 3     -- ❌ 无法命中索引
WHERE c = 3               -- ❌ 无法命中索引
```

## 设计建议

1. **高频查询列放左边**：把查询需求频繁的列放在联合索引的最左边
2. **区分度高的列放左边**：区分度 = `COUNT(DISTINCT col) / COUNT(*)`，比例越大扫描记录越少
3. **等值查询在前，范围查询在后**：避免范围查询后的列无法使用索引
