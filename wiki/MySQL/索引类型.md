# 索引类型

## 定义

MySQL 索引可以从三个维度进行分类：存储结构、应用层次、物理顺序与逻辑顺序的关系。

## 分类维度

### 1. 从存储结构划分

| 索引类型 | 说明 |
|---------|------|
| BTree 索引 | B-Tree 或 B+Tree 索引，最常用 |
| Hash 索引 | 基于哈希表，仅支持等值查询 |
| Full-Text 全文索引 | 用于全文搜索 |
| R-Tree 索引 | 空间索引，用于地理数据 |

### 2. 从应用层次划分

| 索引类型 | 说明 |
|---------|------|
| **主键索引** | 唯一标识一行，不允许 NULL |
| **普通索引** | 一个索引只包含一个列，一个表可以有多个单列索引 |
| **唯一索引** | 索引列的值必须唯一，但允许有空值 |
| **复合索引（联合索引）** | 多列值组成一个索引，专门用于组合搜索，效率大于索引合并 |
| **空间索引** | 用于空间数据类型（GIS） |

### 3. 从物理与逻辑顺序关系划分

| 索引类型 | 说明 |
|---------|------|
| **聚簇索引（聚集索引）** | 也称为主键索引，是一种数据存储方式。B+Tree 结构，非叶子节点包含键值和指针，叶子节点包含索引列和行数据。一张表只能有一个聚簇索引 |
| **非聚簇索引（非聚集索引）** | 不是聚簇索引就是非聚簇索引。叶子节点只存索引列和主键 id。如果 SQL 还要返回除了索引列的其他字段信息，需要回表。第一次索引一般是顺序 IO，回表属于随机 IO，回表次数越多性能越差 |

## 核心要点

- InnoDB 的主键索引是聚簇索引，二级索引是非聚簇索引
- MyISAM 无论主键索引还是二级索引都是非聚簇索引
- 我们自己建的索引基本都是非聚簇索引
