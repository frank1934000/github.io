# 索引概念

## 定义

索引是一种特殊的文件（InnoDB 数据表上的索引是表空间的一个组成部分），它们包含着对数据表里所有记录的引用指针。索引是一种**数据结构**，是数据库管理系统中一个排序的数据结构，以协助快速查询、更新数据库表中数据。

索引的实现通常使用 **B树** 及其变种 **B+树**。

> 通俗地说，索引就相当于**目录**。为了方便查找书中的内容，通过对内容建立索引形成目录。

## 核心原理

- 索引是一个文件，要占据物理空间
- 索引可以大大提高 MySQL 的检索速度
- 类似字典的拼音和偏旁索引，找到对应页码后直接翻到目标页
- 但索引不是越多越好，过多索引会降低写入性能

## 索引的代价

| 代价 | 说明 |
|------|------|
| 存储空间 | 索引需要额外的磁盘空间 |
| 写入性能 | INSERT/DELETE/UPDATE 时需要同步维护索引，多付出 4-5 次磁盘 I/O |
| 维护成本 | 大量增删改后，索引统计信息可能过期 |

## 索引优化的范围查询

使用索引查询不一定能提高查询性能。索引范围查询（INDEX RANGE SCAN）适用于：

1. 基于一个范围的检索，查询返回结果集**小于表中记录数的 30%**
2. 基于非唯一性索引的检索
