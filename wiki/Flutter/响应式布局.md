# 响应式布局

## 定义

响应式布局（Responsive Layout）是指应用界面能够根据不同屏幕尺寸、方向和设备类型自动调整布局和显示方式的设计策略。在Flutter中，响应式布局通过`MediaQuery`、`LayoutBuilder`、`OrientationBuilder`等工具实现，支持从手机到平板、折叠屏、桌面等多种设备形态的适配。良好的响应式设计能确保应用在各种设备上都提供最佳的用户体验。

## 核心原理

### 1. MediaQuery — 获取屏幕信息

`MediaQuery`是Flutter中获取设备屏幕信息的核心工具：

```dart
// 获取屏幕尺寸
final size = MediaQuery.of(context).size;
final width = size.width;
final height = size.height;

// 获取设备像素比
final pixelRatio = MediaQuery.of(context).devicePixelRatio;

// 获取屏幕方向
final orientation = MediaQuery.of(context).orientation;

// 获取安全区域（刘海屏、圆角等）
final padding = MediaQuery.of(context).padding;
final topPadding = padding.top;
final bottomPadding = padding.bottom;

// 获取系统字体缩放
final textScale = MediaQuery.of(context).textScaleFactor;
```

### 2. LayoutBuilder — 基于父约束的布局

`LayoutBuilder`根据父组件提供的约束条件来构建子组件，是响应式布局的核心工具：

```dart
LayoutBuilder(
  builder: (context, constraints) {
    if (constraints.maxWidth > 1200) {
      return _buildDesktopLayout();
    } else if (constraints.maxWidth > 600) {
      return _buildTabletLayout();
    } else {
      return _buildMobileLayout();
    }
  },
)
```

### 3. 响应式断点（Breakpoints）

定义断点是响应式设计的基础，常用的断点划分：

| 断点名称 | 宽度范围 | 设备类型 | 典型布局 |
|----------|----------|----------|----------|
| xs | 0 - 599px | 手机竖屏 | 单列布局 |
| sm | 600 - 959px | 手机横屏/小平板 | 双列布局 |
| md | 960 - 1279px | 平板 | 三列布局 |
| lg | 1280 - 1919px | 桌面 | 多列布局 |
| xl | ≥ 1920px | 大桌面 | 宽屏布局 |

```dart
// 响应式断点工具类
class Breakpoints {
  static const double mobile = 600;
  static const double tablet = 960;
  static const double desktop = 1280;
  static const double largeDesktop = 1920;

  static bool isMobile(BuildContext context) =>
      MediaQuery.of(context).size.width < mobile;

  static bool isTablet(BuildContext context) {
    final width = MediaQuery.of(context).size.width;
    return width >= mobile && width < desktop;
  }

  static bool isDesktop(BuildContext context) =>
      MediaQuery.of(context).size.width >= desktop;
}
```

### 4. 折叠屏适配

折叠屏设备带来了新的布局挑战，Flutter通过`DisplayFeature` API支持折叠屏检测：

```dart
// 检测折叠屏铰链位置
final displayFeatures = MediaQuery.of(context).displayFeatures;
final hinge = displayFeatures.firstWhere(
  (f) => f.type == DisplayFeatureType.hinge,
  orElse: () => null,
);

if (hinge != null) {
  // 铰链模式：左右分屏
  return Row(
    children: [
      SizedBox(width: hinge.bounds.left, child: LeftPanel()),
      SizedBox(width: hinge.bounds.right - hinge.bounds.left), // 铰链区域
      Expanded(child: RightPanel()),
    ],
  );
}

// 检测折叠状态
final posture = MediaQuery.of(context).displayFeatures
    .any((f) => f.state == DisplayFeatureState.postureHalfOpened);
```

### 5. 平板适配策略

平板设备需要特别考虑的布局策略：

- **Master-Detail布局**：左侧列表 + 右侧详情，经典平板布局
- **自适应网格**：根据屏幕宽度调整网格列数
- **侧边导航**：平板使用侧边导航栏替代底部导航
- **多面板**：同时显示多个功能面板

```dart
// Master-Detail 布局
class MasterDetailLayout extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        if (constraints.maxWidth > Breakpoints.tablet) {
          // 平板/桌面：左右分栏
          return Row(
            children: [
              SizedBox(width: 350, child: ListPanel()),
              VerticalDivider(width: 1),
              Expanded(child: DetailPanel()),
            ],
          );
        } else {
          // 手机：只显示列表，点击后导航到详情
          return ListPanel();
        }
      },
    );
  }
}
```

### 6. 自适应组件模式

创建可复用的响应式组件：

```dart
// 自适应网格组件
class ResponsiveGrid extends StatelessWidget {
  final List<Widget> children;

  const ResponsiveGrid({required this.children});

  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        final columns = _getColumnCount(constraints.maxWidth);
        return GridView.builder(
          gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
            crossAxisCount: columns,
            crossAxisSpacing: 16,
            mainAxisSpacing: 16,
            childAspectRatio: 1.0,
          ),
          itemCount: children.length,
          itemBuilder: (_, index) => children[index],
        );
      },
    );
  }

  int _getColumnCount(double width) {
    if (width > 1200) return 4;
    if (width > 900) return 3;
    if (width > 600) return 2;
    return 1;
  }
}
```

### 7. 屏幕方向适配

处理横竖屏切换：

```dart
// 方向感知布局
OrientationBuilder(
  builder: (context, orientation) {
    if (orientation == Orientation.landscape) {
      return Row(
        children: [
          Expanded(child: ContentPanel()),
          SizedBox(width: 300, child: SidePanel()),
        ],
      );
    } else {
      return Column(
        children: [
          Expanded(child: ContentPanel()),
          SizedBox(height: 200, child: BottomPanel()),
        ],
      );
    }
  },
)
```

## 常见问题

### Q1: MediaQuery和LayoutBuilder如何选择？
`MediaQuery`获取整个屏幕信息，适合全局判断；`LayoutBuilder`获取父组件约束，适合局部布局决策。优先使用`LayoutBuilder`，因为它更精确且不会导致不必要的重建。

### Q2: 如何处理屏幕旋转导致的布局问题？
使用`OrientationBuilder`或在`MediaQuery`中监听方向变化，为不同方向提供独立布局。同时注意在`build`方法中避免硬编码尺寸。

### Q3: 折叠屏适配有哪些注意事项？
- 检测铰链位置和折叠状态
- 避免在铰链区域放置重要内容
- 处理展开/折叠时的状态保持
- 考虑多窗口模式（分屏）

### Q4: 如何实现自适应字体大小？
使用`Theme.of(context).textTheme`获取主题字体样式，配合`MediaQuery.textScaleFactor`处理系统字体缩放。避免使用固定`fontSize`。

### Q5: 平板横竖屏切换时如何保持状态？
使用`PageStorage`或状态管理方案（如Riverpod/Provider）保存滚动位置和选中状态，避免方向切换时丢失用户操作。

### Q6: 如何测试响应式布局？
- 使用Flutter DevTools的设备模拟功能
- 在真实设备上测试（不同尺寸的手机、平板、折叠屏）
- 使用`SizedBox`包裹组件模拟不同约束条件
- 编写Widget测试验证不同宽度下的布局行为
