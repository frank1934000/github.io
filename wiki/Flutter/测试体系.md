# Flutter 测试体系

## 定义

Flutter 测试体系是一套覆盖单元测试（Unit Test）、组件测试（Widget Test）、集成测试（Integration Test）
的三层自动化测试方案。Flutter 框架原生内置测试支持，无需额外测试框架即可编写和运行所有层级的测试。

测试金字塔模型：
- **单元测试**（底层，数量最多）：测试单个函数、方法、类的逻辑
- **组件测试**（中层）：测试 Widget 的 UI 行为和交互
- **集成测试**（顶层，数量最少）：测试完整应用或跨模块的端到端流程

### 测试工具链

| 工具 | 用途 | 包名 |
|------|------|------|
| `flutter_test` | 单元测试 + Widget 测试框架 | Flutter 内置 |
| `integration_test` | 集成测试框架 | Flutter 内置 |
| `mockito` | Mock 对象生成 | `mockito` |
| `mocktail` | 零代码生成的 Mock | `mocktail` |
| `build_runner` | 代码生成（配合 Mockito） | `build_runner` |
| `golden_toolkit` | Golden 测试增强 | `golden_toolkit` |
| `bloc_test` | Bloc 状态测试 | `bloc_test` |
| `fake_cloud_firestore` | Firestore 测试替身 | `fake_cloud_firestore` |
| `patrol` | 高级集成测试（原生交互） | `patrol` |

## 核心原理

### 1. 单元测试 (Unit Tests)

测试纯 Dart 逻辑，不涉及 Flutter 框架或 UI：

```dart
// test/services/calculator_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:my_app/services/calculator.dart';

void main() {
  group('Calculator', () {
    late Calculator calculator;

    setUp(() {
      calculator = Calculator();
    });

    group('add', () {
      test('should return sum of two positive numbers', () {
        expect(calculator.add(2, 3), equals(5));
      });

      test('should handle negative numbers', () {
        expect(calculator.add(-1, 1), equals(0));
      });

      test('should handle zero', () {
        expect(calculator.add(0, 0), equals(0));
      });
    });

    group('divide', () {
      test('should return correct result', () {
        expect(calculator.divide(10, 2), equals(5.0));
      });

      test('should throw when dividing by zero', () {
        expect(() => calculator.divide(10, 0), throwsArgumentError);
      });
    });
  });
}
```

**异步测试**：
```dart
test('should fetch user data', () async {
  final service = UserService(mockClient);
  when(() => mockClient.get(any())).thenAnswer(
    (_) async => Response('{"name": "test"}', 200),
  );

  final user = await service.fetchUser('123');
  expect(user.name, equals('test'));
});
```

### 2. Widget 测试 (Widget Tests)

在虚拟环境中渲染 Widget，测试 UI 行为：

```dart
// test/widgets/login_form_test.dart
import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:mocktail/mocktail.dart';
import 'package:my_app/widgets/login_form.dart';
import 'package:my_app/services/auth_service.dart';

class MockAuthService extends Mock implements AuthService {}

void main() {
  late MockAuthService mockAuthService;

  setUp(() {
    mockAuthService = MockAuthService();
  });

  Widget buildLoginForm() {
    return MaterialApp(
      home: Scaffold(
        body: LoginForm(authService: mockAuthService),
      ),
    );
  }

  group('LoginForm', () {
    testWidgets('should show email and password fields', (tester) async {
      await tester.pumpWidget(buildLoginForm());

      expect(find.byType(TextField), findsNWidgets(2));
      expect(find.text('邮箱'), findsOneWidget);
      expect(find.text('密码'), findsOneWidget);
      expect(find.byType(ElevatedButton), findsOneWidget);
    });

    testWidgets('should show error when fields are empty', (tester) async {
      await tester.pumpWidget(buildLoginForm());

      await tester.tap(find.byType(ElevatedButton));
      await tester.pumpAndSettle();

      expect(find.text('请输入邮箱'), findsOneWidget);
      expect(find.text('请输入密码'), findsOneWidget);
    });

    testWidgets('should call authService.login on valid submit', (tester) async {
      when(() => mockAuthService.login(any(), any()))
          .thenAnswer((_) async => true);

      await tester.pumpWidget(buildLoginForm());

      await tester.enterText(
        find.byKey(const Key('email_field')),
        'test@example.com',
      );
      await tester.enterText(
        find.byKey(const Key('password_field')),
        'password123',
      );
      await tester.tap(find.byType(ElevatedButton));
      await tester.pumpAndSettle();

      verify(() => mockAuthService.login('test@example.com', 'password123'))
          .called(1);
    });

    testWidgets('should show loading indicator during login', (tester) async {
      final completer = Completer<bool>();
      when(() => mockAuthService.login(any(), any()))
          .thenAnswer((_) => completer.future);

      await tester.pumpWidget(buildLoginForm());

      await tester.enterText(find.byKey(const Key('email_field')), 'test@example.com');
      await tester.enterText(find.byKey(const Key('password_field')), 'pass123');
      await tester.tap(find.byType(ElevatedButton));
      await tester.pump(); // 触发一帧，不等待动画

      expect(find.byType(CircularProgressIndicator), findsOneWidget);

      completer.complete(true);
      await tester.pumpAndSettle();

      expect(find.byType(CircularProgressIndicator), findsNothing);
    });
  });
}
```

**常用测试方法**：
- `tester.pumpWidget()` — 渲染 Widget
- `tester.tap()` — 点击操作
- `tester.enterText()` — 输入文本
- `tester.drag()` — 滑动操作
- `tester.pumpAndSettle()` — 等待所有动画和异步完成
- `tester.longPress()` — 长按
- `tester.pump(Duration())` — 推进指定时间

### 3. Golden Tests

通过截图对比检测 UI 回归：

```dart
testWidgets('login form golden test', (tester) async {
  await tester.pumpWidget(buildLoginForm());

  // 与已保存的截图进行像素级对比
  await expectLater(
    find.byType(LoginForm),
    matchesGoldenFile('goldens/login_form.png'),
  );
});

// 更新 Golden 文件：
// flutter test --update-goldens
```

**Golden 测试最佳实践**：
```dart
// 使用 golden_toolkit 增强多设备测试
testGoldens('login form responsive', (tester) async {
  final builder = GoldenBuilder.column()
    ..addScenario('Default', const LoginForm())
    ..addScenario('With Error', const LoginForm(error: 'Invalid'));

  await tester.pumpWidgetBuilder(builder.build());
  await screenMatchesGolden(tester, 'login_form_variants');
});
```

### 4. Mock 策略

#### Mockito（代码生成方式）
```dart
// 需要 @GenerateMocks 注解和 build_runner
@GenerateMocks([AuthService, UserRepository])
void main() {}

// 使用
final mockAuth = MockAuthService();
when(() => mockAuth.login(any(), any())).thenAnswer((_) async => true);
verify(() => mockAuth.login('a', 'b')).called(1);
```

#### Mocktail（无代码生成）
```dart
class MockAuthService extends Mock implements AuthService {}

final mockAuth = MockAuthService();
when(() => mockAuth.login(any(), any())).thenAnswer((_) async => true);

// 重置
reset(mockAuth);
```

#### 集成测试中的 Mock
```dart
// 使用 Provider 覆盖注入 Mock 依赖
testWidgets('integration with mocked services', (tester) async {
  await tester.pumpWidget(
    MultiProvider(
      providers: [
        Provider<AuthService>.value(value: mockAuthService),
        Provider<UserRepository>.value(value: mockUserRepo),
      ],
      child: const MyApp(),
    ),
  );
  // ...
});
```

### 5. 集成测试 (Integration Tests)

在真实设备/模拟器上运行的端到端测试：

```dart
// integration_test/app_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:my_app/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  group('End-to-End Test', () {
    testWidgets('login flow', (tester) async {
      app.main();
      await tester.pumpAndSettle();

      // 输入凭据
      await tester.enterText(find.byKey(Key('email')), 'test@example.com');
      await tester.enterText(find.byKey(Key('password')), 'password123');
      await tester.tap(find.byKey(Key('login_btn')));
      await tester.pumpAndSettle();

      // 验证登录成功后跳转到主页
      expect(find.text('欢迎回来'), findsOneWidget);
      expect(find.byType(HomePage), findsOneWidget);
    });

    testWidgets('create post flow', (tester) async {
      // ... 导航到发布页
      // ... 填写内容
      // ... 验证发布成功
    });
  });
}
```


### 7. BLoC 专用测试

```dart
// 使用 bloc_test 包
import 'package:bloc_test/bloc_test.dart';

class MockAuthBloc extends MockBloc<AuthEvent, AuthState> implements AuthBloc {}

void main() {
  group('AuthBloc', () {
    blocTest<AuthBloc, AuthState>(
      'emits [Loading, Authenticated] when login succeeds',
      build: () {
        when(() => mockAuth.login(any(), any())).thenAnswer((_) async => true);
        return AuthBloc(authService: mockAuth);
      },
      act: (bloc) => bloc.add(LoginRequested('test@email.com', 'pass')),
      expect: () => [
        isA<AuthLoading>(),
        isA<Authenticated>(),
      ],
    );

    blocTest<AuthBloc, AuthState>(
      'emits [Loading, Error] when login fails',
      build: () {
        when(() => mockAuth.login(any(), any())).thenThrow(AuthException('失败'));
        return AuthBloc(authService: mockAuth);
      },
      act: (bloc) => bloc.add(LoginRequested('test@email.com', 'wrong')),
      expect: () => [
        isA<AuthLoading>(),
        isA<AuthError>(),
      ],
    );
  });
}
```

## 相关概念

- **架构模式** — 可测试的 [架构模式](/wiki/Flutter/架构模式/)（如 Clean Architecture）天然支持分层测试
- **状态管理** — [状态管理](/wiki/Flutter/状态管理/) 的选择直接影响测试策略（bloc_test、riverpod_test 等）
- **CICD与发布** — 测试是 [CI/CD 与发布](/wiki/Flutter/CICD与发布/) 流水线的核心环节
- **Firebase与BaaS** — [Firebase](/wiki/Flutter/Firebase与BaaS/) 的 mock 测试策略（fake_cloud_firestore 等）
- **性能优化** — 集成测试可用于 [性能优化](/wiki/Flutter/性能优化/) 中的帧率基准测试

## 常见问题

### Q1: `pumpAndSettle` 超时怎么办？
`pumpAndSettle` 会等待所有动画完成。如果有无限循环动画（如 Lottie），改用 `pump()`：
```dart
await tester.pump(const Duration(seconds: 1));
await tester.pump(const Duration(seconds: 1));
```

### Q2: 如何测试使用 InheritedWidget/Provider 的 Widget？
```dart
await tester.pumpWidget(
  Provider<AuthService>.value(
    value: mockAuthService,
    child: const MaterialApp(home: MyWidget()),
  ),
);
```

### Q3: Mockito 和 Mocktail 选哪个？
- **Mocktail**：无需代码生成，零配置，推荐新项目使用
- **Mockito**：更成熟，社区资源多，需要 `build_runner` 生成代码

### Q4: 如何测试网络请求？
1. Mock HTTP Client（推荐 `http_mock_adapter` 或自定义 `MockClient`）
2. 使用 `HttpOverrides` 全局替换
3. 使用 DI 注入 Mock HTTP Client

```dart
class MockClient extends Mock implements http.Client {}

test('fetches data', () async {
  final client = MockClient();
  when(() => client.get(any())).thenAnswer(
    (_) async => http.Response('{"title": "test"}', 200),
  );
  final data = await fetchData(client);
  expect(data.title, 'test');
});
```

### Q5: 如何在 CI 中生成测试覆盖率报告？
```bash
flutter test --coverage
genhtml coverage/lcov.info -o coverage/html  # 需要 lcov
# 或使用 Codecov/Coveralls 等服务
```

### Q6: Widget 测试和集成测试的本质区别是什么？
- **Widget 测试**运行在桌面环境的虚拟渲染器中，速度快但不包含原生代码
- **集成测试**运行在真实设备/模拟器上，包含完整的平台通道和原生交互

### Q7: 如何测试 `Navigator.push` 等导航行为？
```dart
testWidgets('navigates to detail page', (tester) async {
  await tester.pumpWidget(MaterialApp(home: const ListPage()));

  await tester.tap(find.text('Item 1'));
  await tester.pumpAndSettle();

  expect(find.byType(DetailPage), findsOneWidget);
  expect(find.text('Item 1 Detail'), findsOneWidget);
});
```
