# 热更新与动态化

## 定义

热更新（Hot Update / Hot Fix）是指在不通过应用商店重新发布的情况下，动态更新应用的部分代码或资源，实现快速修复和功能迭代。在 Flutter 生态中，热更新是一个复杂且敏感的话题，涉及技术实现、风险控制和平台政策合规等多个维度。

Flutter 的热更新方案主要分为以下几类：

- **Shorebird Code Push**：Flutter 官方推荐的热更新方案，基于 Dart VM 的代码推送
- **自研 Dart 热修复**：基于 Dart JIT/AOT 编译原理的自研方案
- **资源热更新**：仅更新静态资源（图片、JSON、配置文件）
- **WebView 动态化**：将部分页面迁移到 WebView，通过 H5 实现动态更新
- **插件化架构**：将功能模块化，支持动态加载和更新

**核心挑战：**
- **平台政策**：Apple 和 Google 对热更新有严格限制
- **安全风险**：热更新可能引入安全漏洞
- **兼容性**：不同设备和系统版本的兼容性问题
- **调试困难**：热更新后的代码难以调试和回滚
- **性能影响**：热更新可能影响应用性能

热更新的核心概念：

- **Code Push**：将 Dart 代码推送到客户端执行
- **Patch**：热更新包，包含差异化的代码或资源
- **Rollout**：分批发布，控制更新范围
- **Rollback**：版本回滚机制
- **Diff/Patch**：差异比较和补丁生成
- **AOT/IOT**：编译模式对热更新的影响

## 核心原理

### 1. Shorebird Code Push 架构

Shorebird 是目前最成熟的 Flutter 热更新方案，由 Flutter 核心团队成员创建：

**Shorebird 工作原理：**
```
开发者代码 → Shorebird CLI 构建 → 生成 Patch Bundle
    ↓
Shorebird Cloud（存储/分发/管理）
    ↓
客户端 SDK 检查更新 → 下载 Patch → 应用到 Dart VM
```

**核心组件：**

- **Shorebird CLI**：命令行工具，用于构建、发布和管理热更新
- **Shorebird Engine**：定制的 Flutter 引擎，支持代码热替换
- **Shorebird Code Push Client**：客户端 SDK，负责检查、下载和应用 Patch
- **Shorebird Cloud**：云端服务，管理版本、分发 Patch、统计分析

**Shorebird 集成步骤：**
```bash
# 安装 Shorebird CLI
dart pub global activate shorebird_cli

# 登录 Shorebird
shorebird login

# 初始化项目
shorebird init

# 构建 Release 版本（包含 Shorebird 引擎）
shorebird release android -- --dart-define=SOME_VAR=value

# 发布热更新
shorebird patch android
```

**客户端集成：**
```dart
import 'package:shorebird_code_push/shorebird_code_push.dart';

final _shorebirdCodePush = ShorebirdCodePush();

class MyApp extends StatefulWidget {
  @override
  State<MyApp> createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  bool _isLoadingPatch = false;

  @override
  void initState() {
    super.initState();
    _checkForUpdate();
  }

  Future<void> _checkForUpdate() async {
    // 检查是否有可用更新
    final isUpdateAvailable = await _shorebirdCodePush.isNewPatchAvailableForDownload();
    
    if (isUpdateAvailable) {
      setState(() => _isLoadingPatch = true);
      
      // 下载 Patch
      await _shorebirdCodePush.downloadUpdateIfAvailable();
      
      // 提示用户重启应用以应用更新
      _showRestartDialog();
    }
  }

  void _showRestartDialog() {
    showDialog(
      context: context,
      builder: (context) => AlertDialog(
        title: Text('应用更新'),
        content: Text('发现新版本，需要重启应用以应用更新'),
        actions: [
          TextButton(
            onPressed: () => SystemNavigator.pop(),
            child: Text('立即重启'),
          ),
          TextButton(
            onPressed: () => Navigator.pop(context),
            child: Text('稍后'),
          ),
        ],
      ),
    );
  }
}
```

### 2. 热更新技术原理

**Dart 编译模式与热更新：**

- **JIT (Just-In-Time)**：解释执行，支持热重载，但性能较低
- **AOT (Ahead-Of-Time)**：预编译为机器码，性能高，但不支持热重载
- **Shorebird**：基于 AOT 编译，通过替换 Dart 代码段实现热更新

**Patch 生成原理：**
1. 基准版本（Base Release）构建
2. 修复版本（Patched Release）构建
3. Shorebird 生成差异 Patch
4. 客户端下载 Patch 并应用

**Patch 应用机制：**
```dart
// Shorebird 内部机制（简化）
class ShorebirdPatch {
  // 1. 下载 Patch 到本地存储
  Future<void> downloadPatch(String patchUrl) async {
    final patchData = await http.get(Uri.parse(patchUrl));
    await _savePatchToLocal(patchData.bodyBytes);
  }
  
  // 2. 验证 Patch 完整性
  Future<bool> verifyPatch(Uint8List patchData) async {
    final expectedHash = await _getExpectedHash();
    final actualHash = sha256.convert(patchData);
    return expectedHash == actualHash;
  }
  
  // 3. 应用 Patch（替换 Dart 代码段）
  Future<void> applyPatch() async {
    await _nativeApplyPatch(); // 调用原生代码
  }
}
```

**更新时机策略：**

- **后台检查**：应用进入后台时检查
- **定时检查**：定时任务周期性检查
- **强制更新**：关键修复强制用户更新

**分发策略：**

```dart
  Future<void> rollout(String version, double percentage) async {
    final users = await _getAllUsers();
    final rolloutUsers = users.take((users.length * percentage / 100).toInt());
    
    for (final user in rolloutUsers) {
      await _sendUpdateNotification(user);
    }
  }
  
  // 灰度发布
  Future<void> grayRelease(String version, List<String> tags) async {
    final users = await _getUsersByTags(tags);
    for (final user in users) {
      await _sendUpdateNotification(user);
    }
  }
  
  // A/B 测试
  Future<void> abTest(String versionA, String versionB, double ratio) async {
    final users = await _getAllUsers();
    final splitIndex = (users.length * ratio / 100).toInt();
    
    final groupA = users.take(splitIndex);
    final groupB = users.skip(splitIndex);
    
    await _sendUpdateToGroup(groupA, versionA);
    await _sendUpdateToGroup(groupB, versionB);
  }
}
```

### 4. 版本管理

**语义化版本与热更新：**
```
Major.Minor.Patch+BuildNumber
  │     │     │        │
  │     │     │        └── 构建号（每次构建递增）
  │     │     └─────────── 补丁版本（热更新）
  │     └───────────────── 次版本（功能更新）
  └─────────────────────── 主版本（重大更新）
```

**版本管理策略：**
- **主版本**：必须通过应用商店发布
- **次版本**：重大功能更新通过商店发布，小修复可热更新
- **补丁版本**：完全通过热更新发布
- **构建号**：每次构建递增，用于追踪具体版本

**版本检查机制：**

### 5. 风险控制

**热更新风险矩阵：**
```
风险类型 │ 影响程度 │ 发生概率 │ 控制措施
─────────┼─────────┼─────────┼──────────
功能异常 │   高    │   中    │ 灰度发布、快速回滚
性能下降 │   中    │   低    │ 性能监控、阈值告警
安全漏洞 │   高    │   低    │ 代码审查、签名验证
兼容性问题 │  中    │   中    │ 多设备测试、兼容性矩阵
数据损坏 │   高    │   低    │ 数据备份、事务处理
```

**风险控制措施：**

1. **灰度发布**：先小范围验证，逐步扩大
2. **快速回滚**：支持一键回滚到上一版本
3. **监控告警**：实时监控崩溃率、性能指标
4. **签名验证**：验证 Patch 的完整性
5. **回退机制**：应用失败时自动回退
6. **审计日志**：记录所有热更新操作

### 6. Apple/Google 政策合规

**Apple 政策（App Store Review Guideline）：**
- 禁止下载可执行代码（2.5.2）
- 允许下载 JavaScript 和 Web 内容
- 允许更新二进制资源（图片、配置）
- 必须在审核时说明热更新功能

**Google Play 政策：**
- 禁止动态加载违反安全策略的代码
- 允许通过 WebView 加载 Web 内容
- 允许更新静态资源
- 需要声明动态代码加载功能

**合规建议：**
- **资源更新**：完全合规，可放心使用
- **WebView 动态化**：合规，但需要良好的 WebView 管理
- **Shorebird**：需要评估合规性，可能面临审核风险
- **原生热修复**：通常不合规，不建议使用

**合规检查清单：**
```yaml
合规检查:
  - 是否下载可执行代码:
      - 是: 可能违规，需要评估
      - 否: 合规
  - 是否更新 WebView 内容:
      - 是: 合规
      - 否: 不适用
  - 是否更新静态资源:
      - 是: 合规
      - 否: 不适用
  - 是否声明热更新功能:
      - 是: 合规
      - 否: 需要补充声明
```

```dart
class ResourceUpdateService {
  final String resourceBaseUrl = 'https://your-cdn.com/resources';
  
  Future<void> checkResourceUpdate() async {
    // 获取资源版本
    final versionInfo = await _fetchVersionInfo();
    final localVersion = await _getLocalVersion();
    
    if (versionInfo.version > localVersion) {
      // 下载资源包
      final resourceZip = await _downloadResourcePack(versionInfo.url);
      
      // 解压并替换本地资源
      await _extractAndReplace(resourceZip);
      
      // 更新本地版本号
      await _updateLocalVersion(versionInfo.version);
    }
  }
  
  Future<void> _extractAndReplace(File zipFile) async {
    // 解压资源包
    final zip = ZipFile(zipFile);
    
    // 备份当前资源
    await _backupCurrentResources();
    
    // 替换资源
    for (final file in zip.files) {
      final localPath = _getLocalPath(file.name);
      await file.writeAsBytes(localPath);
    }
  }
}
```

### 案例 4：热更新风险控制

**场景**：实现热更新的风险控制和监控体系。

**实现：**
```dart
class HotUpdateRiskManager {
  // 分批发布
  Future<void> phasedRollout(String patchId, List<double> percentages) async {
    for (final percentage in percentages) {
      print('发布到 $percentage% 用户');
      
      // 发布到指定比例用户
      await _rolloutPatch(patchId, percentage);
      
      // 监控崩溃率
      final crashRate = await _monitorCrashRate(patchId);
      
      if (crashRate > 0.5) {
        // 崩溃率过高，回滚
        await _rollbackPatch(patchId);
        throw HotUpdateException('崩溃率过高: $crashRate%');
      }
      
      // 等待一段时间
      await Future.delayed(Duration(hours: 1));
    }
    
    // 全量发布
    await _rolloutPatch(patchId, 100);
  }
  
  // 快速回滚
  Future<void> emergencyRollback(String patchId) async {
    print('紧急回滚 Patch: $patchId');
    
    // 通知所有设备
    await _notifyRollback(patchId);
    
    // 清除 Patch
    await _clearPatch(patchId);
    
    // 恢复到基准版本
    await _restoreBaseRelease();
  }
}
```

## 常见问题

### Q1：Shorebird 热更新是否安全？

**安全性分析：**
- **代码签名**：Shorebird 使用代码签名验证 Patch 完整性
- **传输加密**：Patch 使用 HTTPS 传输
- **本地验证**：客户端验证 Patch 签名后再应用
- **回退机制**：应用失败自动回退到基准版本

**潜在风险：**
- 服务器被攻击可能导致恶意 Patch 分发
- 客户端被 Root 可能绕过验证
- Patch 本身可能包含 Bug

**缓解措施：**
- 使用安全的服务器和存储
- 实施代码审查和测试
- 分批发布和监控
- 支持快速回滚

### Q2：热更新的性能影响有多大？

**性能开销：**
- **下载**：Patch 文件大小通常几 KB 到几 MB
- **应用**：应用 Patch 耗时约 100-500ms
- **运行时**：几乎无性能影响（AOT 编译）

**优化建议：**
- **增量更新**：只下载差异部分
- **后台下载**：在 WiFi 下预下载
- **延迟应用**：在应用空闲时应用 Patch
- **压缩优化**：使用高效的压缩算法

### Q3：如何测试热更新功能？

**测试策略：**
1. **单元测试**：测试 Patch 生成和验证逻辑
2. **集成测试**：测试完整的更新流程
3. **设备测试**：在多种设备上测试兼容性
4. **性能测试**：测试下载和应用性能
5. **安全测试**：测试签名验证和回退机制

### Q4：热更新失败后如何回滚？

**回滚机制：**
1. **自动回退**：应用 Patch 失败自动回退到基准版本
2. **手动回滚**：通过 API 触发全局回滚
3. **版本切换**：支持切换到任意历史版本
4. **数据保护**：确保回滚不丢失用户数据

**回滚流程：**
```dart
class RollbackManager {
  Future<void> rollback(String targetVersion) async {
    // 1. 验证目标版本
    await _validateVersion(targetVersion);
    
    // 2. 备份当前状态
    await _backupCurrentState();
    
    // 3. 应用回滚
    await _applyRollback(targetVersion);
    
    // 4. 验证回滚结果
    await _verifyRollback();
    
    // 5. 清理旧 Patch
    await _cleanupOldPatches();
    
    // 6. 通知相关方
    await _notifyRollbackComplete(targetVersion);
  }
}
```

### Q5：热更新与原生模块的兼容性问题？

**常见问题：**
- 原生模块 API 变更导致热更新后不兼容
- 平台特定功能在热更新后失效
- 版本依赖冲突

**解决方案：**
1. **接口抽象**：将原生调用抽象为接口，热更新不直接修改原生代码
2. **版本管理**：维护原生模块版本与 Dart 代码版本的对应关系
3. **兼容性测试**：热更新前后进行兼容性测试
4. **降级策略**：热更新失败时降级到兼容版本

### Q6：如何监控热更新的效果？

**监控指标：**
- **更新成功率**：成功应用 Patch 的用户比例
- **更新耗时**：下载和应用 Patch 的时间
- **崩溃率**：热更新后的崩溃率变化
- **用户反馈**：热更新相关的用户投诉

**监控实现：**
```dart
class UpdateMonitor {
  Future<void> trackUpdateEvent(String patchId, UpdateEvent event) async {
    await Analytics.track('hot_update', {
      'patch_id': patchId,
      'event_type': event.type.toString(),
      'timestamp': DateTime.now().toIso8601String(),
      'user_id': await _getUserId(),
      'device_info': await _getDeviceInfo(),
    });
  }
  
  Future<void> trackPerformance(String patchId, Duration downloadTime, Duration applyTime) async {
    await Analytics.track('hot_update_performance', {
      'patch_id': patchId,
      'download_ms': downloadTime.inMilliseconds,
      'apply_ms': applyTime.inMilliseconds,
    });
  }
}
```
