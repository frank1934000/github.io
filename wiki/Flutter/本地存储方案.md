# 本地存储方案

## 定义

本地存储方案是指在移动设备上持久化保存数据的技术手段，用于实现数据缓存、离线访问、用户偏好设置等功能。Flutter生态中提供了多种本地存储选择，主要包括键值对存储（SharedPreferences/Hive）、轻量级数据库（Isar）、关系型数据库（SQLite/sqflite）等。选择合适的存储方案需要根据数据结构复杂度、查询需求、性能要求等因素综合考量。

## 核心原理

### 1. 存储方案对比

| 特性 | SharedPreferences | Hive | Isar | SQLite (sqflite) |
|------|-------------------|------|------|------------------|
| **类型** | 键值对 | 键值对 | NoSQL文档 | 关系型数据库 |
| **性能** | 中等 | 高 | 极高 | 中高 |
| **复杂查询** | 不支持 | 有限 | 支持 | 支持 |
| **数据模型** | 基本类型 | 对象 | 对象 | 表/行 |
| **索引** | 不支持 | 支持 | 支持 | 支持 |
| **事务** | 不支持 | 有限 | 支持 | 支持 |
| **平台支持** | 全平台 | 全平台 | 全平台 | 全平台 |
| **适用场景** | 配置/偏好 | 缓存/简单数据 | 复杂NoSQL | 关系型数据 |

### 2. Hive — 轻量级键值存储

Hive是一个用纯Dart编写的高性能键值数据库，无需原生依赖：

```dart
// 初始化Hive
await Hive.initFlutter();
Hive.registerAdapter(UserAdapter());
await Hive.openBox<User>('users');

// 读写操作
final box = Hive.box<User>('users');
await box.put('currentUser', user);
final cachedUser = box.get('currentUser');
```

**优点**：速度快、跨平台、支持加密、TypeAdapter自定义类型
**缺点**：不支持复杂查询、索引能力有限

### 3. Isar — 高性能NoSQL数据库

Isar是Flutter官方推荐的新一代NoSQL数据库，性能极高：

```dart
// 初始化Isar
final dir = await getApplicationDocumentsDirectory();
final isar = await Isar.open(
  [UserSchema, PostSchema],
  directory: dir.path,
);

// 查询操作
final users = await isar.users
  .filter()
  .ageGreaterThan(18)
  .sortByCreatedAtDesc()
  .limit(20)
  .findAll();

// 索引支持
@collection
class User {
  Id id = Isar.autoIncrement;
  @Index(unique: true)
  late String email;
  late String name;
}
```

**优点**：查询性能极强、支持复杂索引、响应式查询、零原生依赖
**缺点**：相对年轻，社区生态不如SQLite成熟

### 4. SQLite — 关系型数据库

SQLite是经典的关系型数据库，通过sqflite包在Flutter中使用：

```dart
// 初始化数据库
final db = await openDatabase(
  'app.db',
  version: 1,
  onCreate: (db, version) async {
    await db.execute('''
      CREATE TABLE users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        email TEXT UNIQUE NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
      )
    ''');
  },
);

// CRUD操作
await db.insert('users', {'name': 'John', 'email': 'john@example.com'});
final users = await db.query('users', where: 'age > ?', whereArgs: [18]);
```

**优点**：SQL语法强大、成熟稳定、适合关系型数据
**缺点**：性能不如Hive/Isar、配置相对复杂

### 5. 离线优先模式（Offline-First）

离线优先是一种应用架构模式，优先从本地存储读取数据，同时后台同步远程数据：

```dart
class UserRepository {
  final LocalDataSource local;
  final RemoteDataSource remote;

  Future<User> getUser(String id) async {
    // 1. 先返回本地缓存
    final cached = await local.getUser(id);
    if (cached != null) {
      // 2. 后台同步最新数据
      _syncInBackground(id);
      return cached;
    }
    // 3. 无缓存时从远程获取
    final user = await remote.getUser(id);
    await local.saveUser(user);
    return user;
  }

  void _syncInBackground(String id) async {
    try {
      final fresh = await remote.getUser(id);
      await local.saveUser(fresh);
    } catch (_) {
      // 同步失败静默处理
    }
  }
}
```

### 6. 缓存策略

常见的缓存策略包括：

- **TTL缓存**：设置过期时间，过期后重新获取
- **LRU缓存**：最近最少使用淘汰策略
- **增量同步**：仅同步变更数据，减少流量消耗
- **ETag/Last-Modified**：利用HTTP缓存头减少重复传输

```dart
// TTL缓存示例
class TtlCache<T> {
  final Duration ttl;
  final Map<String, _CacheEntry<T>> _cache = {};

  Future<T?> get(String key, Future<T> Function() loader) async {
    final entry = _cache[key];
    if (entry != null && !entry.isExpired) {
      return entry.value;
    }
    final value = await loader();
    _cache[key] = _CacheEntry(value, DateTime.now().add(ttl));
    return value;
  }
}
```

## 常见问题

### Q1: Hive和Isar如何选择？
简单键值存储选Hive，需要复杂查询和索引选Isar。Isar性能更优但学习曲线稍高。

### Q2: SharedPreferences适合存储大量数据吗？
不适合。SharedPreferences会将所有数据加载到内存，大量数据会导致启动变慢，建议限制在几KB以内。

### Q3: 如何实现数据加密存储？
Hive支持AES加密（`Hive.openBox('box', encryptionCipher: HiveAesCipher(key))`），Isar支持数据库级加密，SQLite可使用SQLCipher。

### Q4: 离线同步如何处理数据冲突？
常见策略包括：最后写入胜出（Last Write Wins）、时间戳合并、版本号冲突检测等。

### Q5: 如何优化大量数据的查询性能？
- 添加合适的索引
- 分页查询，避免一次性加载所有数据
- 使用Isar的响应式查询减少手动刷新

### Q6: 数据库迁移如何处理？
SQLite：在`onUpgrade`回调中执行ALTER TABLE；Isar：使用版本号和迁移脚本；Hive：通常需要重建Box。
