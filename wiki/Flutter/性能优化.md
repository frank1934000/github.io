# Flutter 性能优化

## 定义

Flutter 性能优化是指通过分析工具定位瓶颈、优化渲染管线、减少包体积、管理内存等手段，
使 Flutter 应用在目标设备上达到 60fps（或 120fps）流畅运行的系统工程。

性能优化涵盖多个维度：
- **帧率与渲染**：消除 Jank（卡顿），保持 UI 流畅
- **启动时间**：缩短冷启动耗时
- **包体积**：减小安装包大小，提升下载和安装速度
- **内存管理**：减少内存占用，避免泄漏
- **能耗优化**：降低 CPU/GPU 使用率，延长电池续航

### 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 帧率 (FPS) | ≥ 60fps | 120Hz 设备目标 120fps |
| 帧构建时间 | < 16ms | 120Hz 时 < 8ms |
| 帧光栅化时间 | < 16ms | GPU 侧渲染 |
| 冷启动时间 | < 2s | 用户可感知的启动 |
| APK 大小 | < 30MB | 按业务复杂度调整 |
| 内存占用 | 合理范围 | 避免持续增长 |

## 核心原理

### 1. Flutter 渲染管线

```
 UI Thread                    GPU Thread             Display
┌──────────┐   ┌─────────────┐   ┌──────────┐   ┌──────────┐
│  Build    │──▶│   Layout    │──▶│  Paint   │──▶│ Composite│──▶ 屏幕
│ (Widget)  │   │  (Sliver)   │   │ (Layer)  │   │  (Layer  │
│           │   │             │   │          │   │   Tree)  │
└──────────┘   └─────────────┘   └──────────┘   └──────────┘
     ▲               ▲               ▲              ▲
  Dart 侧        Dart 侧         Dart 侧        GPU 侧
  (CPU)           (CPU)           (CPU)          (GPU)
```

- **Build**：Widget 树构建（应尽量轻量）
- **Layout**：计算 Widget 大小和位置
- **Paint**：绘制到 Layer
- **Composite**：Layer 合成上屏

### 3. 渲染优化技巧

#### 避免不必要的重建
```dart
// ❌ 错误：整个 Widget 树重建
class BadWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        const HeavyHeader(),  // 不需要重建但被重建了
        Text('${DateTime.now()}'),  // 只有这个需要更新
      ],
    );
  }
}

// ✅ 正确：最小化重建范围
class GoodWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return const Column(
      children: [
        HeavyHeader(),  // const 构造，不会重建
        _TimeDisplay(), // 抽离为独立 Widget
      ],
    );
  }
}

class _TimeDisplay extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('${DateTime.now()}');
  }
}
```

#### 使用 const Widget
```dart
// const Widget 编译期常量，不会参与 rebuild
const Icon(Icons.star, color: Colors.yellow)
const Text('Hello', style: TextStyle(fontSize: 16))
const Padding(padding: EdgeInsets.all(8.0))
```

#### 列表优化
```dart
// ❌ 使用 Column + map 会一次性构建所有项
Column(
  children: items.map((item) => ListItem(item)).toList(),
)

// ✅ 使用 ListView.builder 懒加载
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListItem(items[index]),
)

// ✅ 带预加载的列表
ListView.builder(
  itemCount: items.length,
  cacheExtent: 250,  // 预加载 250 像素范围
  itemBuilder: (context, index) => ListItem(items[index]),
)

// ✅ 固定高度列表（跳过 layout 计算）
ListView.builder(
  itemExtent: 80.0,  // 固定行高
  itemCount: items.length,
  itemBuilder: (context, index) => ListItem(items[index]),
)
```

#### RepaintBoundary
```dart
// 隔离重绘区域
RepaintBoundary(
  child: ExpensiveToPaintWidget(),  // 变化不会影响兄弟 Widget
)

// 在 DevTools 中检查 Widget 重叠情况
// 用 debugPaintLayerBordersEnabled = true 查看 Layer 边界
```

#### 避免 Opacity Widget
```dart
// ❌ Opacity 会导致整个子树离屏渲染
Opacity(opacity: 0.5, child: LargeWidget())

// ✅ 使用 AnimatedOpacity（Flutter 引擎优化过）
AnimatedOpacity(opacity: 0.5, duration: Duration(milliseconds: 300), child: ...)

// ✅ 对于图片，直接使用 color 和 colorBlendMode
Image.network(
  url,
  color: Color.fromRGBO(255, 255, 255, 0.5),
  colorBlendMode: BlendMode.modulate,
)
```

### 5. 内存管理

#### 常见内存泄漏场景
```dart
// ❌ 忘记取消 StreamSubscription
class _MyState extends State<MyWidget> {
  StreamSubscription? _subscription;

  @override
  void initState() {
    super.initState();
    _subscription = stream.listen((data) {
      setState(() {});  // 如果 Widget 已 dispose，会崩溃
    });
  }

  @override
  void dispose() {
    _subscription?.cancel();  // ✅ 必须取消
    super.dispose();
  }
}

// ❌ 忘记移除 listener
late final ScrollController _controller;

@override
void initState() {
  super.initState();
  _controller = ScrollController();
  _controller.addListener(_onScroll);
}

@override
void dispose() {
  _controller.removeListener(_onScroll);  // ✅ 移除 listener
  _controller.dispose();                  // ✅ 释放 controller
  super.dispose();
}
```

### 6. Jank 检测与诊断

```dart
// 自动检测 Jank
import 'package:flutter/scheduler.dart';

// 设置帧回调监控
SchedulerBinding.instance.addTimingsCallback((List<FrameTiming> timings) {
  for (final timing in timings) {
    final buildDuration = timing.buildDuration.inMilliseconds;
    final rasterDuration = timing.rasterDuration.inMilliseconds;
    final totalDuration = timing.totalSpan.inMilliseconds;

    if (totalDuration > 16) {
      print('⚠️ Jank detected: ${totalDuration}ms '
            '(build: ${buildDuration}ms, raster: ${rasterDuration}ms)');
    }
  }
});
```

#### 常见 Jank 原因及解决方案

| 原因 | 诊断方式 | 解决方案 |
|------|---------|---------|
| Build 过重 | DevTools Timeline: build 耗时长 | 拆分 Widget，使用 const |
| 布局过于复杂 | Layout 耗时 > 10ms | 减少嵌套，使用 Sliver |
| 主线程 I/O | Timeline 中有 I/O 事件 | 使用 compute() 或 Isolate |
| 大图片解码 | Image cache miss | 预缓存、限制解码尺寸 |
| 过度绘制 | DevTools: 多层重叠绘制 | 减少层叠，使用 RepaintBoundary |
| setState 范围过大 | 整棵树 rebuild | 使用 Selector/Consumer |
| 动画卡顿 | GPU 线程超时 | 使用 AnimatedBuilder 优化 |

## 常见问题

### Q1: 为什么 Debug 模式很卡但 Release 模式流畅？
Debug 模式启用了断言检查、JIT 编译、调试信息等额外开销。性能测试必须使用 `--profile` 或 `--release` 模式。

### Q2: 如何判断卡顿发生在 UI 线程还是 GPU 线程？
查看 Performance Overlay：
- **上半部分红色**（UI Thread）：Dart 代码太慢（Build/Layout 重）
- **下半部分红色**（GPU Thread）：渲染太慢（Paint 层过多、大图片、ClipPath 等）
- **两部分都红**：两者都有问题

### Q3: ListView 和 GridView 如何优化大量数据？
```dart
// 使用 Sliver 优化复杂滚动布局
CustomScrollView(
  slivers: [
    SliverList(
      delegate: SliverChildBuilderDelegate(
        (context, index) => ListItem(items[index]),
        childCount: items.length,
      ),
    ),
    SliverGrid(
      delegate: SliverChildBuilderDelegate(
        (context, index) => GridItem(gridItems[index]),
        childCount: gridItems.length,
      ),
      gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(crossAxisCount: 2),
    ),
  ],
)
```

### Q4: 如何减少 Widget 重建次数？
1. 使用 `const` 构造函数
2. 使用 `Selector` / `Consumer` 替代 `Provider.of` 监听整个 Widget
3. 使用 `Bloc` 的 `buildWhen` / `listenWhen`
4. 将变化部分提取为独立 Widget
5. 使用 `RepaintBoundary` 隔离重绘

### Q5: `compute()` 和 Isolate 有什么区别？
- `compute()` 是便捷 API，适合一次性后台任务（单次计算）
- `Isolate` 适合长期运行的后台任务（消息通信、持续处理）
- 两者都运行在独立 Dart Isolate 中，不阻塞 UI 线程

```dart
// compute 用法
final result = await compute(heavyComputation, data);

// Isolate 用法
final receivePort = ReceivePort();
await Isolate.spawn(_isolateEntry, receivePort.sendPort);
```

### Q6: 包体积太大如何排查？
```bash
# 生成 APK 体积分析
flutter build apk --analyze-size --target-platform android-arm64

# 使用 apkanalyzer（Android SDK 工具）
apkanalyzer apk file-size app-release.apk
apkanalyzer apk resources app-release.apk --file-pattern ".*\\.png"
```

常见大体积来源：
1. 原生库（如 TensorFlow、OpenCV）
2. 未压缩的图片资源
3. 字体文件
4. 未使用的依赖包

### Q7: 图片加载慢如何优化？
1. 使用 `cached_network_image` 缓存网络图片
2. 指定 `cacheWidth` / `cacheHeight` 限制解码尺寸
3. 使用 WebP 格式替代 PNG/JPG
4. 使用 CDN 加速图片加载
5. 使用 `precacheImage()` 预缓存关键图片
6. 对列表中的图片使用 `ListView.builder` 懒加载
