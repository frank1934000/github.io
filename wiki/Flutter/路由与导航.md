# 路由与导航

## 定义

路由（Routing）是指在 Flutter 应用中管理页面之间导航的机制。它定义了 URL/路径与页面 Widget 的映射关系，控制页面的跳转、返回、动画过渡以及深链接处理。

**核心概念：**

- **Route**：一个页面的抽象表示
- **Navigator**：管理路由栈（Route Stack）的 Widget，支持 push/pop 操作
- **Router**（Router API / Navigator 2.0）：Flutter 的声明式路由 API，支持深链接和 Web URL 同步
- **GoRouter**：基于 Navigator 2.0 的高层封装，提供声明式、易用的路由配置
- **Deep Link**：通过外部 URL 直接打开应用内特定页面
- **Named Route**：通过字符串名称标识页面路径
- **Nested Navigation**：在子区域（如 BottomNavigationBar 的各 Tab）内独立管理导航栈

## 核心原理

### 1. Flutter 路由演进

**Navigator 1.0（命令式）：**
```dart
// Push
Navigator.push(context, MaterialPageRoute(
  builder: (context) => DetailPage(id: 42),
));

// Named route
Navigator.pushNamed(context, '/detail/42');

// Pop
Navigator.pop(context);
```

**Navigator 2.0（声明式）：**
- 路由状态驱动 UI，而非命令式 push/pop
- 支持深链接和 Web URL
- `Router` + `RouteInformationParser` + `RouterDelegate` 三件套

**GoRouter（推荐）：**
- 在 Navigator 2.0 基础上简化了大量样板代码
- 声明式路由配置
- 内置深链接支持
- 类型安全的路径参数

### 2. GoRouter 核心用法

```dart
final goRouter = GoRouter(
  routes: [
    GoRoute(
      path: '/',
      builder: (context, state) => const HomeScreen(),
      routes: [
        GoRoute(
          path: 'detail/:id',
          builder: (context, state) {
            final id = state.pathParameters['id']!;
            return DetailScreen(id: id);
          },
        ),
      ],
    ),
  ],
);

// MaterialApp 中使用
MaterialApp.router(
  routerConfig: goRouter,
)
```

### 3. 声明式路由（Declarative Routing）

声明式路由的核心思想是**用状态决定路由**，而非用命令操作路由栈：

```dart
GoRoute(
  path: '/login',
  redirect: (context, state) {
    final isLoggedIn = authService.isLoggedIn;
    if (!isLoggedIn) return '/login';
    return null; // 不重定向
  },
),
```

这种方式让路由逻辑与业务状态解耦，更容易理解和维护。

### 4. 深链接（Deep Links）

深链接允许外部 URL 直接导航到应用内特定页面：

**Android 配置（AndroidManifest.xml）：**
```xml
<intent-filter>
  <action android:name="android.intent.action.VIEW" />
  <category android:name="android.intent.category.DEFAULT" />
  <category android:name="android.intent.category.BROWSABLE" />
  <data android:scheme="https" android:host="example.com" />
</intent-filter>
```

**iOS 配置（Associated Domains）：**
```
applinks:example.com
```

**GoRouter 自动处理路径匹配：**
```dart
GoRoute(
  path: '/product/:id',  // 匹配 /product/123
  builder: (context, state) {
    final id = state.pathParameters['id'];
    return ProductScreen(id: id!);
  },
),
```

### 5. Named Routes

GoRouter 支持通过 `name` 命名路由，方便类型安全地引用：

```dart
GoRoute(
  name: 'product',
  path: '/product/:id',
  builder: (context, state) => ProductScreen(
    id: state.pathParameters['id']!,
  ),
);

// 导航时使用 name
context.pushNamed('product', pathParameters: {'id': '42'});
// 或
context.goNamed('product', pathParameters: {'id': '42'});
```

**`go()` vs `push()`：**
- `context.go('/path')`：替换路由栈（适用于深链接/主路径切换）
- `context.push('/path')`：压入新页面（保留返回栈）

### 6. Nested Navigation（嵌套路由）

对于 BottomNavigationBar + Tab 切换的场景，每个 Tab 需要独立的导航栈：

```dart
StatefulShellRoute.indexedStack(
  builder: (context, state, navigationShell) {
    return ScaffoldWithNavBar(navigationShell: navigationShell);
  },
  branches: [
    StatefulShellBranch(routes: [
      GoRoute(path: '/home', builder: (_, __) => HomeScreen()),
      GoRoute(path: '/home/detail', builder: (_, __) => HomeDetail()),
    ]),
    StatefulShellBranch(routes: [
      GoRoute(path: '/profile', builder: (_, __) => ProfileScreen()),
      GoRoute(path: '/profile/settings', builder: (_, __) => Settings()),
    ]),
  ],
)
```

### 7. 路由守卫与重定向

```dart
GoRouter(
  redirect: (context, state) {
    final isLoggedIn = ref.read(authProvider);
    final isOnLogin = state.matchedLocation == '/login';

    if (!isLoggedIn && !isOnLogin) return '/login';
    if (isLoggedIn && isOnLogin) return '/';
    return null;
  },
  routes: [/* ... */],
)
```

### 8. 页面过渡动画

```dart
GoRoute(
  path: '/detail',
  pageBuilder: (context, state) {
    return CustomTransitionPage(
      child: DetailScreen(),
      transitionsBuilder: (context, animation, secondaryAnimation, child) {
        return FadeTransition(opacity: animation, child: child);
      },
    );
  },
)
```

## 常见问题

### Q1: `go()` 和 `push()` 有什么区别？

- `context.go('/path')`：等价于"替换整个路径"，适用于顶级导航切换（如 Tab 切换、深链接跳转），不会在栈中保留前一个路由
- `context.push('/path')`：等价于 `Navigator.push`，在栈上压入新页面，可以 pop 返回
- 简单理解：`go` 是"去那里"，`push` 是"打开一个新页面"

### Q2: GoRouter 的 `path` 和 `name` 该用哪个？

- **path**：适合简单场景和深链接映射
- **name**：适合复杂项目，重构时改路径不会影响已有的导航调用，编译时类型检查

推荐在项目中统一使用 `name` 导航。

### Q3: 嵌套路由中如何保持各 Tab 的页面状态？

使用 `StatefulShellRoute.indexedStack`（GoRouter 10+），它内部用 `IndexedStack` 保持各 Branch 的 Widget 存活。旧版本需要手动实现 `Navigator` 嵌套。

### Q4: 深链接测试怎么搞？

- **Android**：`adb shell am start -a android.intent.action.VIEW -d "https://example.com/product/42"`
- **iOS**：在 Safari 中输入 URL 或使用 `xcrun simctl openurl booted "https://example.com/product/42"`
- **Flutter**：使用 `uni_links` 或 `app_links` 包的调试模式

### Q5: 路由守卫（redirect）不生效？

常见原因：
1. `redirect` 中需要返回 `null` 表示不重定向，而非空字符串
2. 确保 `refreshListenable` 传入了认证状态的 `Listenable`（如 `ChangeNotifier`），否则路由不会在状态变化时重新评估
3. 嵌套路由中，父路由的 redirect 子路由不一定继承，需在对应层级设置

```dart
GoRouter(
  refreshListenable: authStateNotifier, // 关键！
  redirect: (context, state) { /* ... */ },
)
```

### Q6: 如何在 GoRouter 中处理 404 页面？

```dart
GoRouter(
  errorBuilder: (context, state) => NotFoundScreen(error: state.error),
  routes: [/* ... */],
)
```

`errorBuilder` 会在没有匹配到任何路由时调用。
