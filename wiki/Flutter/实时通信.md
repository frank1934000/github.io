# 实时通信

## 定义

实时通信（Real-time Communication）是指客户端与服务器之间建立持久连接，实现消息即时推送和双向数据交换的技术体系。

- **WebSocket**：基于 TCP 的全双工通信协议，客户端和服务器可以在任意时刻互相发送数据，无需轮询。
- **实时聊天（Real-time Chat）**：基于 WebSocket 的即时消息收发系统，支持文本、图片、文件等多种消息类型。
- **通知推送（Notification Push）**：服务端主动向客户端推送通知消息，包括应用内通知和系统级推送。
- **长连接管理（Long Connection Management）**：维持 WebSocket 持久连接的生命周期管理，包括心跳检测、状态监控等。
- **重连策略（Reconnection Strategy）**：连接断开后的自动恢复机制，包括重试间隔、退避算法、最大重试次数等。

## 核心原理

### WebSocket 协议原理

WebSocket 通过 HTTP 升级握手建立连接，之后在同一个 TCP 连接上进行双向通信：

```
客户端                                          服务器
  │                                               │
  │──── HTTP GET /chat (Upgrade: websocket) ─────→│
  │←─── HTTP 101 Switching Protocols ─────────────│
  │                                               │
  │◄══════ WebSocket 全双工通信通道 ══════════════►│
  │    (Frame: 文本 / 二进制 / Ping / Pong / Close)│
  │                                               │
  │──── Close Frame ─────────────────────────────→│
  │←─── Close Frame ─────────────────────────────│
```

**WebSocket 帧格式**：
- **Text Frame**（opcode 0x1）：UTF-8 文本数据
- **Binary Frame**（opcode 0x2）：二进制数据
- **Ping/Pong**（opcode 0x9/0xA）：心跳保活
- **Close**（opcode 0x8）：关闭连接

**与 HTTP 轮询对比**：

| 特性 | HTTP 轮询 | WebSocket |
|------|----------|-----------|
| 连接方式 | 请求-响应 | 持久连接 |
| 实时性 | 取决于轮询间隔 | 毫秒级 |
| 服务器开销 | 每次请求都需解析 HTTP 头 | 仅连接建立时握手 |
| 数据方向 | 客户端发起 | 双向 |
| 流量消耗 | 高（重复 HTTP 头） | 低（轻量帧头） |

### Flutter 中的 WebSocket 集成

Flutter 通过 `dart:io` 提供的 `WebSocket` 类或第三方包 `web_socket_channel` 实现 WebSocket 通信：

```
┌──────────────────────────────────────────────┐
│                 Flutter App                   │
│                                              │
│  UI Layer (Widget Tree)                      │
│    ↓                                         │
│  State Management (Provider/Bloc/Riverpod)   │
│    ↓                                         │

│    ↓                                         │
│  WebSocket Client (连接管理层)               │
│    │  - 连接建立/断开                         │

│    │  - 自动重连                              │
│    │  - 消息队列                              │
│    ↓                                         │
│  WebSocket Channel (底层传输)                │
│    ↓↓↓ TCP / TLS                            │
│  Server                                      │
└──────────────────────────────────────────────┘
```

### 长连接管理核心机制

**心跳检测（Heartbeat）**：
- 客户端定期发送 Ping 帧（通常 15-30 秒间隔）
- 服务器返回 Pong 帧
- 超过 N 次未收到 Pong，判定连接断开
- 心跳失败后触发重连流程

**连接状态机**：

```
         connect()
Idle ──────────→ Connecting
                   │
         connected │  error / timeout
                   ↓
              Connected ←──── ReconnectSuccess
                   │                ↑
         disconnect│                │
                   ↓                │
              Disconnected ──→ Reconnecting
                (手动)     (自动重连)  │
                   ↑                   │ reconnect()
                   │                   │
                   └───────────────────┘
                        多次失败后 → Failed
```

### 重连策略设计

**指数退避（Exponential Backoff）**：
```
delay = min(baseDelay * 2^attempt + random_jitter, maxDelay)

示例：
  第 1 次重连：1 秒 + 随机抖动
  第 2 次重连：2 秒 + 随机抖动
  第 3 次重连：4 秒 + 随机抖动
  第 4 次重连：8 秒 + 随机抖动
  ...最大间隔 30 秒
```

**抖动（Jitter）的作用**：防止大量客户端同时重连导致服务器雪崩（Thundering Herd）。

### 消息可靠投递

- **消息确认（ACK）**：客户端发送消息后等待服务器 ACK 确认
- **消息去重**：每条消息携带唯一 ID，接收端根据 ID 去重
- **离线消息队列**：连接断开期间的消息暂存本地队列，重连后批量补发
- **消息排序**：使用递增序列号保证消息顺序

完整实现参考：[Flutter + WebSocket 实战：实时聊天、通知推送、长连接管理](/categories/Flutter/Flutter-WebSocket-实战-实时聊天-通知推送-长连接管理/)

**案例一：WebSocket 连接管理器**

```dart
class WebSocketManager {
  WebSocket? _socket;
  Timer? _heartbeatTimer;
  Timer? _reconnectTimer;
  int _reconnectAttempt = 0;
  final _maxReconnectAttempts = 10;
  final _baseDelay = Duration(seconds: 1);
  final _maxDelay = Duration(seconds: 30);
  final _heartbeatInterval = Duration(seconds: 15);

  final _messageController = StreamController<Map<String, dynamic>>.broadcast();
  Stream<Map<String, dynamic>> get messageStream => _messageController.stream;

  final _stateController = StreamController<ConnectionState>.broadcast();
  Stream<ConnectionState> get stateStream => _stateController.stream;

  Future<void> connect(String url) async {
    _stateController.add(ConnectionState.connecting);
    try {
      _socket = await WebSocket.connect(url);
      _stateController.add(ConnectionState.connected);
      _reconnectAttempt = 0;
      _startHeartbeat();
      _socket!.listen(
        _onMessage,
        onDone: _onDisconnected,
        onError: _onError,
      );
    } catch (e) {
      _onError(e);
    }
  }

  void _startHeartbeat() {
    _heartbeatTimer?.cancel();
    _heartbeatTimer = Timer.periodic(_heartbeatInterval, (_) {
      if (_socket?.readyState == WebSocket.open) {
        _socket!.add(jsonEncode({'type': 'ping', 'timestamp': DateTime.now().millisecondsSinceEpoch}));
      } else {
        _onDisconnected();
      }
    });
  }

  void _onMessage(dynamic data) {
    final message = jsonDecode(data as String) as Map<String, dynamic>;
    if (message['type'] == 'pong') return; // 心跳响应，忽略
    _messageController.add(message);
  }

  void _onDisconnected() {
    _heartbeatTimer?.cancel();
    _stateController.add(ConnectionState.disconnected);
    _scheduleReconnect();
  }

  void _onError(dynamic error) {
    _heartbeatTimer?.cancel();
    _stateController.add(ConnectionState.disconnected);
    _scheduleReconnect();
  }

  void _scheduleReconnect() {
    if (_reconnectAttempt >= _maxReconnectAttempts) {
      _stateController.add(ConnectionState.failed);
      return;
    }
    final delay = _calculateBackoff(_reconnectAttempt);
    _reconnectAttempt++;
    _reconnectTimer?.cancel();
    _reconnectTimer = Timer(delay, () => connect(_currentUrl!));
  }

  Duration _calculateBackoff(int attempt) {
    final ms = _baseDelay.inMilliseconds * pow(2, attempt).toInt();
    final jitter = Random().nextInt(1000); // 随机抖动
    return Duration(milliseconds: min(ms + jitter, _maxDelay.inMilliseconds));
  }

  void send(Map<String, dynamic> message) {
    if (_socket?.readyState == WebSocket.open) {
      _socket!.add(jsonEncode(message));
    } else {
      // 离线消息队列：暂存待发送
      _offlineQueue.add(message);
    }
  }

  void dispose() {
    _heartbeatTimer?.cancel();
    _reconnectTimer?.cancel();
    _socket?.close();
    _messageController.close();
    _stateController.close();
  }
}
```

**案例二：实时聊天消息 Widget**

```dart
class ChatScreen extends StatefulWidget {
  @override
  _ChatScreenState createState() => _ChatScreenState();
}

class _ChatScreenState extends State<ChatScreen> {
  final _messages = <ChatMessage>[];
  final _controller = TextEditingController();
  late WebSocketManager _ws;

  @override
  void initState() {
    super.initState();
    _ws = WebSocketManager();
    _ws.connect('wss://chat.example.com/ws');
    _ws.messageStream.listen((data) {
      setState(() {
        _messages.add(ChatMessage.fromJson(data));
      });
    });
  }

  void _sendMessage() {
    if (_controller.text.isEmpty) return;
    final msg = {
      'type': 'chat',
      'content': _controller.text,
      'timestamp': DateTime.now().toIso8601String(),
    };
    _ws.send(msg);
    setState(() {
      _messages.add(ChatMessage.fromLocal(msg));
    });
    _controller.clear();
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        // 连接状态指示器
        StreamBuilder<ConnectionState>(
          stream: _ws.stateStream,
          builder: (context, snapshot) {
            if (snapshot.data == ConnectionState.connected) return SizedBox.shrink();
            return Container(
              color: Colors.orange,
              padding: EdgeInsets.all(8),
              child: Text('正在重连...', style: TextStyle(color: Colors.white)),
            );
          },
        ),
        // 消息列表
        Expanded(
          child: ListView.builder(
            itemCount: _messages.length,
            itemBuilder: (context, index) => MessageBubble(message: _messages[index]),
          ),
        ),
        // 输入框
        Padding(
          padding: EdgeInsets.all(8),
          child: Row(
            children: [
              Expanded(child: TextField(controller: _controller)),
              IconButton(icon: Icon(Icons.send), onPressed: _sendMessage),
            ],
          ),
        ),
      ],
    );
  }
}
```

## 常见问题

### 1. WebSocket 连接在后台被系统断开怎么办？

移动系统（iOS/Android）在应用进入后台时会限制网络活动。解决方案：
- 使用 `flutter_background_service` 或 WorkManager 维持后台连接
- iOS 可配置 `Info.plist` 的 `UIBackgroundModes` 中的 `fetch` 和 `remote-notification`
- 应用回到前台时主动触发重连
- 对于真正需要后台推送的场景，结合 FCM/APNs 原生推送

### 2. 如何处理 WebSocket 消息的乱序到达？

每条消息携带递增的序列号（sequence number），接收端维护 `lastProcessedSeq`。收到消息后按序列号存入缓冲区，只有连续的消息才能被消费。间隔消息等待补发。

### 3. 心跳 Ping/Pong 多久发一次合适？

通常 15-30 秒。太频繁浪费电量和流量，太稀疏无法及时发现断连。建议：
- Wi-Fi 环境：30 秒
- 移动网络：15 秒
- 根据网络类型动态调整
- 服务器应在超时（如 60 秒无任何帧）后主动断开连接

### 4. 消息发送失败如何保证可靠性？

实现消息确认（ACK）机制：
1. 客户端发送消息，本地标记为 `sending` 状态
2. 服务器收到后返回 ACK（携带消息 ID）
3. 客户端收到 ACK，更新为 `sent` 状态
4. 超时未收到 ACK，重发消息（需幂等处理）
5. 服务器根据消息 ID 去重

### 5. 多个页面共用同一个 WebSocket 连接怎么设计？

使用单例模式管理 WebSocket 连接，通过依赖注入（如 Provider）在全局共享。各页面通过 `StreamBuilder` 监听感兴趣的消息类型。消息路由层根据 `type` 字段分发到不同的 Stream：

```dart
// 按消息类型过滤
Stream<Map<String, dynamic>> messagesOfType(String type) {
  return messageStream.where((msg) => msg['type'] == type);
}
```

### 6. WebSocket 连接安全如何保障？

- **WSS 协议**：使用 `wss://` 代替 `ws://`，数据通过 TLS 加密传输
- **Token 认证**：连接 URL 中携带 token 参数或在握手后发送认证消息
- **证书固定（Certificate Pinning）**：防止中间人攻击
- **消息签名**：关键消息添加 HMAC 签名防篡改
- **速率限制**：防止客户端恶意发送大量消息
