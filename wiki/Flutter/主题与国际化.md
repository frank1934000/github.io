# 主题与国际化

## 定义

主题（Theme）与国际化（Internationalization, i18n）是 Flutter 应用在视觉呈现和语言适配方面的两大核心体系。

- **ThemeData**：Flutter 中描述应用全局视觉样式的数据结构，包含颜色方案（ColorScheme）、字体样式（TextTheme）、按钮样式、输入框装饰等几乎所有 Material 组件的默认外观配置。
- **暗黑模式（Dark Mode）**：根据系统或用户偏好，在浅色主题和深色主题之间动态切换的能力。
- **flutter_localizations**：Flutter 官方提供的国际化支持包，为 Material 和 Cupertino 组件提供多语言翻译文本。
- **多语言支持（Multi-language）**：通过 `intl` 和 `arb` 文件为应用添加多种语言翻译，并在运行时动态切换语言。
- **RTL 布局（Right-to-Left）**：支持阿拉伯语、希伯来语等从右到左书写方向的布局自动翻转。

## 核心原理

### ThemeData 的设计架构

Flutter 的主题系统基于 InheritedWidget 模式，`ThemeData` 通过 `Theme` Widget 注入到 Widget 树中，任何子 Widget 都可以通过 `Theme.of(context)` 获取当前主题配置。

```
ThemeData
├── colorScheme          // 颜色方案（primary, secondary, surface 等）
├── textTheme            // 文字样式（headline, body, caption 等）
├── appBarTheme          // AppBar 专属样式
├── buttonTheme          // 按钮样式
├── cardTheme            // 卡片样式
├── inputDecorationTheme // 输入框样式
├── brightness           // Brightness.light 或 Brightness.dark
└── ...更多 Material 组件主题
```

**暗黑模式切换原理**：
1. 应用维护一个 `ThemeMode` 状态（`light` / `dark` / `system`）
2. `MaterialApp` 的 `theme` 属性设置浅色 ThemeData，`darkTheme` 属性设置深色 ThemeData
3. 当 `ThemeMode` 变化时，Flutter 框架自动选择对应的 ThemeData 并重建依赖 `Theme.of(context)` 的 Widget

**主题持久化**：
- 使用 `shared_preferences` 存储用户选择的 ThemeMode
- 应用启动时读取持久化配置，决定初始主题
- 如选择 `system` 模式，则监听 `MediaQuery.platformBrightnessOf(context)` 跟随系统变化

### 国际化原理

Flutter 国际化体系由三部分组成：

1. **flutter_localizations**：提供 Material/Cupertino 组件的官方翻译（如日期选择器、对话框按钮等）
2. **intl 包**：提供消息翻译、日期/数字格式化、复数规则等工具
3. **ARB（Application Resource Bundle）文件**：JSON 格式的翻译资源文件，每种语言一个 `.arb` 文件

```
lib/l10n/
├── app_en.arb    // 英文翻译
├── app_zh.arb    // 中文翻译
├── app_ar.arb    // 阿拉伯语翻译（RTL）
└── app_localizations.dart  // 自动生成的访问类
```

**ARB 文件工作流程**：
1. 在 `pubspec.yaml` 中配置 `generate: true` 和 `l10n.yaml`
2. 编写 ARB 翻译文件（`app_en.arb` 为主语言）
3. 运行 `flutter gen-l10n` 自动生成 Dart 访问类
4. 通过 `AppLocalizations.of(context)!.helloWorld` 访问翻译文本

### RTL 布局支持

Flutter 的 RTL 支持基于 `Directionality` Widget：
- 设置 `MaterialApp` 的 `localizationsDelegates` 和 `supportedLocales`
- 当语言切换为 RTL 语言时，`Directionality` 自动设为 `TextDirection.rtl`
- 使用 `EdgeInsetsDirectional`（替代 `EdgeInsets`）和 `AlignmentDirectional` 确保布局正确翻转
- `mainAxisAlignment` 和 `crossAxisAlignment` 自动根据方向调整

## 实战案例

### 暗黑模式与主题持久化

完整实现参考：[Flutter 暗黑模式实战：ThemeData 动态切换与主题持久化](/categories/Flutter/Flutter-暗黑模式实战-ThemeData-动态切换与主题持久化/)

**核心实现思路**：

```dart
// 主题状态管理
class ThemeProvider extends ChangeNotifier {
  ThemeMode _themeMode = ThemeMode.system;

  ThemeMode get themeMode => _themeMode;

  Future<void> setThemeMode(ThemeMode mode) async {
    _themeMode = mode;
    notifyListeners();
    final prefs = await SharedPreferences.getInstance();
    await prefs.setString('themeMode', mode.toString());
  }

  Future<void> loadThemeMode() async {
    final prefs = await SharedPreferences.getInstance();
    final saved = prefs.getString('themeMode');
    if (saved != null) {
      _themeMode = ThemeMode.values.firstWhere(
        (e) => e.toString() == saved,
        orElse: () => ThemeMode.system,
      );
    }
  }
}

// MaterialApp 配置
MaterialApp(
  themeMode: themeProvider.themeMode,
  theme: ThemeData.light().copyWith(
    colorScheme: ColorScheme.fromSeed(
      seedColor: Colors.blue,
      brightness: Brightness.light,
    ),
  ),
  darkTheme: ThemeData.dark().copyWith(
    colorScheme: ColorScheme.fromSeed(
      seedColor: Colors.blue,
      brightness: Brightness.dark,
    ),
  ),
)
```

**关键点**：
- 使用 `ColorScheme.fromSeed` 生成协调的配色方案
- `system` 模式下自动跟随系统的明暗切换
- 切换过程是原子性的，不会出现中间状态

### 国际化与多语言支持

完整实现参考：[Flutter 国际化实战：flutter_localizations 多语言与 RTL 支持](/categories/Flutter/Flutter-国际化实战-flutter_localizations-多语言与-RTL-支持/)

**核心实现思路**：

```yaml
# l10n.yaml
arb-dir: lib/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
```

```json
// app_en.arb
{
  "@@locale": "en",
  "helloWorld": "Hello World!",
  "@helloWorld": { "description": "A greeting message" },
  "greeting": "Hello, {name}!",
  "@greeting": {
    "description": "A personalized greeting",
    "placeholders": { "name": { "type": "String" } }
  },
  "itemCount": "{count,plural, =0{No items} =1{1 item} other{{count} items}}",
  "@itemCount": {
    "placeholders": { "count": { "type": "int" } }
  }
}
```

```dart
// 使用
Text(AppLocalizations.of(context)!.helloWorld)
Text(AppLocalizations.of(context)!.greeting('Flutter'))
```

**语言切换实现**：

```dart
MaterialApp(
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: AppLocalizations.supportedLocales,
  locale: localeProvider.currentLocale, // 用户选择的语言
)
```

## 常见问题

### 1. 如何让应用跟随系统深浅色模式自动切换？

设置 `MaterialApp` 的 `themeMode: ThemeMode.system`，并同时提供 `theme` 和 `darkTheme`。Flutter 会自动读取系统的亮度设置并应用对应主题。

### 2. 主题切换后页面没有更新怎么办？

确保主题状态变更时调用了 `notifyListeners()`（对于 ChangeNotifier）或等效的状态通知机制。同时确认依赖主题的 Widget 通过 `Theme.of(context)` 或 `context.watch<ThemeProvider>()` 建立了响应关系。

### 3. 如何自定义暗黑模式下的特定组件样式？

使用 `copyWith` 方法覆盖 `darkTheme` 中的特定样式。例如 `ThemeData.dark().copyWith(appBarTheme: AppBarTheme(...))`。也可以通过 `Theme.of(context).brightness == Brightness.dark` 判断当前模式。

### 4. ARB 文件中的占位符如何使用？

在 ARB 文件中使用 `{variableName}` 定义占位符，对应生成的方法会包含同名参数。复数用 `{count,plural, =0{...} =1{...} other{...}}` 语法。日期格式用 `{date,datetime,short}` 语法。

### 5. 如何实现 RTL 布局下的自定义 Widget？

使用 `Directionality.of(context)` 获取当前文字方向，使用 `EdgeInsetsDirectional` 代替 `EdgeInsets`，使用 `AlignmentDirectional` 代替 `Alignment`，使用 `AnimatedPositionedDirectional` 等方向感知 Widget。避免在布局代码中硬编码 `left`/`right`，改用 `start`/`end`。

### 6. flutter gen-l10n 不生成代码怎么办？

检查以下几点：
- 确认 `pubspec.yaml` 中设置了 `generate: true`
- 确认 `l10n.yaml` 配置文件路径正确
- 确认 ARB 文件的 JSON 格式正确（无语法错误）
- 运行 `flutter gen-l10n --arb-dir=lib/l10n` 指定路径重试
- 检查 `pubspec.yaml` 中是否正确依赖了 `flutter_localizations`
