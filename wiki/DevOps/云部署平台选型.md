# 云部署平台选型

## 定义

云部署平台（PaaS — Platform as a Service）为开发者提供"Git Push 即部署"的应用托管能力，隐藏底层基础设施复杂度（服务器、网络、负载均衡、SSL），让开发者专注于业务代码。Heroku 取消免费套餐后，2026 年的 PaaS 市场形成了 Coolify（自托管）、Railway、Fly.io、Render 四强格局。

## 核心对比

### Coolify（自托管 PaaS）
- **定位**：开源 Heroku/Vercel 替代，自托管在自己的 VPS 上
- **优势**：数据完全自主、零平台费用（仅 VPS 成本）、支持 Git/Docker/Docker Compose 多种部署方式
- **劣势**：需要自行维护服务器、无全球边缘网络
- **适合**：注重数据主权、成本敏感、有运维能力的团队

### Railway
- **定位**：极简 PaaS，强调开发者体验
- **优势**：部署速度最快（秒级）、界面直观、内置数据库、按用量计费
- **劣势**：免费额度有限、无自定义域名免费方案、高级功能需付费
- **适合**：原型验证、Hackathon、快速 MVP

### Fly.io
- **定位**：全球化边缘部署平台
- **优势**：全球 30+ 区域边缘节点、支持 GPU 实例、Dockerfile 原生支持、WebSocket 友好
- **劣势**：配置复杂（`fly.toml`）、冷启动问题、定价模型不够透明
- **适合**：需要全球低延迟、实时应用、国际化产品

### Render
- **定位**：Heroku 的直接继承者
- **优势**：与 Heroku 体验最接近、内置 CDN、自动 SSL、Preview Environments、Zero-Downtime Deploy
- **劣势**：功能比 Fly.io 少、扩展性不如 K8s
- **适合**：从 Heroku 迁移、中等规模 Web 应用

## 选型决策矩阵

| 维度 | Coolify | Railway | Fly.io | Render |
|---|---|---|---|---|
| 部署方式 | 自托管 | 全托管 | 全托管 | 全托管 |
| 全球边缘 | ❌ | ❌ | ✅ 30+ | ✅ CDN |
| 数据主权 | ✅ 完全 | ❌ | ❌ | ❌ |
| 免费额度 | 无限（自付 VPS） | $5/月 | 3 shared VMs | 750h/月 |
| Queue Worker | ✅ | ✅ | ✅ | ✅ |
| 数据库 | ✅ | ✅ 内置 | ✅ 内置 | ✅ 内置 |
| 自动扩缩 | ❌ | ✅ | ✅ | ✅ |
| 配置复杂度 | 中 | 低 | 高 | 低 |
| 最适合 | 自主可控 | 快速原型 | 全球化 | Heroku 迁移 |

## 部署配置示例

### Laravel + Docker Compose（Coolify）
```yaml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    environment:
      - APP_ENV=production
      - DB_HOST=mysql
    depends_on:
      - mysql
      - redis
  queue:
    build: .
    command: php artisan queue:work
  scheduler:
    build: .
    command: php artisan schedule:work
  mysql:
    image: mysql:8.0
    volumes:
      - mysql-data:/var/lib/mysql
```

### fly.toml（Fly.io）
```toml
app = "b2c-api"
primary_region = "nrt"

[build]
  dockerfile = "Dockerfile"

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = true
  min_machines_running = 1

[[vm]]
  cpu_kind = "shared"
  cpus = 1
  memory_mb = 512
```

## 常见问题

### 冷启动延迟
- 设置 `min_machines_running >= 1` 保持至少一个实例常驻
- 使用 Health Check 确保实例就绪后再接流量
- 优化 Docker 镜像体积减少拉取时间

### 费用失控
- 设置用量告警和预算上限
- 使用 `auto_stop_machines` 自动停止空闲实例
- 对比按量计费 vs 固定价格方案
