# 基础设施即代码

## 定义

基础设施即代码（Infrastructure as Code, IaC）是用代码定义、管理和配置 IT 基础设施的实践。与手动在云控制台"点点点"不同，IaC 将 VPC、EC2、RDS、S3 等资源声明为可版本控制的代码文件，实现基础设施的可复现、可审计、可扩展。Terraform 是当前最流行的 IaC 工具，由 HashiCorp 开发。

## 核心原理

### Terraform 工作流
```
terraform init    # 初始化 Provider 插件
terraform plan    # 生成执行计划（预览变更）
terraform apply   # 执行变更
terraform destroy # 销毁资源
```

### HCL 声明式语法
Terraform 使用 HashiCorp Configuration Language（HCL）描述期望状态：

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
  subnet_id     = aws_subnet.public.id

  tags = {
    Name = "b2c-web-${var.environment}"
  }
}
```

### Provider 生态
Provider 是 Terraform 与云平台/服务的连接器。主流 Provider：
- **aws** / **alicloud** / **azure** / **gcp** — 云平台
- **github** — GitHub 仓库/Secret 管理
- **docker** / **kubernetes** — 容器编排
- **mysql** / **postgresql** — 数据库管理

### State 管理
State 文件记录了 Terraform 管理的所有资源的当前状态，是 IaC 的核心数据：

- **本地 State**：适合个人开发，但团队协作时存在冲突风险
- **远程 Backend**：S3 + DynamoDB（AWS）、OSS（阿里云）、Terraform Cloud
- **State Lock**：通过 DynamoDB/文件锁防止并发修改

```hcl
terraform {
  backend "s3" {
    bucket         = "terraform-state-prod"
    key            = "b2c/terraform.tfstate"
    region         = "ap-southeast-1"
    dynamodb_table = "terraform-locks"
    encrypt        = true
  }
}
```

### Module 模块化
Module 是 Terraform 的复用单元，将相关资源封装为可组合的模块：

```
modules/
├── vpc/
│   ├── main.tf
│   ├── variables.tf
│   └── outputs.tf
├── rds/
│   ├── main.tf
│   ├── variables.tf
│   └── outputs.tf
└── ec2/
    ├── main.tf
    ├── variables.tf
    └── outputs.tf
```

### 资源导入与迁移
对于已有的手动创建资源，可以使用 `terraform import` 将其纳入 IaC 管理：

```bash
terraform import aws_instance.web i-0123456789abcdef0
```

## 最佳实践

### 环境隔离
```
environments/
├── dev/
│   └── main.tf    # 引用 modules，设置 dev 变量
├── staging/
│   └── main.tf    # 引用 modules，设置 staging 变量
└── prod/
    └── main.tf    # 引用 modules，设置 prod 变量
```

### 安全最佳实践
- 敏感信息使用 `sensitive = true` 标记或外部密钥管理
- `terraform plan` 输出不显示敏感值
- 使用 `.gitignore` 排除 `.terraform/` 和 `*.tfstate`
- 开启 State 文件加密

### CI/CD 集成
```yaml
# GitHub Actions
- name: Terraform Plan
  run: terraform plan -out=tfplan
- name: Terraform Apply
  if: github.ref == 'refs/heads/main'
  run: terraform apply -auto-approve tfplan
```

## 常见问题

### State 文件冲突
- 使用远程 Backend + State Lock
- 拆分 State，每个环境/模块独立 State
- 使用 `terraform state mv` 迁移资源而非手动编辑

### Plan 与 Apply 结果不一致
- 检查是否有外部变更（手动修改了云资源）
- 使用 `terraform refresh` 同步远程状态
- 避免在 Plan 和 Apply 之间有时间间隔

### Provider 版本升级导致破坏性变更
- 使用 `required_providers` 锁定版本范围
- 先在 dev 环境验证，再到 prod 执行
- 阅读 Provider 的 CHANGELOG 了解破坏性变更
