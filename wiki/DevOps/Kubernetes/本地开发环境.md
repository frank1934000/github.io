# 本地开发环境：minikube / kind / k3s

## 定义

在 macOS（Apple Silicon）上搭建本地 Kubernetes 开发环境，minikube、kind、k3s 是三大主流方案。目标是创建一个轻量、快速、尽量贴近生产的本地 K8s 环境，避免"本地跑得好好的，上线就炸"的问题。

## 核心原理

### 为什么需要本地 K8s？

Docker Compose 无法模拟 K8s 特有的行为：
- Service Discovery（DNS 解析）
- ConfigMap / Secret 注入
- HPA 自动扩缩容
- Ingress 路由规则
- Pod 调度与资源限制

### 深度对比

| 维度 | minikube | kind | k3s |
|---|---|---|---|
| **启动速度** | 30-60s | 10-20s | 5-15s |
| **内存占用** | 2-4GB | 1-2GB | 512MB-1GB |
| **多节点支持** | ✅ | ✅ | ✅ |
| **Ingress 支持** | ✅ addon | 需手动配置 | ✅ 内置 Traefik |
| **LoadBalancer** | ✅ tunnel | 需 MetalLB | ✅ 内置 |
| **持久卷** | ✅ hostPath | ✅ hostPath | ✅ local-path |
| **CI/CD 集成** | 一般 | ✅ 最佳 | 一般 |
| **Apple Silicon** | ✅ | ✅ | ✅ |
| **功能完整度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |

### 选型建议

| 场景 | 推荐方案 | 理由 |
|---|---|---|
| 日常开发（功能完整） | minikube | addon 生态最丰富，最贴近生产 |
| CI/CD 测试 | kind | 启动最快，Docker-in-Docker 友好 |
| 资源受限/轻量开发 | k3s | 内存占用最低，内置 Ingress |
| 多节点测试 | kind | 原生多节点支持，配置简单 |

## 实战案例

来自博客文章：
- [minikube vs kind vs k3s 选型实战](/categories/DevOps/kubernetes-minikube-kind-k3s-guide-laravel/) - macOS 本地 K8s 开发环境深度对比

### minikube 快速启动

```bash
# 安装
brew install minikube

# 启动（推荐 Docker driver + M 芯片 Mac）
minikube start \
  --driver=docker \
  --cpus=4 \
  --memory=8192 \
  --kubernetes-version=v1.31.0 \
  --addons=ingress,metrics-server,registry

# 启用 Ingress addon
minikube addons enable ingress

# 查看 dashboard
minikube dashboard
```

### kind 快速启动

```bash
# 安装
brew install kind

# 创建多节点集群
cat <<EOF | kind create cluster --config=-
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
  kubeadmConfigPatches:
  - |
    kind: InitConfiguration
    nodeRegistration:
      kubeletExtraArgs:
        node-labels: "ingress-ready=true"
  extraPortMappings:
  - containerPort: 80
    hostPort: 80
    protocol: TCP
  - containerPort: 443
    hostPort: 443
    protocol: TCP
- role: worker
- role: worker
EOF
```

### k3s 快速启动

```bash
# 安装（使用 k3d，k3s 的 Docker 封装）
brew install k3d

# 创建集群
k3d cluster create laravel-dev \
  --servers 1 \
  --agents 2 \
  --port "8080:80@loadbalancer" \
  --port "8443:443@loadbalancer"

# 使用 kubectl
kubectl cluster-info
```

## 常见问题

### minikube 在 Apple Silicon 上启动失败
- 确保 Docker Desktop 已安装且运行正常
- 使用 `--driver=docker` 而非 HyperKit
- 检查是否分配了足够资源（至少 4GB 内存）

### kind 的 Ingress 不工作
- kind 需要手动安装 Ingress Controller
- 端口映射需要在创建集群时配置
- 参考 [kind 官方文档](https://kind.sigs.k8s.io/docs/user/ingress/)

### k3d 端口冲突
- 检查 hostPort 是否被占用：`lsof -i :80`
- 使用 `--port` 参数映射到不同端口
