# 渐进式发布：Argo Rollouts

## 定义

**Argo Rollouts** 是 Kubernetes 的渐进式发布控制器，支持金丝雀发布（Canary）和蓝绿部署（Blue/Green），通过 Prometheus 指标自动分析发布质量，实现带度量、可暂停、可自动回滚的发布流程。

## 核心原理

### 为什么需要 Argo Rollouts？

Deployment 的滚动更新只代表容器活着，不代表业务安全。一次发布可能：
- Pod 启来很快，但上线 3 分钟后 P95 飙到 1.8s
- 错误率被 Redis 超时拉高
- Deployment 还在继续滚，等人工回退时坏版本已吃掉大半流量

**核心价值：把发布从一次性切换，改成带度量、可暂停、可自动回滚的过程。**

### Rollout CRD（金丝雀发布）

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: laravel-api
spec:
  replicas: 4
  selector:
    matchLabels:
      app: laravel-api
  template:
    metadata:
      labels:
        app: laravel-api
    spec:
      containers:
      - name: php-fpm
        image: registry.example.com/laravel-api:v1.2.4
  strategy:
    canary:
      # 流量切分步骤
      steps:
      - setWeight: 10         # 10% 流量到 canary
      - pause: {duration: 2m} # 观察 2 分钟
      - analysis:             # 自动分析
          templates:
          - templateName: success-rate
          - templateName: latency
      - setWeight: 30
      - pause: {duration: 5m}
      - analysis:
          templates:
          - templateName: success-rate
      - setWeight: 60
      - pause: {duration: 5m}
      - setWeight: 100
      # 流量路由
      trafficRouting:
        nginx:
          stableIngress: laravel-api-ingress
```

### AnalysisTemplate（自动分析）

```yaml
apiVersion: argoproj.io/v1alpha1
kind: AnalysisTemplate
metadata:
  name: success-rate
spec:
  args:
  - name: service-name
    value: laravel-api-svc
  metrics:
  - name: success-rate
    interval: 30s
    count: 5
    successCondition: result[0] >= 0.99
    failureLimit: 2
    provider:
      prometheus:
        address: http://prometheus:9090
        query: |
          sum(rate(http_requests_total{service="{{args.service-name}}",status=~"2.."}[2m]))
          /
          sum(rate(http_requests_total{service="{{args.service-name}}"}[2m]))
```

### 蓝绿部署

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: laravel-api
spec:
  strategy:
    blueGreen:
      activeService: laravel-api-active    # 生产流量
      previewService: laravel-api-preview  # 预览流量
      autoPromotionEnabled: false          # 手动确认切换
      prePromotionAnalysis:
        templates:
        - templateName: smoke-test
```

## 常见问题

### AnalysisTemplate 指标持续失败
- 检查 Prometheus 查询语法
- 确认指标名称和标签是否正确
- 调整 `failureLimit` 允许一定失败次数

### 金丝雀流量切分不准确
- 确保 Ingress Controller 支持流量权重（Nginx Ingress 需要额外配置）
- 检查 stable/canary Service 的 selector 是否正确
- 使用 Istio VirtualService 实现精确的流量百分比

### 发布中断后的恢复
- Argo Rollouts 会记住发布状态，恢复后继续执行
- 使用 `kubectl argo rollouts abort` 手动中止并回滚
- 使用 `kubectl argo rollouts promote` 手动推进到下一步
