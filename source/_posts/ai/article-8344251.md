## 前言

2026 年，AI 已经深度融入技术写作的每个环节。作为一名写了 400+ 篇技术文章的博主，我深刻感受到这场变革带来的机遇与挑战。

这篇文章不是在告诉你"用 AI 写文章"，而是分享一套经过实战验证的**人机协作写作方法论**——如何让 AI 成为你的写作加速器，而不是替代品。

## 为什么需要新的写作方法论？

### 传统写作的困境

技术博客写作的痛点一直存在：

- **选题难**：技术热点轮转快，今天写 K8s 还是写 Rust？
- **效率低**：一篇文章从构思到发布，可能需要 4-8 小时
- **质量不稳定**：灵感好时产出精品，状态差时凑字数
- **知识更新慢**：手动追踪技术演进，容易遗漏重要变化

### AI 带来的范式转变

AI 不是让你放弃思考，而是帮你：

1. **加速信息获取**：从 30 分钟调研缩短到 5 分钟
2. **突破创作瓶颈**：用 AI 生成初稿框架，你负责打磨
3. **保持更新频率**：降低单篇成本，提高产出
4. **提升内容质量**：AI 补充你知识盲区，你专注深度思考

## 方法论框架：PACP 模型

我总结出 **PACP 写作模型**：

- **P**lan（规划）：选题、大纲、目标读者
- **A**rchitect（架构）：结构设计、内容分层
- **C**reate（创作）：AI 辅助写作 + 人工打磨
- **P**ublish（发布）：SEO 优化、多平台分发

### Plan 阶段：选题决策矩阵

别再纠结"今天写什么"，用数据驱动选题：

```php
<?php

namespace App\Blog\Planning;

class TopicDecisionMatrix
{
    /**
     * 选题评估维度
     */
    private array $dimensions = [
        'relevance'    => 0.3,  // 技术热度
        'expertise'    => 0.3,  // 个人深度
        'audience'     => 0.2,  // 读者需求
        'uniqueness'   => 0.2,  // 内容差异
    ];

    /**
     * 评估选题得分
     */
    public function evaluate(string $topic, array $context): float
    {
        $scores = [
            'relevance'   => $this->scoreRelevance($topic, $context),
            'expertise'   => $this->scoreExpertise($topic),
            'audience'    => $this->scoreAudience($topic, $context),
            'uniqueness'  => $this->scoreUniqueness($topic),
        ];

        $total = 0;
        foreach ($this->dimensions as $dim => $weight) {
            $total += $scores[$dim] * $weight;
        }

        return round($total, 2);
    }

    /**
     * 技术热度评分（基于搜索趋势、GitHub star 等）
     */
    private function scoreRelevance(string $topic, array $context): float
    {
        // 实际项目中可接入 Google Trends API、GitHub API
        $trendData = $context['trends'] ?? [];

        if (empty($trendData)) {
            return 5.0; // 默认中等热度
        }

        // 简化逻辑：根据近期搜索量估算
        $recentSearches = $trendData['last_30_days'] ?? 0;
        $maxSearches = $trendData['max_ever'] ?? 1;

        return min(10, ($recentSearches / $maxSearches) * 10);
    }

    /**
     * 个人专业度评分
     */
    private function scoreExpertise(string $topic): float
    {
        // 基于你写过的相关文章数量、项目经验
        $relatedArticles = $this->countRelatedArticles($topic);
        $projectExperience = $this->hasProjectExperience($topic);

        $score = min(10, $relatedArticles * 2);

        if ($projectExperience) {
            $score = min(10, $score + 3);
        }

        return $score;
    }

    private function countRelatedArticles(string $topic): int
    {
        // 实际项目中扫描 _posts 目录
        $postsPath = '/Users/michael/GitHub/mikeah2011.github.io/source/_posts/';
        $count = 0;

        $iterator = new \RecursiveIteratorIterator(
            new \RecursiveDirectoryIterator($postsPath)
        );

        foreach ($iterator as $file) {
            if ($file->getExtension() !== 'md') continue;

            $content = file_get_contents($file->getPathname());
            if (str_contains($content, $topic)) {
                $count++;
            }
        }

        return $count;
    }

    private function hasProjectExperience(string $topic): bool
    {
        // 检查是否有相关项目
        $projectDirs = [
            '~/KKday/kkday-b2c-api',
            '~/GitHub/claude-watch',
        ];

        foreach ($projectDirs as $dir) {
            $realDir = realpath($dir);
            if ($realDir && file_exists($realDir)) {
                $readme = $realDir . '/README.md';
                if (file_exists($readme)) {
                    $content = file_get_contents($readme);
                    if (str_contains($content, $topic)) {
                        return true;
                    }
                }
            }
        }

        return false;
    }

    private function scoreAudience(string $topic, array $context): float
    {
        // 读取评论、邮件反馈中的高频问题
        $feedback = $context['feedback'] ?? [];
        $mentionCount = 0;

        foreach ($feedback as $item) {
            if (str_contains($item, $topic)) {
                $mentionCount++;
            }
        }

        return min(10, $mentionCount * 2 + 3);
    }

    private function scoreUniqueness(string $topic): float
    {
        // 已有文章太多的话题，分数降低
        $existingArticles = $this->countRelatedArticles($topic);

        if ($existingArticles === 0) return 10.0;
        if ($existingArticles <= 2) return 7.0;
        if ($existingArticles <= 5) return 4.0;
        return 2.0;
    }
}
```

### Architect 阶段：AI 辅助大纲生成

<?php

namespace App\Blog\Architect;

class OutlineGenerator
{
    /**
     * 生成大纲候选方案

### Create 阶段：分层写作策略

这是核心环节。我的策略是**分层写作**：

**第一层：骨架层（人工完成）**
- 确定核心论点
- 写出每段的第一句话（主题句）
- 标注代码示例的位置和功能

**第二层：血肉层（AI 辅助）**
- 用 AI 扩展每个段落
- 生成代码示例的框架
- 补充背景知识和引用

**第三层：灵魂层（人工完成）**
- 注入个人经验和见解
- 添加"踩坑记录"和"注意事项"
- 确保代码可运行且经过测试

```php
<?php

namespace App\Blog\Create;

class ContentWriter
{
    /**
     * 第一层：构建骨架
     */
    public function buildSkeleton(string $topic, array $outline): string
    {
        $skeleton = "# {$outline['title']}\n\n";

        foreach ($outline['sections'] as $section) {
            $skeleton .= "## {$section['title']}\n\n";
            $skeleton .= "【TODO: {$section['key_point']}】\n\n";
            $skeleton .= "```php\n// TODO: {$section['code_example']}\n```\n\n";
        }

        return $skeleton;
    }

    /**
     * 第二层：AI 扩展
     */
    public function expandWithAI(string $skeleton, array $context): string
    {
        $sections = $this->parseSkeleton($skeleton);

        foreach ($sections as &$section) {
            if (str_contains($section, '【TODO:')) {
                $expanded = $this->aiExpand($section, $context);
                $section = $expanded;
            }
        }

        return implode("\n", $sections);
    }

    private function aiExpand(string $todo, array $context): string
    {
        $topic = $context['topic'];
        $prompt = <<<PROMPT
请扩展以下内容，要求：
1. 用自然的中文技术写作风格
2. 保留代码示例的占位符
3. 不要使用"首先"、"其次"等连接词
4. 加入实际的使用场景和思考

原始内容：{$todo}
PROMPT;

        // 调用 AI API
        return $this->callAI($prompt);
    }

    /**
     * 第三层：注入灵魂
     */
    public function injectSoul(string $content, array $experiences): string
    {
        // 添加个人踩坑记录
        $soulSections = [];

        foreach ($experiences as $exp) {
            $soulSections[] = <<<MARKDOWN

### ⚠️ 踩坑记录

> {$exp['description']}

```php
// 问题代码
{$exp['bad_code']}

// 解决方案
{$exp['good_code']}
```

**教训：** {$exp['lesson']}
MARKDOWN;
        }

        // 插入到文章适当位置
        return $this->insertAtStrategicPoints($content, $soulSections);
    }

    private function parseSkeleton(string $skeleton): array
    {
        return explode("\n\n", $skeleton);
    }

    private function callAI(string $prompt): string
    {
        // AI API 调用
        return "AI 生成的内容";
    }

    private function insertAtStrategicPoints(string $content, array $sections): string
    {
        // 在每个代码示例后插入相关踩坑记录
        $inserted = 0;
        $result = preg_replace_callback(
            '/(```[\s\S]*?```)/',
            function ($matches) use ($sections, &$inserted) {
                $code = $matches[1];
                if ($inserted < count($sections)) {
                    return $code . "\n\n" . $sections[$inserted++];
                }
                return $code;
            },
            $content
        );

        return $result ?? $content;
    }
}
```

### Publish 阶段：自动化发布

文章写完后，发布流程也应该自动化：

```php
<?php

namespace App\Blog\Publish;

use Symfony\Component\Process\Process;

class Publisher
{
    private string $blogPath = '/Users/michael/GitHub/mikeah2011.github.io';

    /**
     * 发布流程
     */
    public function publish(string $articlePath): array
    {
        $steps = [
            'validate'   => $this->validateArticle($articlePath),
            'optimize'   => $this->optimizeSEO($articlePath),
            'git_add'    => $this->gitAdd($articlePath),
            'git_commit' => $this->gitCommit($articlePath),
            'git_push'   => $this->gitPush(),
            'notify'     => $this->notifyChannels($articlePath),
        ];

        $results = [];
        foreach ($steps as $step => $result) {
            if (!$result['success']) {
                return [
                    'success' => false,
                    'failed_at' => $step,
                    'error' => $result['error'] ?? 'Unknown error',
                ];
            }
            $results[$step] = $result;
        }

        return ['success' => true, 'steps' => $results];
    }

    private function validateArticle(string $path): array
    {
        $content = file_get_contents($path);

        // 检查 frontmatter
        if (!preg_match('/^---\n[\s\S]+?\n---/', $content)) {
            return ['success' => false, 'error' => 'Missing frontmatter'];
        }

        // 检查字数
        $plainText = preg_replace('/```[\s\S]*?```/', '', $content);
        $wordCount = mb_strlen($plainText);

        if ($wordCount < 2000) {
            return ['success' => false, 'error' => "Too short: {$wordCount} chars"];
        }

        if ($wordCount > 6000) {
            return ['success' => false, 'error' => "Too long: {$wordCount} chars"];
        }

        return ['success' => true, 'word_count' => $wordCount];
    }

    private function optimizeSEO(string $path): array
    {
        $content = file_get_contents($path);

        // 确保有 description
        if (!preg_match('/^description:/m', $content)) {
            return ['success' => false, 'error' => 'Missing description'];
        }

        // 确保有 tags
        if (!preg_match('/^tags:/m', $content)) {
            return ['success' => false, 'error' => 'Missing tags'];
        }

        return ['success' => true];
    }

    private function gitAdd(string $path): array
    {
        $relativePath = str_replace($this->blogPath . '/', '', $path);

        $process = new Process(['git', 'add', $relativePath]);
        $process->setWorkingDirectory($this->blogPath);
        $process->run();

        return [
            'success' => $process->isSuccessful(),
            'error' => $process->getErrorOutput(),
        ];
    }

    private function gitCommit(string $path): array
    {
        $filename = basename($path, '.md');
        $title = str_replace('2026-06-09-', '', $filename);
        $title = str_replace('-', ' ', $title);

        $process = new Process([
            'git', 'commit', '-m', "feat: 新增文章「{$title}」"
        ]);
        $process->setWorkingDirectory($this->blogPath);
        $process->run();

        return [
            'success' => $process->isSuccessful(),
            'error' => $process->getErrorOutput(),
        ];
    }

    private function gitPush(): array
    {
        $process = new Process(['git', 'push']);
        $process->setWorkingDirectory($this->blogPath);
        $process->run();

        return [
            'success' => $process->isSuccessful(),
            'error' => $process->getErrorOutput(),
        ];
    }

    private function notifyChannels(string $path): array
    {
        // 通知飞书、Discord 等渠道
        $title = basename($path, '.md');

        // 这里调用消息工具
        return ['success' => true];
    }
}
```

## 踩坑记录

### 坑 1：AI 生成的代码不可运行

**问题**：AI 经常生成"看起来正确"但实际无法运行的代码。

**解决方案**：所有 AI 生成的代码必须本地测试。我写了一个简单的验证脚本：

```php
<?php

namespace App\Blog\Testing;

class CodeValidator
{
    /**
     * 验证代码示例是否可运行
     */
    public function validate(string $code, string $language): array
    {
        if ($language !== 'php') {
            return ['valid' => true, 'note' => 'Non-PHP code not validated'];
        }

        // 保存到临时文件
        $tempFile = tempnam(sys_get_temp_dir(), 'blog_code_');
        file_put_contents($tempFile, "<?php\n{$code}");

        // 执行语法检查
        $process = new \Symfony\Component\Process\Process([
            'php', '-l', $tempFile
        ]);
        $process->run();

        unlink($tempFile);

        return [
            'valid' => $process->isSuccessful(),
            'error' => $process->getErrorOutput() ?: null,
        ];
    }
}
```

### 坑 2：文章太长没人看

**问题**：AI 倾向于生成长内容，但技术博客读者时间有限。

**解决方案**：
- 严格控制在 3000-5000 字
- 每个代码示例不超过 30 行
- 使用 `<!-- more -->` 控制首页显示
- 关键内容前置（倒金字塔结构）

### 坑 3：SEO 优化容易被忽略

**问题**：AI 不知道你的博客 SEO 策略。

**解决方案**：在 Publish 阶段强制检查：
- title 是否包含关键词
- description 是否在 120-160 字符之间
- tags 是否相关且不超过 5 个
- 文章结构是否使用 H2/H3 层级

### 坑 4：多 Worker 并发写作冲突

**问题**：多个 Agent 同时写文章，可能出现选题重复。

**解决方案**：使用文件锁机制：
```bash
# 锁定选题
sed -i '' 's/^- \[ \] <topic>/- [x] <topic>/' .writing-backlog.md

# 检查锁定状态
grep '^\- \[x\]' .writing-backlog.md
```

## 总结

AI 时代的写作不是"让 AI 替你写"，而是**建立一套人机协作的工作流**：

1. **Plan**：用数据驱动选题，告别选择困难
2. **Architect**：用 AI 做头脑风暴，你负责结构设计
3. **Create**：分层写作，AI 负责扩展，你负责灵魂
4. **Publish**：自动化发布，减少重复劳动

关键原则：
- **AI 是工具，你是作者**：最终决定权在你
- **质量 > 数量**：AI 可以提高效率，但不能降低标准
- **持续迭代**：这套方法论本身也在不断进化

技术写作的本质是**分享知识和经验**。AI 帮我们降低了表达的门槛，但真正有价值的内容，永远来自于你的实战积累和深度思考。
