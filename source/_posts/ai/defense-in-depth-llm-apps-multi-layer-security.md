LLM 应用上线后，安全问题往往比功能问题更致命。Prompt 注入、数据泄露、越权操作、输出幻觉——任何一个环节被突破，后果都可能不可逆。

单点防御远远不够。就像传统安全领域的纵深防御（Defense in Depth）策略一样，LLM 应用需要多层独立的安全防线，每一层都假设上一层可能被突破。本文用 PHP/Laravel 实现一套完整的四层安全架构。

<!-- more -->

## 一、LLM 应用面临的核心威胁

先明确敌人是谁：

| 威胁类型 | 攻击方式 | 危害 |
|---------|---------|------|
| Prompt 注入 | 在输入中嵌入恶意指令覆盖系统提示 | AI 绕过安全策略执行恶意操作 |
| 数据泄露 | 通过巧妙提问诱导模型暴露训练数据或上下文信息 | 敏感数据外泄 |
| 越权操作 | 利用 AI 执行超出用户权限的操作 | 非授权数据访问/修改 |
| 输出投毒 | 操控 AI 生成误导性内容 | 用户信任崩塌 |
| 资源滥用 | 无限制调用消耗计算资源 | 成本失控/服务不可用 |

单靠「加个 filter」解决不了这些。我们需要分层防御。

## 二、纵深防御架构总览

```
┌─────────────────────────────────────────────────┐
│                   用户请求                        │
└──────────────────────┬──────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│  Layer 1: 输入过滤 (Input Sanitization)           │
│  - Prompt 注入检测                                │
│  - 内容合规过滤                                   │
│  - 长度/格式限制                                   │
└──────────────────────┬──────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│  Layer 2: 权限沙箱 (Permission Sandbox)           │
│  - 用户权限校验                                   │
│  - 工具调用白名单                                  │
│  - 数据访问范围控制                                │
└──────────────────────┬──────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│  Layer 3: 输出审查 (Output Sanitization)           │
│  - 敏感信息脱敏                                   │
│  - 内容合规检查                                   │
│  - 幻觉检测                                       │
└──────────────────────┬──────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│  Layer 4: 审计日志 (Audit Logging)                 │
│  - 全链路追踪                                     │
│  - 异常行为告警                                   │
│  - 合规审计                                       │
└──────────────────────┬──────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│                   最终响应                        │
└──────────────────────────────────────────────────┘
```
